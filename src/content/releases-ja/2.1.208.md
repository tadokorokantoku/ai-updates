- スクリーンリーダーモードを追加。スクリーンリーダー利用者向けのオプトイン方式プレーンテキスト表示。`claude --ax-screen-reader` を実行するか、CLAUDE_AX_SCREEN_READER=1 を設定するか、設定に "axScreenReader": true を追加することで有効化
- `vimInsertModeRemaps` 設定を追加。vim モードで `jj` のような2キーの挿入モードシーケンスを Escape にマッピング可能に
- `CLAUDE_CODE_PROCESS_WRAPPER` を追加。agent view とバックグラウンドサービスが、Claude Code の自己起動のたびに必須のラッパー実行ファイルを経由させることで、企業のランチャーに対応するようになった
- フルスクリーンモードで、複数選択メニューと「Other」入力行のマウスクリック操作に対応
- サポート対象モデルに切り替えた後も fast mode が無効のままになっていた問題を修正。設定で有効化されていれば自動的に復元されるようになった
- バックグラウンドエージェントへの返信が配信失敗時に失われていた問題を修正。テキストは保存され、セッション再開時に配信されるようになった
- 実行中の `claude agents` プロセスが起動時に使用していたバイナリがアップデートで置き換えられた後、バックグラウンドセッションへのアタッチが恒久的に失敗する（「Couldn't start the background daemon」）問題を修正
- CLI の自動アップデート後にコンテキストウィンドウ（および自動圧縮インジケーター）が一時的に200kにリセットされ、長いコンテキストのセッションを再開した際に誤った「100% context used」表示が出る問題を修正
- サーバーがリクエスト処理中に GOAWAY で HTTP/2 接続を閉じた際、supervised セッションおよびバックグラウンドセッションがクラッシュしていた問題を修正
- `claude -p` から大きなレスポンスをパイプした際に stream-json/JSON 出力が切り詰められ、result メッセージが欠落していた問題を修正
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS` などの環境変数が、指数表記の値（`1e6` が `1` になる）の仮数部だけを黙って使用していた問題を修正
- 非常に大きな markdown テーブルがレンダリングを停止させたり過剰なメモリを使用したりする問題を修正。200行を超えるテーブルは最初の200行を表示し、「…N more rows」の通知を表示する
- 読み込み後に変更されたファイルに対して、対象テキストが一意に一致する場合でも Edit ツールが失敗していた問題を修正
- Read が空ファイルを「shorter than offset」と報告していた問題、Grep が不正な正規表現パターンに対して黙って「No files found」を返していた問題、Grep のカウントモードがページネーション時に合計を過小報告していた問題、pattern・path・作業ディレクトリに null バイトが含まれる場合に Glob が不明瞭なエラーでクラッシュしていた問題を修正
- `apiKeyHelper` スクリプトの失敗が、約10回の無言リトライ後に一般的な401エラーの裏に隠れていた問題を修正。スクリプト自身のエラーが3回の試行以内に表示されるようになった
- ゲートウェイがレスポンスを変換する際、Bedrock ストリーミングリクエストが誤解を招く「Truncated event message received」で失敗していた問題を修正。エラーに content-type とプロキシへの案内が表示されるようになった
- ブラウザが開けない場合に `/upgrade` がログインフローを表示していた問題を修正。アップグレード URL を表示するようになった
- Windows 系 SDK ホストからの CRLF の空行や空白のみの行で stream-json 入力がセッションを強制終了させていた問題を修正
- `control_request` が文字列以外の `set_model` ペイロードを持つ場合、ヘッドレス stream-json セッションが恒久的にハングしていた問題を修正。CLI がエラーレスポンスで応答するようになった
- セッション再開時に「No completion record was found」の通知が繰り返し表示されていた問題を修正。孤立したバックグラウンドタスクが1つのサマリーにまとめられるようになった
- ターミナルでホストされたセッションにアタッチした Remote Control クライアントが、タスクの開始・停止まで バックグラウンドエージェントとワークフローの進捗を確認できなかった問題を修正
- サブエージェントの `tools` リストが空に解決される場合に Agent ツールがツールなしで起動していた問題を修正。認識できない項目名を示す明確なエラーを返すようになった
- `/usage` が新しいデータよりも古いキャッシュのバーを表示していた問題、設定編集後に `/mcp` がプレースホルダーサーバーを再分類していなかった問題を修正
- 実行中のバックグラウンドタスクを持つアイドルセッションで、SDK ホスト（Claude Desktop など）の「Change directory」が「A turn is in progress」で失敗していた問題を修正
- ワークフロー保存ダイアログが、user スコープの保存時に `CLAUDE_CONFIG_DIR` の場所ではなく `~/.claude/workflows/` を表示していた問題を修正
- `/release-notes` が閲覧したノートをモデルのコンテキストに追加していた問題を修正。「Show all」が以前は変更履歴全体を以降の全リクエストに注入していた
- agent view で、peek への返信送信後に貼り付けた画像が画面のライフサイクル全体で保持されていたメモリリークを修正
- プラグインのリフレッシュがクライアントのアタッチより前に実行された場合、SDK セッションが initialize リクエストで定義されたエージェントを失っていた問題を修正
- 長時間セッションでの複数のメモリリークを修正: MCP stdio サーバーの stderr がサーバーごとに最大64MB蓄積する問題、LSP ドキュメントが無期限に開いたままになる問題（50ドキュメント上限の LRU に変更）、バックグラウンド化後も保持され続ける非同期フック出力、ヘッドレス/SDK セッションでの大きなツール結果ペイロードによる無制限な増加
- offset/limit を使用して非常に長い単一行を含むファイルを読み込む際のメモリ膨張を修正。行全体を読み込む代わりにクリーンなエラーを返すようになった
- 権限の deny/ask ルールが多数あるセッションでの数秒単位のターンごとの速度低下を修正。ルールマッチャーが一度だけコンパイルされキャッシュされるようになった
- エージェントタスクリスト更新中の入力応答性を改善。タスク更新が UI 全体を再レンダリングしなくなった
- 多数の MCP ツールを持つ print/SDK セッションでのツール呼び出しごとの CPU オーバーヘッドを削減。ツールプール構築をキャッシュすることで、ツール数が多い場合のツールラウンドを最大7倍高速化
- ファイル編集の読み込みキャッシュを、最大1,000個のフルファイルを固定する代わりに16MBに制限することでメモリ使用量を削減
- セッショントランスクリプトのサイズを削減（編集の多いセッションで最大79倍）し、上書きされたファイル履歴バックアップを整理することでチェックポイントのディスク使用量を制限
- 大きな会話から生成されたバックグラウンドエージェントやフォークを持つセッションの再開時のメモリ使用量を削減
- 完了したバックグラウンドエージェントが、完了と同時に消えるのではなく、クリーンアップされるまで `/tasks` に表示され続けるようになった
- 停止したバックグラウンドエージェントへのアタッチが、空白の「Session is starting」画面ではなく、セッションのウォームアップ中にすぐトランスクリプトを表示するようになった
- バックグラウンドセッション: 古いデーモンが、新しいバージョンが起動したワーカーを黙って古いバイナリで再起動しなくなった
- agent view: Ctrl+X が、リネームされたブランチの worktree を削除するようになり、未プッシュのコミットを破壊することがなくなり、worktree を保持した場合はセッション行が残るようになり、再利用された worktree 名が現在のベースにリセットされるようになった
- `$(…)`/バッククォート/`<(…)` を含むコマンド内の破壊的な削除（例: `rm -rf ~`）が、通常形式と同様に `--dangerously-skip-permissions` と auto mode で確認を求めるようになった
- `/install-github-app` と `/mcp` の設定メニューがバックグラウンドセッションで開かなくなった
- 空の URL で設定された MCP サーバーが、設定エラーではなく `/mcp` で「not configured」と表示されるようになった
- usage エンドポイントがレート制限されている場合、`/usage` がエラー画面ではなく「as of」の注記付きで最後に取得した使用量バーを表示するようになった
- sso_region が Bedrock リージョンと異なる AWS SSO プロファイルで、Bedrock 認証が「Session token not found or invalid」で失敗していた問題を修正（2.1.207 のリグレッション）
