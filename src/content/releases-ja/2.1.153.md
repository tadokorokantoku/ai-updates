- `github`/`git` プラグインマーケットプレイスソースに `skipLfs` オプションを追加。クローンおよびアップデート時に Git LFS ダウンロードをスキップできる
- npm グローバルインストールで自動アップデートが失敗した場合、Claude Code が一度だけ通知を表示するようになった。`/doctor` で修正方法を確認できる
- ステータスラインコマンドが `COLUMNS` と `LINES` 環境変数を受け取るようになった。スクリプトが出力をターミナル幅に合わせてサイズ調整できる
- `claude agents`: ディスパッチ入力のオートコンプリートがプロジェクトスキルだけでなく、ネイティブスラッシュコマンドとバンドルスキルも提案するようになった
- `claude agents`: PR カラムが単一 PR の場合は `PR #N`、複数の場合は `N PRs` を表示するようになった
- `claude doctor` が最後のアップデート試行結果を表示するようになった
- MCP サーバーとコネクターの「認証が必要」起動通知を単一メッセージに統合した
- macOS: バックグラウンドエージェントがプライバシーとセキュリティに "Claude Code" として表示されるようになり、アップグレード後も権限付与が維持されるようになった
- オプションの GET SSE ストリームを持たないステートフル MCP サーバーが `tools/list` で再接続ループする問題を修正（v2.1.147 のリグレッション）
- カスタム API ゲートウェイがゲートウェイ自身のトークンの代わりにユーザーの Anthropic OAuth クレデンシャルを受け取るリグレッションを修正
- サブエージェント（Agent ツール）の frontmatter MCP サーバーが `--strict-mcp-config`、`--bare`、リモートモード、エンタープライズ管理 MCP 設定、マネージド設定の MCP サーバー許可/拒否ポリシーを無視する問題を修正
- `--strict-mcp-config` が明示的に渡されたエージェント定義（`--agents` / SDK `agents`）のインライン `mcpServers` を除去しなくなった。ブロックされたサブエージェント MCP サーバーで警告が表示されるようになった
- Windows PowerShell インストーラーがインストール失敗時に "Installation complete!" と報告する問題を修正
- npm インストールで `claude update` が設定されたリリースチャンネルのバージョンではなく最新バージョンをインストールする問題を修正
- 多数のセッションが保存されているマシンでトランスクリプトファイルパスによるセッション再開時に過大なメモリ使用量（数 GB）が発生する問題を修正
- アップグレード後も `claude agents` と `claude --bg` がバイナリ引き継ぎサポート前に起動した古いデーモンで実行される問題を修正
- stream-json モードで EOF なしに stdin が閉じられた場合に CLI が終了できなくなるハングを修正。古いセッションマーカーが残る問題も解消
- Claude のレスポンス内の不正な `file://` リンクがターミナルでクリックできない問題を修正
- 92 カラム未満のターミナルで `claude --help` が折り返されない出力をレンダリングする問題を修正
- MCP ツールの進捗通知が折りたたまれたツールビューでレンダリングされない問題を修正
- `subagent_type: 'claude'` の `Agent` ツールが非公開の一時ワークツリーで実行され、gitignore 対象パスへの出力が暗黙的に破棄される問題を修正
- Claude がレスポンス中に `/bg` を実行するとレスポンスがバックグラウンドセッションで継続されるようになった（以前はドロップされていた）
- タスク実行中にバックグラウンドセッションで `/btw` キーボードショートカットが反応しなくなる問題を修正
- バックグラウンドセッションが `$CLAUDE_JOB_DIR` に一時ファイルを書き込む際に "sensitive file" 権限プロンプトが表示される問題を修正
- 作業ディレクトリが削除されたバックグラウンドエージェントを復元する際に、明確なエラーメッセージの代わりに短縮されたスタックトレースが表示される問題を修正
- バックグラウンドセッションで `EnterWorktree` が即座に利用できない問題を修正（以前は `ToolSearch` が先に必要だった）
- iTerm2/Terminal.app で `cmd+k` がアタッチされたバックグラウンドセッションを再描画しない問題を修正
- Windows のアタッチされたバックグラウンドセッションで IME 候補ウィンドウが入力キャレット横ではなく画面下部に表示される問題を修正
- エージェントがファイル差分をレンダリングした後、256 色のみのターミナルからバックグラウンドエージェントにアタッチすると背景色がにじむ問題を修正
- tmux 内のバックグラウンドセッションにアタッチ中に `/copy` とコピーオンセレクトがシステムクリップボードの更新に暗黙的に失敗する問題を修正
- Remote Control が有効な状態で `claude agents` を開くと、終了後に Code タブにゾンビセッションエントリが残る問題を修正
- バックグラウンドセッションで `/rename` がセッションバナーを即座に更新しない問題を修正
- Windows アップデートのロールバックを修正: Windows アップデートが失敗した場合、Claude Code が元の実行ファイルをコピーで復元し、回復方法を案内するようになった
- [VSCode] VS Code が Windows で閉じられた際に Claude Code プロセスが正常にシャットダウンしない問題を修正。"unclean exit" の誤報告と孤立した MCP サーバーが解消された
- `/model` がモデル選択を新規セッションのデフォルトとして保存するようになった（IDE と同様）。`s` キーで現在のセッションのみのモデル切り替えが可能
- `modelPicker:setAsDefault` キーバインドをカスタマイズしている場合は keybindings.json で `modelPicker:thisSessionOnly` にリネームが必要（`d` アクションが `s` に置き換えられた）
