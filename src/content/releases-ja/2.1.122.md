- Bedrock サービスティア（`default`、`flex`、`priority`）を選択する `ANTHROPIC_BEDROCK_SERVICE_TIER` 環境変数を追加 — `X-Amzn-Bedrock-Service-Tier` ヘッダーとして送信されます
- `/resume` の検索ボックスに PR の URL を貼り付けると、その PR を作成したセッションを検索できるようになりました（GitHub、GitHub Enterprise、GitLab、Bitbucket 対応）
- `/mcp` が、同じ URL を持つ手動追加サーバーによって非表示になっていた claude.ai コネクタを表示し、重複を削除するヒントを表示するようになりました
- ブラウザのサインインフロー後も MCP サーバーが未認証のままの場合に表示される `/mcp` のメッセージを明確化しました
- OpenTelemetry: `api_request`/`api_error` ログイベントの数値属性が文字列ではなく数値として送出されるようになりました
- OpenTelemetry: `@`-メンション解決のための `claude_code.at_mention` ログイベントを追加しました
- ソースセッションに巻き戻したタイムラインのエントリが含まれている場合、`/branch` が「tool_use ids were found without tool_result blocks」で失敗するフォークを生成する問題を修正
- Bedrock アプリケーション推論プロファイル ARN に対して `/model` が Effort オプションを表示せず、それらの ARN が `output_config.effort` を受け取らない問題を修正
- Vertex AI / Bedrock がセッションタイトル生成などの構造化出力クエリで `invalid_request_error: output_config: Extra inputs are not permitted` を返す問題を修正
- プロキシゲートウェイ経由のユーザーに対して Vertex AI の `count_tokens` エンドポイントが 400 エラーを返す問題を修正
- `spinnerTipsOverride.excludeDefault` が時間ベースのスピナーヒントを抑制しない問題を修正
- 非ブロッキングモードでセッション開始後に接続した MCP ツールが ToolSearch で見つからない問題を修正
- bash モードで `!exit` / `!quit` がシェルコマンドとして実行される代わりに CLI を終了してしまう問題を修正
- 新しいモデルに送信された画像が正しい最大 2000px ではなく 2576px にリサイズされる問題を修正
- リモートコントロールセッションのアイドルステータスが毎秒 2 回再描画され、`tmux -CC` 制御パイプが溢れてターミナルが一時停止する問題を修正
- 古いビュー設定によりアシスタントメッセージが一部のセッションで空白表示される問題を修正
- `settings.json` の不正な hooks エントリがファイル全体を無効にしないよう修正
- Voice モード: ターミナルは Caps Lock をキーイベントとして送信しないため、Caps Lock にバインドされたキーバインドでエラーを表示するようになりました
