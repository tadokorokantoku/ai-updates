- `claude agents --json` に `waitingFor` が追加され、待機中のセッションが何でブロックされているか（例: パーミッションプロンプト）を表示するようになった
- `--tools`: Grep/Glob を明示的に列挙すると、ネイティブビルドで組み込み検索の専用ツールが提供されるようになった（以前はこれらの名前がサイレントに無視されていた）
- `/effort` で選択したレベルが新しいセッションのデフォルトとして永続化される場合に確認メッセージを表示するようになった
- オートコンプリートメニューでスラッシュコマンドをクリックすると即時実行せずにプロンプトに入力されるようになった。Enter を押して実行する
- Remote Control がスタートアップメッセージではなく永続的なフッターピル（セッションへのリンク付き）として表示されるようになった
- エディタのリブランドに伴い、`/ide` メニュー、`/terminal-setup`、`/scroll-speed` で Windsurf が Devin Desktop に名称変更された
- 設定ディレクトリが読み取り専用または書き込み不可の場合にサイレントなスタートアップハングが発生していた問題を修正 — Claude Code がインメモリ設定で起動し、スタートアップエラーを表示するようになった
- WebFetch パーミッションルールが組み込みの事前承認済みドメインに適用されていなかった問題を修正。明示的な `WebFetch(domain:...)` の deny/ask/allow ルールが事前承認ホストの自動許可より優先されるようになった
- Windows のパーミッションルールがバックスラッシュ（`~\`、`\\server\share`）や大文字小文字の違いで一致しなかった問題と、Read deny ルールが Glob/Grep の結果からファイルを非表示にしなかった問題を修正
- stream-json/SDK セッションでターンの開始直後に割り込み（Esc）を送信すると「Interrupted」フィードバックなしでサイレントにドロップされていた問題を修正
- 絵文字を含むトランケーション境界付近の分類器サイドクエリや MCP サーバーの説明で API 400 `no low surrogate in string` エラーが発生していた問題を修正
- MCP のサーバーごとの `timeout` 設定値が 1000 ms 未満の場合に全ツール呼び出しを中断する 1 秒のウォッチドッグに切り上げられていた問題を修正。1000 ms 未満の値は無視され（`MCP_TOOL_TIMEOUT` またはデフォルトにフォールバック）、`claude mcp get` でそれが注記されるようになった
- LSP ツールの `workspaceSymbol` 操作が結果を返さなかった問題を修正。`query` パラメータを受け入れてランゲージサーバーに渡すようになった
- `claude agents` が広いターミナルでライブステータステキスト（ツール引数、返信、プロンプト、実行出力）を 60〜120 カラムで切り詰めていた問題を修正。ステータス詳細がターミナルの全幅を使用するようになった
- `claude agents` が長いセッション名を 40 カラムで切り詰めていた問題を修正。名前カラムがターミナル幅に応じて広がるようになった
- `claude agents` のアタッチがバックグラウンドサービスの再起動後に最初の試行でセッションリストに戻ることがあった問題を修正
- `claude agents` でディスパッチ入力やセッション返信ボックスで Ctrl+V による画像ペーストが機能しなかった問題を修正。画像なしでペーストした場合にヒントを表示するようになった
- ← でセッションをバックグラウンドにした際にバックグラウンドサービスが起動できない場合に会話がサイレントに失われていた問題を修正。セッションがリストに失敗行として残り、Enter で起動できるようになった
- エージェントビューからの返信が送信失敗した場合に失われていた問題を修正。次のセッション開始時に配信のためにキューに入れられるようになった
- `CLAUDE_CODE_TMPDIR` または `$TMPDIR` が深いディレクトリを指している場合にクロスセッションメッセージング（`SendMessage`）がサイレントに機能しなくなっていた問題を修正
- `claude agents` から実行中のバックグラウンドセッションを開くと 5 秒間停止してからアタッチされていた問題を修正
- スタートアップの静粛化: 通知が重要度でグループ化され、セッション情報とアナウンスが起動ごとに 1 行にまとめられるようになった
- スタートアップ警告を短く明確に書き直し、それぞれに具体的な修正方法を添付
- 起動プロンプト警告（ディープリンク/事前入力プロンプト）が操作するまで入力の下にピン留めされ、スクロールで見えなくなることがなくなった
- 失敗したターンが複数行の赤いエラーブロックではなくコンパクトな警告行を表示するようになった
- バックグラウンドサービスの起動と `claude update` の検証が、新しいバイナリのエンドポイントセキュリティスキャンを 5 秒で失敗するのではなく待機するように改善された
- バックグラウンドディスパッチのスポーン失敗時に errno が利用できない場合にエラークラス名を報告するようになった
- 「Claude in Chrome enabled」と「marketplace installed」のスタートアップメッセージを削除。モデルの自動更新とチームオンボーディングのヒントがロゴの下に静かな通知として表示されるようになった
