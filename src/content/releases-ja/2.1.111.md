- Claude Opus 4.7 xhigh が利用可能に。/effort で速度と知性のバランスを調整可能
- Opus 4.7 使用時に Max サブスクライバー向けの auto モードが利用可能に
- Opus 4.7 向けに `high` と `max` の間に位置する `xhigh` エフォートレベルを追加。`/effort`、`--effort`、モデルピッカー経由で利用可能。他のモデルは `high` にフォールバック
- `/effort` を引数なしで呼び出すとインタラクティブスライダーを表示。矢印キーでレベル間を移動し Enter で確定
- ターミナルのダーク/ライトモードに合わせる "Auto (match terminal)" テーマオプションを追加 — `/theme` から選択
- `/less-permission-prompts` スキルを追加 — トランスクリプトをスキャンして一般的な読み取り専用の Bash/MCP ツール呼び出しを検出し、`.claude/settings.json` 向けの優先度付き許可リストを提案
- `/ultrareview` を追加 — クラウド上で並列マルチエージェント分析と批評を使用した包括的なコードレビューを実行。引数なしで現在のブランチをレビュー、`/ultrareview <PR#>` で特定の GitHub PR を取得してレビュー
- auto モードが `--enable-auto-mode` を必要としないように
- Windows: PowerShell tool を段階的にロールアウト中。`CLAUDE_CODE_USE_POWERSHELL_TOOL` でオプトイン/アウト。Linux と macOS では `CLAUDE_CODE_USE_POWERSHELL_TOOL=1` で有効化 (PATH に `pwsh` が必要)
- グロブパターン付きの読み取り専用 bash コマンド (例: `ls *.ts`) と `cd <project-dir> &&` で始まるコマンドが権限プロンプトを発生させないように
- `claude <word>` がタイポに近い場合、最も近いサブコマンドを提案 (例: `claude udpate` → "Did you mean `claude update`?")
- プランファイル名を純粋にランダムな単語ではなくプロンプトに由来するものに (例: `fix-auth-race-snug-otter.md`)
- `/setup-vertex` と `/setup-bedrock` を改善: `CLAUDE_CONFIG_DIR` 設定時に実際の `settings.json` パスを表示、再実行時に既存のピンからモデル候補をシード、対応モデルに "with 1M context" オプションを提供
- `/skills` メニューで推定トークン数でのソートをサポート — `t` で切り替え
- `Ctrl+U` が入力バッファ全体をクリアするように (以前: 行頭まで削除)。`Ctrl+Y` で復元
- `Ctrl+L` がプロンプト入力のクリアに加えて画面全体の再描画を強制するように
- トランスクリプトビューのフッターに `[` (scrollback にダンプ) と `v` (エディタで開く) のショートカットを表示
- 長い貼り付けの切り詰めを示す "+N lines" マーカーを、スキャンしやすい全幅ルールに変更
- ヘッドレス `--output-format stream-json` が、未満足の依存関係でプラグインが降格された場合に init イベントで `plugin_errors` を含むように
- デバッグ用に完全な API リクエスト/レスポンスボディを OpenTelemetry ログイベントとして出力する `OTEL_LOG_RAW_API_BODIES` 環境変数を追加
- 通常動作中に TUI に表示されうる偽の展開、ネットワーク、一時的エラーメッセージを抑制
- v2.1.110 の非ストリーミングフォールバックリトライの上限を元に戻し — API 過負荷時の長時間待機を明示的な失敗と引き換えにしていたため
- ターミナル通知送信時に iTerm2 + tmux 環境でターミナル表示が破れる (ランダム文字、入力のズレ) 問題を修正
- 非 git 作業ディレクトリで `@` ファイル候補が毎ターン全プロジェクトを再スキャンする問題、および追跡ファイルのない新規 git リポジトリで config ファイルのみ表示される問題を修正
- 編集前の LSP 診断が編集後に表示され、モデルが編集したばかりのファイルを再読み込みする問題を修正
- `/resume` のタブ補完がセッションピッカーを表示せず、任意のタイトル付きセッションをすぐに再開する問題を修正
- `/context` グリッドが行間に余分な空行を入れてレンダリングされる問題を修正
- `/clear` が `/rename` で設定したセッション名を破棄し、statusline 出力で `session_name` が失われる問題を修正
- プラグインエラー処理を改善: 依存関係エラーが conflicting、invalid、過度に複雑なバージョン要件を区別。`plugin update` 後の解決済みバージョンの古さを修正。`plugin install` が中断された前回のインストールから回復可能に
- 存在しない `commit` スキルを Claude が呼び出し、カスタム `/commit` コマンドを持たないユーザーに "Unknown skill: commit" を表示する問題を修正
- Bedrock/Vertex/Foundry の 429 レートリミットエラーが status.claude.com を参照する問題を修正 (Anthropic 運用プロバイダのみが対象)
- フィードバックサーベイを 1 つ閉じた後に連続で表示される問題を修正
- bash/PowerShell/MCP ツール出力の裸 URL がターミナルで行をまたいで折り返された際にクリックできない問題を修正
- Windows: `CLAUDE_ENV_FILE` と SessionStart フック環境ファイルが適用されるように (以前は no-op)
- Windows: ドライブレターパスを含む権限ルールが正しくルートアンカーされ、ドライブレターの大文字小文字のみ異なるパスが同じパスとして認識されるように
