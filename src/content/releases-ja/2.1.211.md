- stream-json 出力にサブエージェントのテキストと思考を含めるための `--forward-subagent-text` フラグと `CLAUDE_CODE_FORWARD_SUBAGENT_TEXT` 環境変数を追加
- チャットチャンネルに中継される permission プレビューが双方向オーバーライド文字・ゼロ幅文字・見た目が似た引用符文字を無害化していなかった問題を修正 — これによりツール入力が承認メッセージを視覚的に改ざんできてしまう可能性があった
- サンドボックス化されていない Bash に対する PreToolUse フックの `ask` 判定を auto mode が上書きしてしまう問題を修正 — フックの `ask` は常にプロンプト表示を下限とするように
- 多数のセッションが1つの認証情報ストアを共有している場合、スリープからの復帰後に並行する Claude Code セッションがすべて同時にログアウトしてしまう問題を修正
- アイドル状態の web セッションが復帰した後、プラグインの MCP サーバーが再接続されず、次のメッセージまで MCP 呼び出しが失敗し続ける問題を修正
- モデルが明示的に設定されている場合でも、Vertex および Bedrock 上の Claude Code が起動時にデフォルトの Opus モデルを試行し、誤ったフォールバック通知を表示してしまう問題を修正
- 明示的なモデルオーバーライドで起動されたサブエージェントが、再開時やフォローアップメッセージ送信時に親のモデルに戻ってしまう問題を修正
- 設定ソースがプロジェクト設定を除外している場合でも、ネストされた `.claude/rules/*.md` ファイルが読み込まれてしまう問題を修正
- ファイルアップロードの検証を修正: DOS デバイスサフィックス（`.prn`）や末尾のドットで終わるファイル名が許可されるようになり、複数のハードリンクを持つファイルは拒否されるように
- リモートおよび CLI セッションから Claude in Chrome へのファイルアップロードを修正
- 入力が「?」のままになる編集がサイレントに無視され、ショートカットパネルが切り替わってしまう問題を修正
- Claude in Chrome 拡張機能が有効だが Chrome が起動していない場合の起動時ハングを修正
- 非同期コンテンツ（Settings のタブ、Stats、diff ビュー、その他の読み込み状態）が表示されるまでの 300ms の遅延を修正
- エージェントビューから停止直後のバックグラウンドセッションを再度開くと、同じセッション ID のまま空の会話が開始されてしまう問題を修正
- `/loop` を一度使用した後、セッションが `/resume` から見えなくなる問題を修正
- `/terminal-setup` またはオンボーディングのターミナルセットアップ後、スクリーンリーダーユーザーが可聴のターミナルベルを失う問題を修正
- デーモンが再起動した後、LLM ゲートウェイ認証（`ANTHROPIC_AUTH_TOKEN` + `ANTHROPIC_BASE_URL`）を使用するバックグラウンドジョブが「Not logged in」に戻ってしまう問題を修正
- git がその worktree を認識できなくなった場合に `claude agents` のジョブが永久に削除不能になる問題を修正 — 該当行にはサイレントに再表示される代わりに削除が拒否された理由が表示されるように
- `/clear` がセッションのコスト カウンターをリセットしていなかった問題を修正 — statusline のコストは `/clear` 後に $0 から始まるように
- Windows で Claude in Chrome のセットアップページがブラウザで開けない問題を修正
- stdin が読み取れない場合に Windows 上のヘッドレス print モードセッションがクラッシュまたはサイレントに終了してしまう問題を修正
- プロンプトにリンクが含まれる場合、エージェントビューのバックグラウンドセッションタイトルに命名モデルの拒否テキストが表示されてしまう問題を修正
- ユーザーによって kill されたバックグラウンドエージェントが自動的に再起動してしまう問題と、復元されたエージェントが古いセッションの古びたプロンプトを再実行してしまう問題を修正
- スケジュールのないルーチンが西暦1年の次回実行時刻を報告してしまう問題を修正
- Windows での同期済みスキル/プラグインディレクトリの命名を強化し、`/clear` 後も CCR の web fetch/search プロキシが機能し続けるように
- ターミナルのレイアウトとレンダリングのパフォーマンスを改善
- バックグラウンドエージェントの結果報告を改善 — Claude は結果を捏造する代わりに、実行中のエージェントの状態を報告し実際の完了を待つように
- メモリインデックスの上限超過警告を、frontmatter と HTML コメントを除いた読み込み済みコンテンツのみで測定するように改善
- タイムアウト・トークン予算・リトライ回数などの整数の環境変数が、`1e6` や `64_000` のような科学的記数法や桁区切り表記を受け付けるように更新
- ドキュメントリンクを最新のドキュメントサイトに更新
- 「always allow」の permission ルールがリポジトリのルートに保存されるように変更 — これにより git worktree で許可された承認がセッションや worktree をまたいで永続化されるように
- `/usage-credits` が組織管理者にリクエストを送信する前に確認を求めるように変更
- Vim モードの `s` と `S`（文字/行の置換）が vim の挙動に合わせて NORMAL モードで機能するように変更
- [VSCode] Remote Control のバナーがその機能を説明するように更新
- Claude in Chrome: ファイルアップロードのパス検証を強化
- Claude in Chrome: スクリーンショットアクションの `save_to_disk` が画像をディスクに書き込みパスを返すように — 以前は何も行っていなかった
- Bedrock、Vertex、Mantle、Foundry でのプロンプトキャッシュの回帰を修正 — 末尾のシステムコンテキストブロックがリクエストのたびに新規の入力トークンとして課金されていた
