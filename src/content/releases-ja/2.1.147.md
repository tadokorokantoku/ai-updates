- 決定論的なマルチエージェントオーケストレーション向けに `Workflow` ツールを追加した。デフォルトでは無効で、`CLAUDE_CODE_WORKFLOWS=1` を設定することで有効になる
- `claude agents` でピン留めされたバックグラウンドセッション（`Ctrl+T`）がアイドル時も維持されるようになり、Claude Code のアップデート適用時はその場で再起動され、メモリ逼迫時にはピン留めされていないセッションより後に解放されるようになった
- REPL および Workflow ツールのサンドボックスをプロトタイプ汚染と thenable ベースのエスケープに対して堅牢化した
- 自動アップデーターを改善した。一時的なネットワーク障害のリトライに加え、失敗時に具体的なエラーカテゴリと OS エラーコードを報告するようになった
- プロンプト履歴で連続した重複エントリが記録されなくなった。上矢印キーでプロンプトを呼び出して再送信しても履歴に追加されなくなる
- `!` コマンドの出力中の `&` が `&amp;` として表示される問題を修正した。これにより、ヘッドレスマシンで `gcloud auth login` などのコマンドからの URL のコピー貼り付けが壊れていた
- ヘッドレス/SDK モードで未知のスラッシュコマンドが何もしないまま無視される問題を修正した。エラーメッセージが表示されるようになった
- フルスクリーン以外の小さいターミナルで `/help` が壊れたタブヘッダーをレンダリングし、1 ページにコマンドが 1 つしか表示されない問題を修正した
- シェルスナップショットで名前がアンダースコア 1 つで始まるユーザー関数が消える問題を修正した。これらを参照するエイリアスが壊れていた
- `tools:` frontmatter で複数の `Agent(...)` タイプを宣言しているプラグインエージェントが最後のエントリ以外を削除する問題を修正した
- `PowerShell(git push*)` のようなフック `if` 条件が一致しない問題を修正した。以前は `PowerShell(*)` のみが機能していた
- デフォルトフォーマッターに依存するコマンドで PowerShell ツールが出力を削除する問題を修正した
- Windows で PowerShell スクリプト実行に対して「はい、今後は聞かない」を選択すると、以降の実行で実際に一致するルールが書き込まれるようになった
- `/effort` がスライダーを間違ったレベルで開く問題を修正した。現在のエフォートレベルで開始されるようになった
- Windows でスクロールの安定を待つ際に発生するまれなハングを修正した
- バックグラウンドセッションの結果に全角（CJK）文字が含まれる場合に Windows でエージェントビューリストに古い行や重複した行が表示される問題を修正した
- ペーストされたテキストが実際のコンテンツではなく `[Pasted text #N]` プレースホルダーとしてエージェントに配信される問題を修正した
- プラグインのマニフェストがデフォルトディレクトリと重複するパスを列挙する場合に `claude plugin details` と `/plugin` のプラグインコンポーネント数が 2 倍になる問題を修正した
- タブまたは改行が後続するスラッシュコマンドが未知のコマンドとして処理される問題を修正した
- `/plugin`、`/status`、`/mobile`、`/sandbox`、`/permissions` メニューのいくつかのスペースとレイアウトの不具合を修正した
- ストリップされた画像がモデルに存在しないメディアを繰り返し再読み込みするよう促す問題を修正した
