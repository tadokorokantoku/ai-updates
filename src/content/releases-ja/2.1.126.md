- `ANTHROPIC_BASE_URL` が Anthropic 互換ゲートウェイを指している場合、`/model` ピッカーがゲートウェイの `/v1/models` エンドポイントからモデル一覧を表示するように
- `claude project purge [path]` を追加 — プロジェクトの Claude Code 状態（トランスクリプト、タスク、ファイル履歴、設定エントリ）をすべて削除。`--dry-run`・`-y/--yes`・`-i/--interactive`・`--all` に対応
- `--dangerously-skip-permissions` が `.claude/`・`.git/`・`.vscode/`・シェル設定ファイルなど従来保護されていたパスへの書き込みプロンプトもバイパスするように（致命的な削除コマンドは安全のため引き続きプロンプトを表示）
- `claude auth login` がターミナルへのペーストによる OAuth コード入力に対応 — ブラウザコールバックが localhost に届かない環境（WSL2、SSH、コンテナ）向け
- `claude_code.skill_activated` OpenTelemetry イベントがユーザー入力のスラッシュコマンドでも発火し、`invocation_trigger` 属性（`"user-slash"`・`"claude-proactive"`・`"nested-skill"`）を付与するように
- Auto モード：権限チェックで処理が止まった場合、スピナーが赤色に変化し、ツール実行中と区別できるように
- ホスト管理型デプロイ（`CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST`）が Bedrock/Vertex/Foundry で analytics を自動無効化しないように修正

- Windows：PowerShell ツールが有効な場合、Bash ではなく PowerShell をプライマリシェルとして扱うように
