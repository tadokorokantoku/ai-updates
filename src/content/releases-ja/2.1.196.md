- 組織のデフォルトモデルをサポート — 管理者が組織コンソールで設定すると、自分でモデルを選択していない場合に `/model` に「Org default」（または「Role default」）と表示される
- セッション開始時に分かりやすいデフォルト名を付与し、識別やメッセージ送信をしやすくした
- チャット内のファイル添付をクリック可能に — Cmd/Ctrl クリックで Finder/Explorer 内のファイルを表示
- セキュリティ: `claude mcp list`/`get` が、コミットされた `.claude/settings.json` でリポジトリが自己承認した `.mcp.json` サーバーを起動しないように変更。信頼されていないワークスペースでは「⏸ Pending approval」と表示される
- トランスクリプトの判定処理が実在するトランスクリプトを誤検知し、バックグラウンドジョブの起動時に会話が完全に削除されて元のプロンプトが再実行されてしまう問題を修正。該当ファイルは削除されず退避されるようになった
- 複数の並列リクエストが進行中に使用量制限に達した際、レート制限警告が点滅して消える問題と、レート制限のテレメトリが過剰にカウントされる問題を修正
- バックグラウンドセッションのターン後に重複した recap 行が表示される問題を修正: スキーマ拒否された StructuredOutput の試行が、そのリトライと並んで表示されなくなった
- PowerShell での `git diff`/`git grep`、`egrep`/`fgrep`、および `|` を含む引用検索パターンが、終了コード 1 のときに Bash と同じ挙動にならず失敗として報告される問題を修正
- `claude agents` サイドパネルの複数の問題を修正: エージェントを開いたときにキーボードフォーカスが固まる問題、バックグラウンドジョブが開くたびにサブエージェントタイプを失う問題、実行中のセッションのステータスが誤って表示される問題
- `claude agents --dangerously-skip-permissions` が bypass の免責表示を出さず、生成されたエージェントに bypass モードを適用せずに、サイレントに auto モードへフォールバックしていた問題を修正
- Remote セッションのターン途中のクラッシュ復旧を修正 — サーバー再起動で中断されたセッションが、次のワーカーで自動的に再開するようになった
- `/cd` で移動したセッションが、特殊文字を含む旧パスで非グレースフル終了後に旧ディレクトリの再開リストに再表示される問題を修正
- `claude plugin validate` がソースが "." のローカルプラグインをスキップし、最初のエラークラスで停止していた問題を修正
- アイドル状態のプロンプトで Esc Esc を押しても rewind メニューが開かない問題（リグレッション）を修正。バックグラウンドエージェントの停止には Ctrl+C または Ctrl+X Ctrl+K を使用する
- スコープが指定されていない場合に MCP OAuth が認可サーバーの `scopes_supported` カタログ全体を要求し、GitLab セルフホストなどのエンタープライズ IdP で `invalid_scope` エラーを引き起こす問題を修正
- Bedrock で `/context` がすべてのツールグループに対して 0 トークンと表示される問題を修正
- `/deep-research` が verifier の失敗を `unverified` ではなく「すべての主張が反証された」と誤報告する問題を修正
- マーケットプレイスが git リポジトリに紐づくローカルフォルダパスとして追加されている場合に、プラグインの依存バージョン固定が反映されない問題を修正
- `claude agents` のセッションステータスを修正: 完了した行が「Done」と「Needs your input」の間でちらつかなくなり、停止したエージェントは「Needs attention」とラベル付けされ、PR に言及する結果はクリック可能なリンクとして表示される
- 音声モード有効時、非常に高速なタイピング中に音声ディクテーションがスペースを欠落させたり、誤って録音を開始したりする問題を修正
- バックグラウンドセッションの信頼性を改善: 長時間実行されるコマンドやワークフローが、セッションのプロセスの停止・再起動・更新を経ても継続するようになった（Windows でもバックグラウンドシェルが強制終了されずに引き継がれる）
- バックグラウンドエージェントを改善: デーモン再起動で停止したワーカーが、次回エージェントビューを開いたときに中断箇所から自動的に再開するようになった
- `/code-review` ワークフローを改善: 5 つのクリーンアップ用ファインダーを 1 つに統合し、トークン使用量を約 25% 削減
- ストリーミング中の no-op なサブツリー走査をスキップすることで、ターミナル UI のフレームごとのレンダリング処理を削減
- ストリーミングのアイドルウォッチドッグが全プロバイダーでデフォルトで有効化された — レスポンスストリームが 5 分間イベントを生成しない場合に中断してリトライする。無効化するには `CLAUDE_ENABLE_STREAM_WATCHDOG=0` を設定する
- `ANTHROPIC_BASE_URL` が Anthropic 以外のホストを指している場合、`CLAUDE_CODE_USE_BEDROCK`/`_VERTEX`/`_FOUNDRY` での既存の挙動と同様に Remote Control が無効化されるようになった
- フォアグラウンドセッションからエージェントビューを開く操作を変更: バックグラウンドセッションと同様、`←` を 2 回ではなく 1 回押すだけでよくなった
