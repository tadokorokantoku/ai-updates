- Mantle による Amazon Bedrock のサポートを追加。`CLAUDE_CODE_USE_MANTLE=1` を設定
- API キー、Bedrock/Vertex/Foundry、Team、Enterprise ユーザー向けのデフォルト effort レベルを medium から high に変更（`/effort` で制御）
- Slack MCP send-message ツール呼び出し用のクリック可能なチャンネルリンク付きコンパクト `Slacked #channel` ヘッダーを追加
- プラグイン出力スタイルの `keep-coding-instructions` frontmatter フィールドサポートを追加
- セッションタイトルを設定する `UserPromptSubmit` フック用の `hookSpecificOutput.sessionTitle` を追加
- `"skills": ["./"]` 経由で宣言されたプラグインスキルがディレクトリの basename ではなくスキルの frontmatter の `name` を invocation name として使用するように。インストール方法をまたいで安定した名前を提供
- 長い Retry-After ヘッダー付きの 429 レート制限応答後にエージェントが stuck のように見える問題を修正 — エラーが無言で待機する代わりに即座に表面化
- ログイン keychain がロックされているか、そのパスワードが同期外れの場合、macOS 上の Console ログインが無言で "Not logged in" で失敗する問題を修正 — エラーが表面化し、`claude doctor` が修正を診断
- YAML frontmatter で定義されたプラグインスキルフックが無言で無視される問題を修正
- `CLAUDE_PLUGIN_ROOT` が設定されていないとプラグインフックが "No such file or directory" で失敗する問題を修正
- 起動時にローカルマーケットプレイスプラグインの `${CLAUDE_PLUGIN_ROOT}` がインストールキャッシュではなくマーケットプレイスソースディレクトリに解決される問題を修正
- 長時間実行セッションでの同じ diff の繰り返しと空白ページの scrollback 表示を修正
- トランスクリプト内の複数行ユーザープロンプトでラップされた行がテキストの下ではなく `❯` キャレットの下にインデントされる問題を修正
- 検索入力で Shift+Space がスペース文字ではなくリテラル "space" という単語を挿入する問題を修正
- xterm.js ベースのターミナル（VS Code、Hyper、Tabby）で実行中の tmux 内でクリックするとハイパーリンクが 2 つのブラウザタブを開く問題を修正
- コンテンツ高さの変更がスクロールの途中で ghost 行を複合的に残す alt-screen レンダリングバグを修正
- `settings.json` の `env` 経由で設定されたときに `FORCE_HYPERLINK` 環境変数が無視される問題を修正
- ダイアログで選択されたタブをネイティブターミナルカーソルが追跡しない問題を修正。スクリーンリーダーとルーペがタブナビゲーションに従えるように
- `us.` 推論プロファイル ID を使用して Sonnet 3.5 v2 の Bedrock invocation を修正
- ストリーミング途中で中断されたとき、SDK/print モードが部分的なアシスタント応答を会話履歴に保持しない問題を修正
- `cd` コマンドを印字する代わりに、同じリポジトリの他の worktree からのセッションを直接再開するよう `--resume` を改善
- チャンク境界が UTF-8 シーケンスを分割したときに stream-json 入出力で CJK と他のマルチバイトテキストが U+FFFD で破損する問題を修正
- [VSCode] セッション開始時のコールドオープンサブプロセス作業を削減
- [VSCode] タイピングまたは矢印キー使用中にマウスがリスト上にあるとドロップダウンメニューが間違った項目を選択する問題を修正
- [VSCode] `settings.json` ファイルの解析が失敗したときに警告バナーを追加。権限ルールが適用されていないことをユーザーが把握できるように
