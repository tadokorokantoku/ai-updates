- `worktree.baseRef` 設定（`fresh` | `head`）を追加 — `--worktree`、`EnterWorktree`、エージェント分離ワークツリーが `origin/<default>` またはローカル `HEAD` のどちらからブランチするかを選択できる。**注意:** デフォルトの `fresh` は `EnterWorktree` のベースを `origin/<default>` に戻す（2.1.128 以降はローカル `HEAD` だった）— 新しいワークツリーに未プッシュのコミットを含めたい場合は `worktree.baseRef: "head"` を設定する
- `sandbox.bwrapPath` と `sandbox.socatPath` マネージド設定（Linux/WSL）を追加 — カスタムの bubblewrap と socat バイナリの場所を指定できる
- `parentSettingsBehavior` admin-tier キー（`'first-wins' | 'merge'`）を追加 — 管理者が SDK `managedSettings`（parent ティア）をポリシーマージに組み込めるようにする
- Hooks が `effort.level` JSON 入力フィールドと `$CLAUDE_EFFORT` 環境変数を通じてアクティブな effort レベルを受け取るようになり、Bash ツールのコマンドも `$CLAUDE_EFFORT` を読み取れるようになった
- フォーカスモードの動作を改善
- メモリ圧迫時にウォームスペアのバックグラウンドワーカーを解放することでメモリ使用量を改善
- リフレッシュトークンの競合が共有認証情報を消去した後、並行セッションが全て 401 で行き詰まる問題を修正
- ドライブルート（`C:\`）または POSIX `/` にスコープされた `Edit`/`Write` 許可ルールが誤って一致し、常にプロンプトを表示する問題を修正
- クロックスキューや遅いディスクにより履歴またはセッションログファイルのロックが破損した際の未処理 rejection（`ECOMPROMISED`）を修正
- 会話圧縮中に Esc を押すと偽の「Error compacting conversation」通知が表示される問題を修正
- ディスカバリー、ダイナミッククライアント登録、トークン交換、トークンリフレッシュを含む完全な MCP OAuth フローで `HTTP(S)_PROXY` / `NO_PROXY` / mTLS が尊重されない問題を修正
- `--add-dir` / SDK `additionalDirectories` 経由で渡されたマップドネットワークドライブで Read/Write/Edit が拒否される問題を修正
- claude.ai からの Remote Control の停止/割り込みがローカルの Esc と同様に CLI セッションを完全にキャンセルしない問題を修正 — スタックしたツールやプロンプトを割り込んだ後、キューに入ったメッセージが進まなくなっていた
- 1 つのセッションで `/effort` を変更すると他の並行セッションの effort レベルが予期せず変わる問題と、IDE の effort 変更が黙って消える関連問題を修正
- Skill ツール経由でサブエージェントがプロジェクト、ユーザー、またはプラグインのスキルを見つけられない問題を修正
- `claude --help` に `--remote-control-session-name-prefix` と並んで `--remote-control` が表示されるようになった
- [VSCode] 拡張機能ビルドに Claude バイナリがバンドルされていない場合に `claudeCode.claudeProcessWrapper` が「Unsupported platform」で失敗する問題を修正
