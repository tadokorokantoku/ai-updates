- MCP OAuth 認証を改善。step-up 認証サポートとディスカバリーキャッシュにより、サーバー接続時の冗長なネットワークリクエストを削減
- Claude を隔離された git worktree で起動する `--worktree` (`-w`) フラグを追加
- サブエージェントが `isolation: "worktree"` をサポート。一時的な git worktree で作業可能
- バックグラウンドエージェントを停止する Ctrl+F キーバインドを追加（二度押し確認）
- エージェント定義で `background: true` をサポート。常にバックグラウンドタスクとして実行
- プラグインがデフォルト設定として `settings.json` を同梱可能に
- ファイルが見つからないエラーを修正。モデルがリポジトリフォルダを落としたときに修正されたパスを提案
- バックグラウンドエージェントが実行中でメインスレッドがアイドルのときに Ctrl+C と ESC が無言で無視される問題を修正。3 秒以内に 2 回押すとすべてのバックグラウンドエージェントを停止
- キャッシュヒット率を低下させていたプロンプト候補キャッシュのリグレッションを修正
- `--scope` が指定されていないときに常にユーザースコープにデフォルトではなく、`plugin enable` と `plugin disable` が正しいスコープを自動検出するように修正
- シンプルモード（`CLAUDE_CODE_SIMPLE`）が Bash ツールに加えてファイル編集ツールを含むように。シンプルモードで直接ファイル編集が可能に
- 安全性チェックが ask レスポンスをトリガーしたときに権限候補が populate されるように。SDK コンシューマーが権限オプションを表示可能に
- 1M コンテキスト付きの Sonnet 4.5 が Max プランから削除され、1M コンテキストを持つフロンティア Sonnet 4.6 モデルに。/model で切り替えてね
- `/config` 経由でトグルされたときに verbose モードが thinking ブロックの表示を更新しない問題を修正。memo コンパレーターが正しく verbose の変更を検出
- tree-sitter パーサを定期的にリセットすることで、長いセッション中の無限 WASM メモリ増加を修正
- 古い yoga レイアウト参照による潜在的なレンダリング問題を修正
- 起動時の不要な API 呼び出しをスキップしてノンインタラクティブモード（`-p`）のパフォーマンスを改善
- HTTP と SSE MCP サーバーの認証失敗をキャッシュし、認証を必要とするサーバーへの繰り返し接続試行を回避してパフォーマンスを改善
- Yoga WASM リニアメモリが縮小しないことで発生する長時間セッション中の無限メモリ増加を修正
- SDK モデル情報に `supportsEffort`、`supportedEffortLevels`、`supportsAdaptiveThinking` フィールドを追加。コンシューマーがモデル機能を発見可能に
- セッション中に設定ファイルが変更されたときに発火する `ConfigChange` フックイベントを追加。エンタープライズセキュリティ監査と設定変更のオプショナルなブロックが可能に
- MCP 認証失敗をキャッシュすることで冗長な接続試行を回避し、起動パフォーマンスを改善
- 分析トークンカウント用の HTTP 呼び出しを削減して起動パフォーマンスを改善
- MCP ツールのトークンカウントを単一の API 呼び出しにバッチ化して起動パフォーマンスを改善
- `disableAllHooks` 設定が管理対象設定の階層を尊重するように修正。非管理対象設定はポリシーで設定された管理対象フックを無効化できなくなった (#26637)
- `/clear` などのコマンドで始まるセッションに対して `--resume` セッションピッカーが生の XML タグを表示する問題を修正。セッション ID フォールバックに正しく fallthrough するように
- 権限プロンプトを改善。パスの安全性と作業ディレクトリのブロックについて、コンテキストのないむき出しのプロンプトではなく制限の理由を表示
