- `/code-review --fix` がレビュー後にワーキングツリーへ修正を適用するようになった。再利用・簡略化・効率化の提案が反映される。`/simplify` は `/code-review --fix` を呼び出すように変更
- Skills とスラッシュコマンドの frontmatter に `disallowed-tools` を設定することで、スキル実行中に特定のツールをモデルから除外できるようになった
- `/reload-skills` コマンドを追加。セッションを再起動せずにスキルディレクトリを再スキャンできる
- `SessionStart` フックが `reloadSkills: true` を返せるようになった。フックによってインストールされたスキルを同一セッション内で利用可能にする
- `SessionStart` フックが `hookSpecificOutput.sessionTitle` でセッション開始・再開時のタイトルを設定できるようになった
- `MessageDisplay` フック イベントを追加。フックがアシスタントのメッセージテキストを表示時に変換または非表示にできる
- `pluginSuggestionMarketplaces` マネージド設定を追加。管理者がコンテキスト対応のヒントで提案される可能性のある組織マーケットプレイスを許可リストに登録できる
- `claude plugin marketplace remove` が `--scope user|project|local` を受け付けるようになった。`marketplace add`・`install`・`uninstall` と対称性が取れた
- プライマリモデルが見つからない場合、リクエストを毎回失敗させる代わりに、設定された `--fallback-model` へセッション中に切り替えるようになった
- Auto モードのオプトイン同意が不要になった
- Vim モード: NORMAL モードでの `/` が逆方向履歴検索（`Ctrl+R` と同様）を開くようになった。bash/zsh の vi モードに準拠
- `/usage` の内訳に大きなセッションファイルが含まれるようになった。ファイルはストリーミング読み取りでスキャンされるためメモリ使用量は一定に保たれる
- 折りたたまれたグループの thinking サマリーが最低 3 秒間読める状態を保ち、Markdown でレンダリングされ、10 行で上限を設けるようになった（`Ctrl+O` で全 thinking を表示）
- フルスクリーンモードで、モデルが思考中に "Thinking for Ns" インジケーターがリアルタイムでカウントアップし、思考途中で中断してもその値が保持されるようになった
- Workflow ツールのインライン進捗表示を簡略化。ライブのエージェント数はプロンプト下の固定 workflow ステータス行にのみ表示される
- バックグラウンドのエージェントまたはワークフローが実行中の場合、レスポンス後タイマーに "Waiting for N background agents/workflows to finish" を表示し、結果処理後に累計時間を報告するようになった
- セッションのエントリーポイントを OpenTelemetry メトリクス属性（`app.entrypoint`）として追加（`OTEL_METRICS_INCLUDE_ENTRYPOINT=true` でオプトイン）
- スタイルプールのリサイクルにより、非常に長いセッションでターミナルのスタイリングが劣化する問題を修正
- 縮小起動モードでサンドボックス有効警告が表示されない問題を修正。すべてのレイアウトで表示されるようになった
- ツール実行中にローディングスピナーが "still thinking"/"almost done thinking" を表示する問題を修正。各ツール実行後に thinking ステータスが "thinking" にリセットされる
- フォーカスモードで、非表示アクティビティのないターンに "N messages hidden" が誤って表示される問題を修正
- 展開されたツール結果内のリンクをクリックするとセクションが折りたたまれる問題を修正。リンクが正しく開かれるようになった
- Markdown テーブルのセル枠がインラインコードの色を継承する問題、折り返し継続行のスタイルが失われる問題、狭いターミナルのスタック レイアウトで空のヘッダーセルにラベルが表示される問題を修正
- コマンドが同じで環境変数が異なる plugin MCP サーバーが誤って重複排除される問題を修正
- 削除済みのマーケットプレイスやドロップされたプラグインを参照する古い `enabledPlugins` エントリに対して `/doctor` が "marketplace not found" / "plugin not found" を報告する問題を修正
- git ブランチを追跡するプラグインがプラグインレジストリ再構築後にアップデートを受け取らなくなる問題を修正
- Egress プロキシが有効な場合に Claude Code Remote セッションでリモート MCP サーバーへの接続が失敗する問題を修正
- 会話にメッセージがない場合や同じ基底値に解決される effort レベル間の切り替え時に effort 変更確認ダイアログが表示される問題を修正
- `--bare` 使用時またはアタッチメント無効時に Agent ツールの説明が実際には提供されないエージェントリストを参照する問題を修正
- サブエージェントがキャンセルされた後に古い権限プロンプトを承認すると `claude agents` のバックグラウンドワーカーがクラッシュする問題を修正
- API がネストされた `cache_creation` ブレークダウンのみでキャッシュ書き込みを報告する場合、トランスクリプトと結果の使用状況で `cache_creation_input_tokens` が 0 として報告される問題を修正
- Remote Control が有効な SDK ホストセッションで `PushNotification` ツールが "Mobile push not sent (Remote Control inactive)" と誤って報告する問題を修正
- モデルまたはログインの切り替え後に古い thinking-block シグネチャが履歴に残りセッションが停止する問題を修正。積極的に除去し、リトライのセーフティーネットも追加
