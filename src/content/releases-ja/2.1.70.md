- サードパーティゲートウェイで `ANTHROPIC_BASE_URL` を使用する際の API 400 エラーを修正。ツール検索がプロキシエンドポイントを正しく検出し、`tool_reference` ブロックを無効化
- 標準的な Claude 命名パターンに一致しないカスタム Bedrock 推論プロファイルまたは他のモデル識別子を使用する際の `API Error: 400 This model does not support the effort parameter` を修正
- `ToolSearch` 直後の空のモデルレスポンスを修正。サーバーがプロンプトの末尾にシステムプロンプトスタイルのタグでツールスキーマをレンダリングし、モデルが早期に停止することがあった
- 最初のターン後に `instructions` 付き MCP サーバーが接続するときのプロンプトキャッシュバストを修正
- 遅い SSH 接続でタイピング中に Enter が送信ではなく改行を挿入する問題を修正
- Windows/WSL で PowerShell の `Set-Clipboard` を使用して非 ASCII テキスト（CJK、絵文字）のクリップボード破損を修正
- VS Code 統合ターミナルから実行時に起動時に Windows で余分な VS Code ウィンドウが開く問題を修正
- voice モードが "native audio module could not be loaded" で Windows ネイティブバイナリで失敗する問題を修正
- 設定で `voiceEnabled: true` が設定されているときに push-to-talk がセッション開始時にアクティベートされない問題を修正
- `#NNN` 参照を含む markdown リンクがリンク先 URL ではなく現在のリポジトリを誤って指す問題を修正
- プロジェクトの `.claude/settings.json` にレガシーの Opus モデル文字列が pin されていると "Model updated to Opus 4.6" 通知が繰り返される問題を修正
- プラグインが `/plugin` で不正確にインストール済みとして表示される問題を修正
- マーケットプレイスのインストール後に自動更新することで、起動時にプラグインが「マーケットプレイスに見つかりません」エラーを表示する問題を修正
- 古い git バージョンで `/security-review` コマンドが `unknown option merge-base` で失敗する問題を修正
- `/color` コマンドがデフォルトの色にリセットする方法がなかった問題を修正。`/color default`、`/color gray`、`/color reset`、`/color none` でデフォルトを復元
- notes 入力のキー入力ごとに markdown レンダリングを再実行する `AskUserQuestion` プレビューダイアログのパフォーマンスリグレッションを修正
- 早期起動時に読み取られる feature flag がディスクキャッシュを更新せず、古い値がセッション間で持続する問題を修正
- Claude Code Remote 環境で `acceptEdits` または `plan` 以外の `permissions.defaultMode` 設定値が適用される問題を修正。現在は無視される
- すべての `--resume` でスキルリストが再注入される問題を修正（resume ごとに約 600 トークン節約）
- VS Code テレポートセッションで teleport マーカーがレンダリングされない問題を修正
- マイクが無音をキャプチャしたときのエラーメッセージを改善し、「音声が検出されません」と区別
- コンパクションを改善。画像をサマライザリクエストで保持し、プロンプトキャッシュの再利用でより速く安価なコンパクションを実現
- Claude が処理中にも `/rename` が動作するように改善。無言でキューされる代わりに
- ラピッドな push-to-talk の再押下中の一時的な接続失敗を自動リトライするよう voice モードを改善
- Remote Control spawn モード選択プロンプトをより良いコンテキストで改善
- Bedrock、Vertex、Microsoft Foundry 上のデフォルト Opus モデルを Opus 4.6 に変更（以前は Opus 4.1）
- `/output-style` コマンドを非推奨化 — 代わりに `/config` を使用。プロンプトキャッシングを改善するため、出力スタイルはセッション開始時に固定
- VSCode: Claude 4.5 モデルで Bedrock/Vertex やプロキシ経由のユーザーの HTTP 400 エラーを修正
