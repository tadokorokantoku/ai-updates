highlights:
  - "セッション復帰時のコンテキスト提供のための `/recap` スラッシュコマンドを追加"
  - "`/undo` を `/rewind` のエイリアスとして追加"
---

- API キー、Bedrock、Vertex、Foundry で 1 時間プロンプトキャッシュ TTL にオプトインする `ENABLE_PROMPT_CACHING_1H` 環境変数を追加(`ENABLE_PROMPT_CACHING_1H_BEDROCK` は非推奨だが引き続き有効)、および 5 分 TTL を強制する `FORCE_PROMPT_CACHING_5M` を追加
- セッション復帰時にコンテキストを提供する recap 機能を追加。`/config` で設定可能、`/recap` で手動呼び出し可能。テレメトリが無効の場合は `CLAUDE_CODE_ENABLE_AWAY_SUMMARY` で強制。
- モデルが Skill tool 経由で `/init`、`/review`、`/security-review` などの組み込みスラッシュコマンドを発見・実行可能に
- `/undo` を `/rewind` のエイリアスに
- 会話途中でのモデル切り替え時に警告を表示するよう `/model` を改善。次の応答で履歴全体が未キャッシュで再読み込みされるため
- `/resume` ピッカーを現在のディレクトリのセッションをデフォルト表示するよう改善。`Ctrl+A` で全プロジェクト表示
- エラーメッセージを改善: サーバーのレートリミットとプラン利用制限を区別、5xx/529 エラーには status.claude.com へのリンクを表示、未知のスラッシュコマンドに最も近い候補を提案
- 言語文法をオンデマンドでロードすることでファイル読み込み、編集、シンタックスハイライトのメモリフットプリントを削減
- 詳細トランスクリプト表示時 (`Ctrl+O`) に "verbose" インジケータを追加
- `DISABLE_PROMPT_CACHING*` 環境変数でプロンプトキャッシュが無効化されている場合、起動時に警告を表示
- `/login` コードプロンプトで貼り付けが効かない問題を修正 (2.1.105 のリグレッション)
- `DISABLE_TELEMETRY` を設定したサブスクライバーが 1 時間ではなく 5 分のプロンプトキャッシュ TTL にフォールバックする問題を修正
- safety classifier のトランスクリプトがコンテキストウィンドウを超えた場合に、auto モードでも Agent tool が権限を求めてくる問題を修正
- `CLAUDE_ENV_FILE` (例: `~/.zprofile`) が `#` コメント行で終わる場合に Bash tool が出力を返さない問題を修正
- `claude --resume <session-id>` で `/rename` で設定したセッションのカスタム名と色が失われる問題を修正
- 最初のメッセージが短い挨拶の場合にセッションタイトルがプレースホルダーのサンプルテキストを表示する問題を修正
- `--teleport` 後にプロンプト入力にターミナルエスケープコードがゴミ文字として表示される問題を修正
- `/feedback` リトライ: 失敗後に Enter で再送信する際、説明を先に編集しなくても動作するよう修正
- `--teleport` および `--resume <id>` の前提条件エラー (dirty git tree、セッション未検出など) がエラーメッセージを表示せず無音で終了する問題を修正
- Web UI で設定した Remote Control セッションタイトルが 3 メッセージ目以降に自動生成タイトルで上書きされる問題を修正
- トランスクリプトに自己参照メッセージが含まれる場合に `--resume` がセッションを切り詰める問題を修正
- トランスクリプト書き込み失敗 (ディスク満杯など) がログに記録されず無音でドロップされる問題を修正
- `language` 設定時にレスポンスから発音記号 (アクセント、ウムラウト、セディーユ) が欠落する問題を修正
- 最初にインストールされたプロジェクトと異なるプロジェクトから実行した場合、ポリシー管理のプラグインが自動更新されない問題を修正
