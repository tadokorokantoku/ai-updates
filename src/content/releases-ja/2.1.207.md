- Bedrock、Vertex AI、Foundry で、`CLAUDE_CODE_ENABLE_AUTO_MODE` のオプトインなしに auto mode が利用可能になった。設定の `disableAutoMode` で無効化可能
- 非常に長いリスト、テーブル、段落、コードブロックを含む応答のストリーミング中にターミナルがフリーズし、キー入力が遅延する問題を修正
- 非対話実行（`claude -p`、SDK）からのリモート管理設定が、セキュリティ同意ダイアログを一度も表示しないまま永続的に同意済みとして記録されていた問題を修正
- 無害なシステム生成の会話更新によって誤ったプロンプトインジェクション警告がトリガーされていた問題を修正
- 自動アップデーターが、リリースのたびに `~/.local/bin/claude` にあるカスタムのランチャースクリプトやシンボリックリンクを上書きしていた問題を修正。`/doctor` が外部管理のランチャーを報告するようになった
- `cd` を含む複合コマンドが、出力リダイレクト先が `/dev/null` のみの場合でも権限を求めていた問題を修正
- 応答のストリーミングが完了した際に、トランスクリプトが回答の先頭より上にジャンプしていた問題を修正
- `worktree.sparsePaths` で作成した最後の worktree を削除した後、`extensions.worktreeConfig` がリポジトリの `.git/config` に残ってしまい、`tea` のような go-git ツールを壊していた問題を修正
- rules の glob、skill のパス、`.ignore`、`.worktreeinclude` 内の不正な角括弧パターンが、ファイル読み込み・ファイル候補表示・worktree 作成を壊していた問題を修正
- agent teams で、不正な形式のチームメイトのメールボックスメッセージが原因で、メールボックスファイルを手動削除するまで毎秒エラーが繰り返されるクラッシュループを修正
- プランを承認して自動的に命名されたバックグラウンドセッションが、エージェントビューの行にその名前を表示していなかった問題を修正
- git worktree に入ったバックグラウンドセッションが、コールドリオープン後に空の状態で再開してしまう問題を修正
- Remote Control のタスクステータス更新が、ネットワーク切断や認証情報のリフレッシュから接続が復旧した際に失われていた問題を修正
- デスクトップアプリがホストする Remote Control セッションが、モバイルや web でバックグラウンドエージェントやワークフローの進捗を表示していなかった問題を修正
- Deep research の実行で、Fetch フェーズの全エージェントが「unknown」とラベル付けされていた問題を修正。チップにソースのホスト名が表示されるようになった
- Bedrock が API リクエストのたびに IAM Identity Center へ新しい AWS SSO 認証情報を繰り返し要求していた問題を修正
- エージェントビューの改善: 同じテキストを再度貼り付けると、2つ目のプレースホルダーを追加するのではなく、折りたたまれた `[Pasted text #N]` プレースホルダーが展開されるようになった
- エージェントビューの改善: ブロックされたセッションのプレビューが、同じタイムスタンプを二重表示する代わりに質問を先頭に表示し、経過時間を言葉で示すクロック（`waiting 3m`）を表示するようになった
- Bedrock、Vertex、AWS 上の Claude Platform のデフォルトを Claude Opus 4.8 に変更
- auto mode が `.claude/settings.local.json`（リポジトリ常駐）から `autoMode` を読み込まなくなった。代わりに `~/.claude/settings.json` を使用すること
- Windows で AWS 認証情報の解決がスタックした場合（スタックした `credential_process` など）に無期限にハングしていた問題を修正。60秒のストールガードが無期限待機の代わりに発動するようになった
- プラグインの hooks / monitors / MCP headersHelper: シェル形式のコマンド内の `${user_config.*}` が拒否されるようになった（シェルインジェクション対策）。hooks では exec 形式（`args` 配列）または `$CLAUDE_PLUGIN_OPTION_<KEY>` を使用し、monitors と headersHelper ではスクリプト内（設定ファイルまたはサーバーの `env` ブロック）で値を読み込むこと
- プラグインのオプション値（`pluginConfigs`）がプロジェクトレベルの `.claude/settings.json` から読み込まれなくなった。user、`--settings`、managed settings のみが有効
- `/usage-credits` の金額入力が、不正な値（貼り付けたタイムスタンプなど）を黙って数字にトリミングしていた問題を修正。不正な金額はエラーで拒否されるようになり、$1,000 を超える金額には入力による確認が必要に
