- `CLAUDE_CODE_FORK_SUBAGENT=1` を設定することで、外部ビルドでもフォークされたサブエージェントを有効化できるようになった
- `--agent` で起動するメインスレッドのエージェントセッションで、エージェント frontmatter の `mcpServers` が読み込まれるようになった
- `/model` の改善: プロジェクトが別のモデルをピンしている場合でも、選択したモデルが再起動後も保持されるようになった。また、起動ヘッダーにアクティブなモデルがプロジェクトまたはマネージド設定によるピンであることが表示されるようになった
- `/resume` コマンドが、古くなった大きなセッションを再読み込みする前に要約を提案するようになった（既存の `--resume` 動作と同様）
- ローカルと claude.ai 両方の MCP サーバーが設定されている場合の起動を高速化（並列接続がデフォルトに）
- インストール済みのプラグインに対して `plugin install` を実行すると、"already installed" で停止する代わりに不足している依存関係をインストールするようになった
- プラグインの依存関係エラーが「not installed」とインストールヒント付きで表示されるようになった。また `claude plugin marketplace add` が設定済みのマーケットプレイスから不足している依存関係を自動解決するようになった
- マネージド設定の `blockedMarketplaces` と `strictKnownMarketplaces` が、プラグインのインストール・更新・更新・自動更新時に強制適用されるようになった
- Advisor Tool（実験的）: ダイアログに「experimental」ラベル、詳細リンク、有効時の起動通知が追加された。また、全プロンプトや `/compact` で「Advisor tool result content could not be processed」エラーでセッションが詰まる問題が修正された
- `cleanupPeriodDays` の保持期間スイープが `~/.claude/tasks/`、`~/.claude/shell-snapshots/`、`~/.claude/backups/` も対象とするようになった
- OpenTelemetry: `user_prompt` イベントにスラッシュコマンドの `command_name` と `command_source` が含まれるようになった。また `cost.usage`、`token.usage`、`api_request`、`api_error` にモデルがエフォートレベルをサポートしている場合に `effort` 属性が含まれるようになった。カスタム/MCP コマンド名は `OTEL_LOG_TOOL_DETAILS=1` を設定しない限り削除される
- macOS と Linux のネイティブビルド: `Glob` および `Grep` ツールが、Bash ツール経由で利用できる組み込みの `bfs` および `ugrep` に置き換えられた — 別のツールラウンドトリップなしで高速検索が可能（Windows と npm インストールビルドは変更なし）
- Windows: プロセスごとに `where.exe` の実行可能ファイルルックアップをキャッシュし、サブプロセス起動を高速化
- Pro/Max サブスクライバーの Opus 4.6 と Sonnet 4.6 のデフォルトエフォートが `high` になった（以前は `medium`）
- Plain-CLI OAuth セッションでアクセストークンが期限切れになった際に「Please run /login」で終了してしまう問題を修正 — 401 に対してトークンがリアクティブに更新されるようになった
- 非常に大きな HTML ページで `WebFetch` がハングする問題を修正 — HTML から Markdown への変換前に入力を切り詰めるようになった
- プロキシが HTTP 204 No Content を返した際にクラッシュする問題を修正 — `TypeError` の代わりに明確なエラーが表示されるようになった
- `CLAUDE_CODE_OAUTH_TOKEN` 環境変数でトークンを指定して起動し、そのトークンが期限切れになった場合に `/login` が効かない問題を修正
- 入力直後のプロンプト入力アンドゥ（`Ctrl+_`）が何もしなかったり、各アンドゥステップで状態をスキップしたりする問題を修正
- Bun で実行時にリモート API リクエストで `NO_PROXY` が考慮されない問題を修正
- 低速接続でキー名が結合テキストとして届く際にエスケープ/リターンが誤ってトリガーされる稀な問題を修正
- SDK の `reload_plugins` が全ユーザー MCP サーバーをシリアルで再接続する問題を修正
- Bedrock のアプリケーション推論プロファイルリクエストが、thinking を無効にした Opus 4.7 をバックエンドとする場合に 400 で失敗する問題を修正
- MCP の `elicitation/create` リクエストが、サーバーがターン途中で接続完了した際に print/SDK モードで自動キャンセルされる問題を修正
- メインエージェントと異なるモデルで実行しているサブエージェントが、ファイル読み込みに対してマルウェア警告を誤って表示する問題を修正
- バックグラウンドタスクが存在する場合のアイドル時の再レンダリングループを修正 — Linux でのメモリ増加を軽減
- [VSCode] 複数の大きなマーケットプレイスが設定されている際に「Manage Plugins」パネルが壊れる問題を修正
- Opus 4.7 セッションで `/context` のパーセンテージが膨らみ、自動コンパクションが早まりすぎる問題を修正 — Claude Code が Opus 4.7 のネイティブ 1M コンテキストウィンドウの代わりに 200K で計算していた
