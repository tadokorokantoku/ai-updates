- プロンプト入力欄に絵文字ショートコードの自動補完を追加: `:heart:` と入力すると ❤️ が挿入され、`:hea` のように途中まででも候補が表示される — `emojiCompletionEnabled` 設定で無効化可能
- transcript の書き込みが失敗している場合（ディスクフルなど）や、継承された環境変数によりセッション保存がオフになっている場合に、黙って transcript を失う代わりに警告を表示するように
- 切り詰められた MCP tool の出力について、切り詰め前の完全な結果がセッションの残り期間中メモリに保持され続けていたメモリリークを修正
- Windows での自動更新失敗により `claude.exe` が消失することがあった不具合を修正。更新失敗時は保持されていた実行ファイルが自動的に復元されるように
- バックグラウンドセッションの隔離がシンボリックリンクされた作業ディレクトリを正規化しておらず、セッションが自身のワークスペースフォルダから抜け出せてしまう可能性があった不具合を修正
- Bedrock 上の Claude Opus 4.8 で auto-compact が一切発火しない不具合、および上限超過後に `/compact` が失敗する不具合を修正
- Claude Desktop セッションで企業の mTLS、TLS-verify、OAuth スコープ、proxy 設定が無視されていた不具合を修正
- スクリーンリーダーモードの起動時アナウンスが最初のプロンプト描画で途切れる不具合、および thinking のステータス行が経過時間とトークン数を更新するために数秒ごとに再描画されていた不具合を修正
- `OTEL_EXPORTER_OTLP_ENDPOINT` を設定する managed settings が全シグナルを制御していなかった不具合を修正 — シグナル別のより低スコープな上書き設定が、telemetry を managed endpoint 以外へリダイレクトしなくなった
- transcript に不正な attachment エントリがある場合に `--resume`/`--continue` と `/resume` が TypeError で失敗していた不具合を修正
- Remote Control セッションで、プロンプトが表示された後に接続した viewer に対して保留中の permission プロンプトやダイアログが表示されていなかった不具合を修正
- セッションをバックグラウンドに送った後（`/background` または `←`）や、負荷の高いマシンでセッションが終了した際に、バックグラウンドシェルが停止不能になることがあった不具合を修正（Windows で特に顕著）
- 多数の brace グループを含む `CLAUDE.md` や `SKILL.md` の paths frontmatter 値が、CLI 起動時に OOM で強制終了したりスタールしたりしていた不具合を修正 — brace 展開に処理量の上限を設定
- 起動中のバックグラウンドセッションにアタッチした際、transcript プレビューが入力エリアに密着して表示される不具合を修正。ライブレイアウトと同じ1行分の余白を残すようになり、セッションが引き継がれても transcript がずれなくなった
- footer の PR バッジリンクを、ターミナルのサポートが検出できない場合（ssh/tmux 経由など）でもクリック可能なハイパーリンクにするよう改善。`FORCE_HYPERLINK=0` でオプトアウト可能
- ログイン期限切れ警告の表示タイミングを、期限の5日前から3日前に変更
- frontend-design plugin のおすすめ表示を、繰り返し表示し続ける代わりに生涯3回までに制限
- 同時実行サブエージェント数の上限を追加（デフォルト20、`CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS` で上書き可能）。1つのメッセージから無制限にバックグラウンドエージェントがファンアウトするのを防止
- サブエージェントがデフォルトでネストしたサブエージェントを spawn しないように変更。`CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH` でより深いネストを許可可能
- `--max-budget-usd` がバックグラウンドサブエージェントを停止させていなかった不具合を修正: 上限に達すると新規 spawn が拒否され、実行中のバックグラウンドエージェントも停止するように
