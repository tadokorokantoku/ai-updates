- `/config` の設定（テーマ、エディターモード、verbose など）が `~/.claude/settings.json` に永続化され、プロジェクト/ローカル/ポリシーのオーバーライド優先順位に参加するようになった
- フッターの PR バッジを github.com の代わりにカスタムコードレビュー URL に向ける `prUrlTemplate` 設定を追加
- 起動ロゴの作業ディレクトリを非表示にする `CLAUDE_CODE_HIDE_CWD` 環境変数を追加
- `--from-pr` が GitLab のマージリクエスト、Bitbucket のプルリクエスト、GitHub Enterprise の PR URL を受け付けるようになった
- `--print` モードがエージェントの `tools:` と `disallowedTools:` frontmatter を考慮するようになり、インタラクティブモードの動作と一致
- `--agent <name>` が組み込みエージェントの `permissionMode` をエージェント定義から考慮するようになった
- PowerShell のツールコマンドがパーミッションモードで自動承認できるようになり、Bash の動作と一致
- Hooks: `PostToolUse` と `PostToolUseFailure` のフック入力に `duration_ms`（ツール実行時間。パーミッションプロンプトと PreToolUse フックを除く）が含まれるようになった
- サブエージェントと SDK の MCP サーバー再設定が直列ではなく並列でサーバーに接続するようになった
- 別のプラグインのバージョン制約でピン留めされたプラグインが、満たす最上位の git タグに自動更新されるようになった
- Vim モード: INSERT での Esc がキューに入ったメッセージを入力欄に戻さなくなった。再度 Esc を押すことで中断できる
- スラッシュコマンドの候補がクエリにマッチした文字をハイライト表示するようになった
- スラッシュコマンドピッカーが長い説明を切り捨てる代わりに 2 行目に折り返すようになった
- 出力内の `owner/repo#N` 形式のショートリンクが常に github.com を指す代わりに git リモートのホストを使用するようになった
- Security: `blockedMarketplaces` が `hostPattern` と `pathPattern` エントリを正しく適用するようになった
- OpenTelemetry: `tool_result` と `tool_decision` イベントに `tool_use_id` が含まれるようになった。`tool_result` には `tool_input_size_bytes` も含まれる
- Status line: stdin JSON に `effort.level` と `thinking.enabled` が含まれるようになった
- CRLF コンテンツ（Windows クリップボード、Xcode コンソール）を貼り付けた際に各行間に余分な空行が挿入される問題を修正
- kitty キーボードプロトコルシーケンスをブラケット貼り付けの内部で使用するターミナルで複数行の貼り付けが改行を失う問題を修正
- Bash ツールがパーミッションで拒否された場合に macOS/Linux ネイティブビルドで Glob と Grep ツールが消える問題を修正
- フルスクリーンモードでスクロールアップするとツールが完了するたびに一番下にスナップバックする問題を修正
- サーバーが OAuth ディスカバリーリクエストに対して非 JSON ボディを返した場合に MCP HTTP 接続が「Invalid OAuth error response」で失敗する問題を修正
- Rewind オーバーレイが画像添付付きのメッセージに「(no prompt)」を表示する問題を修正
- Auto mode がプランモードを「Execute immediately」の競合する指示で上書きする問題を修正
- 応答ペイロードを送信しない非同期 `PostToolUse` フックがセッションのトランスクリプトに空のエントリを書き込む問題を修正
- サブエージェントのタスク通知がキューで孤立した場合にスピナーが表示され続ける問題を修正
- Vertex AI でサポートされていないベータヘッダーエラーを回避するためにツール検索がデフォルトで無効化された（`ENABLE_TOOL_SEARCH` でオプトイン可能）
- スラッシュコマンド内で絶対パスを使用した場合に `@` ファイルの Tab 補完がプロンプト全体を置き換える問題を修正
- Docker または SSH 経由の macOS Terminal.app で起動時にプロンプトに余分な `p` 文字が表示される問題を修正
- HTTP/SSE/WebSocket MCP サーバーの `headers` 内の `${ENV_VAR}` プレースホルダーがリクエスト前に置換されない問題を修正
- `client_secret_post` を要求するサーバーに対して `--client-secret` で保存された MCP OAuth クライアントシークレットがトークン交換時に送信されない問題を修正
- `/skills` の Enter キーがダイアログを閉じる代わりに `/<skill-name>` をプロンプトに事前入力する問題を修正
- `/agents` の詳細ビューがサブエージェントで利用できない組み込みツールを「Unrecognized」と誤ってラベル付けする問題を修正
- プラグインキャッシュが不完全な場合に Windows でプラグインからの MCP サーバーが起動しない問題を修正
- `/export` が会話で実際に使用されたモデルではなく現在のデフォルトモデルを表示する問題を修正
- verbose 出力設定が再起動後に維持されない問題を修正
- `/usage` のプログレスバーが「Resets …」ラベルと重なる問題を修正
- `${user_config.*}` が空白のままにされたオプションフィールドを参照する場合にプラグイン MCP サーバーが失敗する問題を修正
- 文末の数字を含むリストアイテムが数字だけを次の行に折り返す問題を修正
- `/plan` と `/plan open` がプランモードに入る際に既存のプランに対して動作しない問題を修正
- 自動コンパクション前に呼び出されたスキルが次のユーザーメッセージに対して再実行される問題を修正
- `/reload-plugins` と `/doctor` が無効なプラグインのロードエラーを報告する問題を修正
- `isolation: "worktree"` を持つ Agent ツールが以前のセッションの古い worktree を再利用する問題を修正
- 無効化された MCP サーバーが `/status` で「failed」として表示される問題を修正
- `TaskList` がタスクを ID 順ではなく任意のファイルシステム順で返す問題を修正
- `gh` の出力が「rate limit」に言及する PR タイトルを含む場合に「GitHub API rate limit exceeded」のヒントが誤表示される問題を修正
- SDK/bridge の `read_file` が増大するファイルのサイズ上限を正しく適用しない問題を修正
- git worktree で作業中に PR がセッションにリンクされない問題を修正
- `/doctor` が上位スコープで上書きされた MCP サーバーエントリについて警告する問題を修正
- Windows: 誤検知の「Windows requires 'cmd /c' wrapper」MCP 設定警告を削除
- [VSCode] マイクのパーミッションプロンプトが表示されている間は macOS で音声入力の最初の録音が何も生成されない問題を修正
