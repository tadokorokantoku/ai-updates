- `managed-settings.json` と並行して `managed-settings.d/` ドロップインディレクトリを追加。別々のチームがアルファベット順にマージされる独立したポリシーフラグメントをデプロイ可能に
- リアクティブ環境管理（例: direnv）向けの `CwdChanged` と `FileChanged` フックイベントを追加
- サンドボックスが有効だが起動できない場合、サンドボックスなしで実行するのではなくエラーで終了する `sandbox.failIfUnavailable` 設定を追加
- `claude-cli://` プロトコルハンドラ登録を防ぐ `disableDeepLinkRegistration` 設定を追加
- サブプロセス環境（Bash ツール、フック、MCP stdio サーバー）から Anthropic とクラウドプロバイダーの認証情報を削除する `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1` を追加
- トランスクリプト検索を追加 — トランスクリプトモード（`Ctrl+O`）で `/` を押して検索、`n`/`N` でマッチをステップ
- 外部エディタを開くエイリアスとして `Ctrl+X Ctrl+E` を追加（readline ネイティブバインディング。`Ctrl+G` は引き続き動作）
- 貼り付けられた画像がカーソル位置に `[Image #N]` チップを挿入するように。プロンプトで位置指定で参照可能
- エージェントが最初のターンを自動送信する `initialPrompt` を frontmatter で宣言できるように
- `chat:killAgents` と `chat:fastMode` が `~/.claude/keybindings.json` で再バインド可能に
- マウス追跡エスケープシーケンスが終了後にシェルプロンプトに漏れる問題を修正
- macOS で終了時に Claude Code がハングする問題を修正
- 数秒アイドルした後に画面が真っ白になる問題を修正
- 共通行が少ない非常に大きなファイルを diff したときのハングを修正 — diff が 5 秒後にタイムアウトしてグレースフルにフォールバック
- voice 入力が有効なときにネイティブオーディオモジュールを eager にロードすることで起動時に 1〜8 秒の UI フリーズを修正
- claude.ai MCP config fetch に約 3 秒待ってから進む起動リグレッションを修正
- `--mcp-config` CLI フラグが `allowedMcpServers`/`deniedMcpServers` 管理対象ポリシー施行をバイパスする問題を修正
- claude.ai MCP コネクタ（Slack、Gmail など）が単一ターン `--print` モードで利用できない問題を修正
- Claude Code 終了時に `caffeinate` プロセスが適切に終了しない問題を修正。Mac がスリープしない問題を防止
- `!` プレフィックス付きコマンド候補を Tab で受け入れる際に bash モードがアクティベートしない問題を修正
- 候補をナビゲートした後に古いスラッシュコマンド選択が間違ったハイライトコマンドを表示する問題を修正
- `/config` メニューが検索カーソルとリスト選択を同時に表示する問題を修正
- コンテキストコンパクション後にバックグラウンドサブエージェントが見えなくなり、重複したエージェントが spawn される可能性がある問題を修正
- クリーンアップ中に git または API 呼び出しがハングすると、バックグラウンドエージェントタスクが "running" 状態でスタックする問題を修正
- アップグレード後の初回起動時に `--channels` が "Channels are not currently available" を表示する問題を修正
- アンインストールされたプラグインフックが次のセッションまで発火し続ける問題を修正
- ストリーミング応答中にキューされたコマンドがちらつく問題を修正
- メッセージ処理中に送信されたスラッシュコマンドがモデルにテキストとして送信される問題を修正
- オフスクリーンスクロール後に折りたたまれた read/search グループが完了すると scrollback がジャンプする問題を修正
- モデルが thinking を開始または停止すると scrollback が上部にジャンプする問題を修正
- フックの progress/attachment メッセージが parentUuid チェーンをフォークすることによる resume 時の SDK セッション履歴損失を修正
- ターミナルウィンドウの外でマウスを放したときに copy-on-select が発火しない問題を修正
- 高さ制約のあるリストでアイテムがオーバーフローするとゴースト文字が表示される問題を修正
- アイドル状態のプロンプトで `Ctrl+B` が readline backward-char に干渉する問題を修正 — フォアグラウンドタスクがバックグラウンド化できるときのみ発火
- ツール結果ファイルがクリーンアップされず `cleanupPeriodDays` 設定を無視する問題を修正
- voice の押しっぱなし解放後に space キーが最大 3 秒間飲み込まれる問題を修正
- オーディオハードウェアのない Linux（Docker、ヘッドレス、WSL1）で voice モード使用時に ALSA ライブラリエラーがターミナル UI を破損する問題を修正
- `which` の spawn がカーネル制限されている Termux/Android での voice モード SoX 検出を修正
- アクティブに実行中にもかかわらず Remote Control セッションが web セッションリストで Idle として表示される問題を修正
- config 駆動モードでフッターナビゲーションが不可視の Remote Control pill を選択する問題を修正
- リモートセッションでツール使用 ID が無期限に蓄積するメモリリークを修正
- プロファイル fetch と他のブートワークをオーバーラップして Bedrock SDK コールドスタートレイテンシーを改善
- 大きなセッションでの `--resume` メモリ使用量と起動レイテンシーを改善
- プラグイン起動を改善 — コマンド、スキル、エージェントが再フェッチなしにディスクキャッシュからロード
- Remote Control セッションタイトルを改善: AI 生成タイトルが最初のメッセージの数秒以内に表示
- サイト運営者が Claude Code トラフィックを `robots.txt` 経由で認識して allowlist できるよう、`WebFetch` が `Claude-User` として識別するよう改善
- 大きなページに対する `WebFetch` のピークメモリ使用量を削減
- 長いセッションでの scrollback リセットをターンごとに 1 回から約 50 メッセージごとに 1 回に削減
- 認証されていない HTTP/SSE MCP サーバーで `claude -p` 起動を高速化（約 600ms 節約）
- Bash のゴーストテキスト候補が送信直後のコマンドを即座に含むように
- フォールバックリクエストが切り詰められる可能性を減らすため、非ストリーミングフォールバックのトークン上限（21k → 64k）とタイムアウト（120s → 300s ローカル）を引き上げ
- 応答前にプロンプトを中断した場合、入力が自動的に復元されて編集・再送信可能に
- Claude 応答中にも `/status` が動作するように、ターン終了までキューされない
- org 管理対象コネクタを複製するプラグイン MCP サーバーが第 2 接続を実行する代わりに抑制されるように
- Linux: `claude-cli://` プロトコルハンドラを登録するときに `XDG_DATA_HOME` を尊重
- "stop all background agents" キーバインドを readline の forward-char をシャドウするのを避けるため `Ctrl+F` から `Ctrl+X Ctrl+K` に変更
- バックグラウンドタスクの出力ファイルパスで `Read` を使うことを優先して `TaskOutput` ツールを非推奨化
- ストリーミングが失敗したときに非ストリーミングフォールバックを無効化する `CLAUDE_CODE_DISABLE_NONSTREAMING_FALLBACK` 環境変数を追加
- プラグインオプション（`manifest.userConfig`）を外部化 — プラグインが enable 時に設定を要求可能、`sensitive: true` 値は keychain（macOS）または保護された認証情報ファイル（その他のプラットフォーム）に保存
- Claude がクリップボードに貼り付けた画像のオンディスクパスをファイル操作のために参照可能に
- `Ctrl+L` が画面をクリアして完全な再描画を強制するように — Cmd+K が UI を部分的に空白のまま残した場合の回復に使用。プロンプト入力をクリアするには `Ctrl+U` またはダブル Esc を使用
- `--bare -p`（SDK パターン）が API リクエストまで約 14% 高速化
- Memory: `MEMORY.md` インデックスが 200 行と同様に 25KB でも切り詰められるように
- `--channels` がアクティブな場合 `AskUserQuestion` とプランモードツールを無効化
- 失敗しているツール呼び出し中に貼り付けられた画像がキューされたときの API 400 エラーを修正
- SSE 接続が呼び出しの途中で切断し再接続試行を使い果たすと MCP ツール呼び出しが無期限にハングする問題を修正
- 最初のユーザーメッセージの前にバックグラウンドエージェントが完了すると Remote Control セッションタイトルが生の XML を表示する問題を修正
- 再開されたトランスクリプトチェーンの progress-message ギャップが原因で、コンテナ再起動後にリモートセッションが会話履歴を忘れる問題を修正
- 自動的にリトライする代わりに、リモートセッションが一時的な認証エラーで再ログインを要求する問題を修正
- `rg ... | wc -l` や類似のパイプされたコマンドが Linux のサンドボックスモードでハングして `0` を返す問題を修正
- CJK IME が全角スペースを挿入すると voice 入力の押しっぱなしがアクティベートしない問題を修正
- worktree 名にスラッシュが含まれると `--worktree` が無言でハングする問題を修正
- [VSCode] バックエンドが 60 秒応答していないとスピナーが赤くなり "Not responding" を表示
- [VSCode] URL または再起動後にセッションを再度開く際、セッション履歴が正しくロードされない問題を修正
- [VSCode] キーボードナビゲート可能な rewind ピッカーを開く Esc 二連打（または `/rewind`）を追加
- [VSCode] セッションキャッシュが古くなった後、「Fork conversation from here」と rewind アクションが無言で失敗する問題を修正
