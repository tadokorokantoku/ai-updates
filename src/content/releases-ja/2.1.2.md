- ターミナルにドラッグされた画像にソースパスメタデータを追加。Claude が画像の出所を理解できるように
- OSC 8 をサポートするターミナル (iTerm など) でツール出力内のファイルパスをクリック可能なハイパーリンクに
- Windows Package Manager (winget) インストールを自動検出し更新手順を案内するサポートを追加
- プランモードで "auto-accept edits" オプションを素早く選択する Shift+Tab キーボードショートカットを追加
- メインの自動アップデータが無効でもプラグインの自動アップデートを許可する `FORCE_AUTOUPDATE_PLUGINS` 環境変数を追加
- `--agent` が指定されている場合に SessionStart フック入力に `agent_type` を追加
- 不正な入力で任意コマンドが実行される可能性があった bash コマンド処理のコマンドインジェクション脆弱性を修正
- tree-sitter パースツリーが解放されず、長時間セッションで WASM メモリが無制限に増加するメモリリークを修正
- CLAUDE.md ファイルで `@include` ディレクティブを使用した際にバイナリファイル (画像、PDF など) が誤ってメモリに含まれる問題を修正
- 別のインストールが進行中と誤って主張するアップデートを修正
- 監視ディレクトリにソケットファイルが存在する場合のクラッシュを修正 (EOPNOTSUPP エラーへの多層防御)
- `/tasks` コマンド使用時にリモートセッション URL と teleport が壊れる問題を修正
- ユーザー固有のサーバー構成をサニタイズすることで、分析イベントに MCP ツール名が露出する問題を修正
- iTerm2、Kitty、WezTerm などのネイティブ CSIu ターミナル向けに macOS の Option-as-Meta ヒントでターミナル固有の手順を表示するよう改善
- SSH 経由での画像貼り付け時のエラーメッセージを改善。役立たないクリップボードショートカットヒントの代わりに `scp` を使用するよう提案
- 日常的な開発ワークフロー (git fetch/rebase、npm install、テスト、PR) を中リスクとしてフラグしないよう権限説明を改善
- 大きな bash コマンド出力を切り詰めず、ディスクに保存するように変更。Claude が完全な内容を読めるように
- 大きなツール出力を切り詰めず、ディスクに永続化するように変更。ファイル参照経由で完全な出力にアクセス可能に
- `/plugins` installed タブをスコープベースのグルーピングでプラグインと MCP を統合するよう変更
- Windows 管理設定パス `C:\ProgramData\ClaudeCode\managed-settings.json` を非推奨化 - 管理者は `C:\Program Files\ClaudeCode\managed-settings.json` に移行を推奨
- [SDK] zod peer dependency の最小バージョンを ^4.0.0 に変更
- [VSCode] 手動コンパクト後に使用量表示が更新されない問題を修正
