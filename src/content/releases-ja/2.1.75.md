- Max、Team、Enterprise プラン向けにデフォルトで Opus 4.6 の 1M コンテキストウィンドウを追加（以前は extra usage が必要）
- すべてのユーザー向けにセッションのプロンプトバー色を設定する `/color` コマンドを追加
- `/rename` 使用時のプロンプトバーにセッション名の表示を追加
- メモリファイルに最終更新タイムスタンプを追加。Claude がどのメモリが新しいか古いかを判断しやすく
- フックが確認を必要とするときの権限プロンプトにフックソース表示（settings/plugin/skill）を追加
- 新規インストールで `/voice` を 2 回切り替えずに voice モードが正しくアクティベートされない問題を修正
- `/model` や Option+P でモデルを切り替えた後、Claude Code ヘッダーが表示されたモデル名を更新しない問題を修正
- アタッチメントメッセージの計算が undefined 値を返したときのセッションクラッシュを修正
- パイプされたコマンドで Bash ツールが `!` を破損する問題を修正（例: `jq 'select(.x != .y)'` が正しく動作するように）
- 管理対象で無効化されたプラグインが `/plugin` Installed タブに表示される問題を修正。組織によって強制無効化されたプラグインは非表示に
- thinking と `tool_use` ブロックのトークン推定の過剰カウントを修正。早期のコンテキストコンパクションを防止
- 破損したマーケットプレイス設定パス処理を修正
- フォークまたは継続されたセッションを再開した後、`/resume` がセッション名を失う問題を修正
- Config タブを訪問した後、Esc が `/status` ダイアログを閉じない問題を修正
- プランを受け入れるまたは拒否する際の入力処理を修正
- エージェントチームのフッターヒントが "↓ to expand" ではなく正しい "shift + ↓ to expand" を表示するよう修正
- 不要なサブプロセス spawn をスキップして macOS の非 MDM マシンでの起動パフォーマンスを改善
- async フック完了メッセージをデフォルトで抑制（`--verbose` またはトランスクリプトモードで可視）
- 破壊的変更: 非推奨の Windows 管理対象設定フォールバック `C:\ProgramData\ClaudeCode\managed-settings.json` を削除 — `C:\Program Files\ClaudeCode\managed-settings.json` を使用
