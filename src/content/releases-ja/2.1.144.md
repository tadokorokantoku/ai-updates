## 変更内容

- バックグラウンドセッションへの `/resume` サポートを追加 — `claude --bg` またはエージェントビューで開始したセッションがインタラクティブなセッションと並んで `bg` マーク付きで表示されるように
- バックグラウンドサブエージェントの完了通知に経過時間を追加（例：「Agent completed · 3h 2m 5s」）
- `/plugin` のブラウズ・ディスカバーペインにプラグインの最終更新日時を表示
- `/model` は現在のセッションのみモデルを変更するように変更；モデルピッカーで `d` を押すと新規セッションのデフォルトを設定可能
- CLI 全体で「extra usage」を「usage credits」に名称変更；`/extra-usage` は `/usage-credits` に（旧名称も引き続き使用可能）
- `api.anthropic.com` に接続できない場合（キャプティブポータル、ファイアウォール、VPN 問題）に起動が最大 75 秒ハングする問題を修正 — サイドチャネル API 呼び出しが 15 秒でタイムアウトするように
- ウィンドウリサイズイベントが失われた後（例：VS Code のスプリットペイン区切りをドラッグ）にターミナル出力が文字化けする問題を修正 — Ctrl+L を要せず次のフレームで自己修復するように
- 長いセッションで発生しターミナルリサイズ・再起動時にしかクリアされなかったターミナル表示の段階的な破損（古い/文字化けしたグリフ）を修正
- スピナーアニメーションの色数を削減し、VS Code でのターミナルレンダリングのちらつきを低減
- プロジェクトが Full Disk Access 保護フォルダ下にある場合に macOS のバックグラウンドセッションが「exit 1 before init」でクラッシュする問題を修正（2.1.143 のリグレッション）
- 画像拡張子と内容が一致しないファイル（例：.png として保存された HTML）を読み込む際に会話が不復旧になる問題を修正 — テキストとしてフォールバックするように
- 検索時のツールエラーを削減：`head`/`tail` ファイルビューが read-before-edit チェックを満たすように、また `egrep`、`fgrep`、`git grep`、`git diff` からの「一致なし」結果（終了コード 1）をコマンド失敗として報告しないように
- ワークツリーに入った後やバックグラウンドセッションで `/branch` が「No conversation to branch」で失敗する問題を修正
- AskUserQuestion のメモフィールドで Escape を押すとターンが中断される代わりに回答選択に戻るように修正
- 起動後に IDE モデルピッカーや `applyFlagSettings` で変更してもモデル選択が適用されない問題を修正
- 再開したセッションが別セッションの `/model` 選択を引き継がず、使用していたモデルを維持するように修正
- Bedrock および Vertex ユーザーが `/model` ピッカーから「Opus (1M context)」を選択できない問題を修正（v2.1.129 のリグレッション）
- `forceLoginMethod` と `forceLoginOrgUUID` を設定したユーザーのリモートセッションログインが「Can't access this organization」で失敗する問題を修正
- ページネーションされた `tools/list` レスポンスを持つ MCP サーバーが最初のページのみ返しツールをサイレントに切り捨てる問題を修正
- サポートされていない MIME タイプ（例：SVG）の MCP 画像が会話を壊す問題を修正 — ディスクに保存してツール結果で参照するように
- スキルディレクトリ内でビルドが実行された際のファイルディスクリプタ枯渇を修正 — `.md` 以外のファイルがスキルリロードをトリガーしないように
- プラグインモニター出力からセッションタイトルが生成される問題を修正（ユーザーの最初のプロンプトから生成されるように）
- ヘッドレスモードで Skill ツールがパーミッションエラーで失敗する問題を修正（v2.1.141 のリグレッション）
- 新しいマシンへの初回読み込み時に自分の設定で有効にしたプラグインが「not cached」エラーを表示する問題を修正；プロジェクトの `.claude/settings.json` のみで有効なプラグインが実行可能な `claude plugin install` ヒントを表示するように
- `.mcp.json` がパースできない場合（例：`"mcpServers"` の代わりに VS Code の `"servers"` キーを使用）に `claude mcp list` がサイレントにサーバーなしと報告する問題を修正 — 設定エラーを表示するように
- カスタム `ANTHROPIC_BASE_URL` 設定と Bedrock Mantle のバックグラウンドサイドクエリが Haiku を使用しない問題を修正 — ファーストパーティ API キーが設定されているか Haiku モデルが設定されていない場合に正しくフォールバックするように
- Windows の接続済みバックグラウンドセッションでのスクロールを修正 — PgUp/PgDn、マウスホイール、Ctrl+O トランスクリプトナビゲーションが動作するように
- バックグラウンドセッションに接続中にターミナルを閉じるとクラッシュする問題を修正
- `! <cmd>` 実行セッションが接続中に Ctrl+C に応答しない問題を修正 — 実行中のコマンドを割り込みするように
- 完了後にエージェントビューのシェルコマンド行が Working に残り、出力が期限切れ後に完了行で Enter を押すとコマンドが再実行される問題を修正
- Windows で `claude agents` 中に ← を押すとリストがキーボード入力に応答しなくなる問題を修正
- Windows Terminal の CJK コンテンツを含む Agent View でペインを切り替えた際に左端にゴースト文字が現れる問題を修正
- `/bg` と `←` によるデタッチが `/add-dir` で追加したディレクトリを保持するように
- インプレース編集中のセッションをデタッチした直後に Edit/Write が「background session hasn't isolated its changes yet」を返す問題を修正
- 停止済みバックグラウンドセッションで `claude respawn <id>` が実行される代わりに「stopped」を表示する問題を修正
- バックグラウンドセッションからフォークされたセッションが `/resume` ピッカーに表示されない問題を修正
- `claude agents` からセッションを開くか `claude logs <id>` を実行した際にバックグラウンドサービスが応答しないとハングする問題を修正 — 10 秒でタイムアウトし復旧ヒントを表示するように
- サブエージェントが生成したバックグラウンド Bash タスクがプロセス終了後も SDK タスクパネルで「Running」のままになる問題を修正
- 完了・停止したバックグラウンドセッションが短期間の起動失敗で恒久的にスタートアップクラッシュとしてマークされる問題を修正
- `claude agents` の接続済みセッションでマークダウンリンクがクリック可能なハイパーリンクではなくプレーンテキストとして表示される問題を修正
- カスタム `spinnerVerbs` がターン後の経過時間メッセージに適用される問題を修正 — 「Worked for 5s」のような過去形の組み込み表現が復元されるように
- `claude agents` / `--bg` の拒否メッセージが汎用メッセージの代わりに具体的なゲート（非 TTY、環境変数、または設定）を名指しするように
- `claude --bg --name <label>` がスポーン後の確認メッセージに名前を反映するように
- `claude agents`：Ctrl+R でバックグラウンドセッションをリネームすると接続済みセッションのバナーが即座に更新されるように
- バックグラウンドセッションのワークツリー分離ガードが `WorktreeCreate` フックを設定した非 git VCS ユーザーにも適用されるように
- プラグインマーケットプレイスの追加/更新が `CLAUDE_CODE_PLUGIN_PREFER_HTTPS` を尊重するように
- `/plugin` がプラグインの有効化・無効化・アンインストール後にインストール済みリストに戻るように
- `/doctor` がコマンドフックに `command` フィールドが欠けている場合に exec 形式の例を表示するように
- スキルリスト切り捨てがスタートアップ通知として表示されなくなり、完全な内訳は `/doctor` で確認可能に
- 稀なレスポンス前のストリームストールからの復旧を改善 — 低速な非ストリーミングリクエストへのフォールバックの代わりにストリーミングを 1 回リトライするように
- SDK/ヘッドレス MCP 起動を改善：事前待機が最初のターン前にブロックする代わりに起動と並行して実行されるように（低速な MCP サーバーで最大 2 秒高速化）
- 調査後のフォローアップヒントがすべての非却下調査回答後にコンテキストに応じたコピーで表示されるようになり、/feedback でより詳細を共有しやすくなった
