- プロンプト、編集 diffstat 付きの 1 行ツール要約、最終応答を表示する `NO_FLICKER` モードのフォーカスビュー切り替え（`Ctrl+O`）を追加
- N 秒ごとにステータスラインコマンドを再実行する `refreshInterval` ステータスライン設定を追加
- 現在のディレクトリがリンクされた git worktree 内にある場合に設定される `workspace.git_worktree` をステータスライン JSON 入力に追加
- `/agents` に live サブエージェントインスタンスを持つエージェントタイプの横に `● N running` インジケータを追加
- Cedar ポリシーファイル（`.cedar`、`.cedarpolicy`）のシンタックスハイライトを追加
- 保護されたパスへの書き込みを承認した後、`--dangerously-skip-permissions` が無言で accept-edits モードにダウングレードされる問題を修正
- Bash ツールの権限を修正と強化。env-var プレフィックスとネットワークリダイレクト周りのチェックを厳格化し、一般的なコマンドでの誤ったプロンプトを削減
- JavaScript プロトタイププロパティ（例: `toString`）と一致する名前の権限ルールが `settings.json` を無言で無視する問題を修正
- 管理対象設定の allow ルールが管理者によって削除された後もプロセス再起動まで有効なままになる問題を修正
- 設定の `permissions.additionalDirectories` 変更がセッションの途中で適用されない問題を修正
- `settings.permissions.additionalDirectories` からディレクトリを削除すると、`--add-dir` 経由で渡された同じディレクトリへのアクセスが失効する問題を修正
- サーバー再接続時に MCP HTTP/SSE 接続が解放されない約 50 MB/hr のバッファを蓄積する問題を修正
- 再起動後のトークン更新時に MCP OAuth の `oauth.authServerMetadataUrl` が尊重されない問題を修正。ADFS および類似の IdP を修正
- サーバーが小さな `Retry-After` を返すときに 429 リトライが約 13 秒ですべての試行を燃やす問題を修正 — 指数バックオフが最小値として適用されるように
- コンテキストコンパクション後にレート制限アップグレードオプションが消える問題を修正
- 複数の `/resume` ピッカー問題を修正: `--resume <name>` が編集不可に開く、Ctrl+A リロードが検索を消す、空のリストがナビゲーションを飲み込む、タスクステータステキストが会話要約を置き換える、クロスプロジェクト stateness
- 編集されたファイルが 10KB を超える場合、`--resume` でファイル編集 diff が消える問題を修正
- トランスクリプトにアタッチメントメッセージが保存されないことによる `--resume` のキャッシュミスとターン中の入力損失を修正
- Claude が作業中に入力されたメッセージがトランスクリプトに永続化されない問題を修正
- 長いセッションでプロンプトタイプの `Stop`/`SubagentStop` フックが失敗する問題、およびフック評価 API エラーが実際のメッセージではなく "JSON validation failed" を表示する問題を修正
- worktree 隔離または `cwd:` 上書きを持つサブエージェントが作業ディレクトリを親セッションの Bash ツールにリークする問題を修正
- プロンプト過長リトライ時にコンパクションが重複したマルチ MB のサブエージェントトランスクリプトファイルを書き込む問題を修正
- リモートに新しいコミットがあるのに git ベースのマーケットプレイスプラグインに対して `claude plugin update` が "already at the latest version" を報告する問題を修正
- プラグインの frontmatter `name` が YAML boolean キーワードのときにスラッシュコマンドピッカーが壊れる問題を修正
- `NO_FLICKER` モードでラップされた URL をコピーすると行区切りにスペースが挿入される問題を修正
- zellij 内で実行中の `NO_FLICKER` モードでのスクロールレンダリングアーティファクトを修正
- MCP ツール結果上にホバーするときの `NO_FLICKER` モードでのクラッシュを修正
- API リトライが古いストリーミング状態を残す `NO_FLICKER` モードのメモリリークを修正
- Windows Terminal の `NO_FLICKER` モードでのマウスホイールスクロールの遅さを修正
- 24 行未満のターミナルで `NO_FLICKER` モードでカスタムステータスラインが表示されない問題を修正
- `NO_FLICKER` モードで Warp の Shift+Enter と Alt/Cmd+arrow ショートカットが動作しない問題を修正
- Windows の no-flicker モードでコピーしたときに韓国語/日本語/Unicode テキストが文字化けする問題を修正
- `AWS_BEARER_TOKEN_BEDROCK` または `ANTHROPIC_BEDROCK_BASE_URL` が空文字列に設定されているとき（GitHub Actions が未設定の入力に対して行うように）、Bedrock SigV4 認証が失敗する問題を修正
- Accept Edits モードを改善。安全な env 変数またはプロセスラッパーでプレフィックスされたファイルシステムコマンドを自動承認（例: `LANG=C rm foo`、`timeout 5 mkdir out`）
- auto モードと bypass-permissions モードがサンドボックスネットワークアクセスプロンプトを自動承認するよう改善
- サンドボックス: macOS で `sandbox.network.allowMachLookup` が有効に
- 画像処理を改善: 貼り付けまたは添付された画像が Read ツール経由で読まれた画像と同じトークン予算に圧縮されるように
- スラッシュコマンドと `@` メンションの補完が CJK の句読点の後にトリガーされるよう改善。日本語/中国語の入力で `/` や `@` の前にスペースが不要に
- claude.ai セッションカードにローカル git リポジトリ、ブランチ、作業ディレクトリを表示するようブリッジセッションを改善
- フッターレイアウトを改善: インジケータ（Focus、notifications）が下にラップせずモード指示行に残るように
- 低コンテキスト警告を永続行ではなく一時的なフッター通知として表示するよう改善
- ラップされた行をまたいで連続的な左バーを表示するよう markdown ブロッククォートを改善
- 空のフックエントリをスキップし、保存された pre-edit ファイルコピーを cap してセッショントランスクリプトサイズを改善
- トランスクリプトの精度を改善: ブロックごとのエントリがストリーミングプレースホルダーではなく最終的なトークン使用量を持つように
- Bash ツールの OTEL トレースを改善: トレースが有効な場合、サブプロセスが W3C `TRACEPARENT` 環境変数を継承
- Claude API と並行して Managed Agents をカバーするよう `/claude-api` スキルを更新
