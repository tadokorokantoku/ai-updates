- Windows: Git for Windows (Git Bash) が不要になりました — 未インストールの場合、Claude Code は PowerShell をシェルツールとして使用します
- CI やスクリプトから `/ultrareview` を非インタラクティブに実行する `claude ultrareview [target]` サブコマンドを追加 — 結果を stdout に出力し（`--json` で raw 出力）、完了時は 0、失敗時は 1 で終了します
- Skills のコンテンツ内で `${CLAUDE_EFFORT}` を使って現在の effort レベルを参照できるようになりました
- サブプロセスに `AI_AGENT` 環境変数をセットし、`gh` が Claude Code からのトラフィックを識別できるようになりました
- デスクトップアプリのインストールや skills/agents の作成を推奨するスピナーのヒントが、既に持っている場合は非表示になるようになりました
- ターミナルがスクロールイベントの代わりに矢印キーを送信する場合に「PgUp/PgDn でスクロールできます」というヒントを表示するようになりました
- 多数の claude.ai コネクタが設定されているが認証されていない場合のセッション開始が高速化されました
- auto モードの拒否メッセージに設定ドキュメントへのリンクが追加されました
- `claude plugin validate` が `marketplace.json` のトップレベルで `$schema`、`version`、`description` を、`plugin.json` で `$schema` を受け付けるようになりました
- auto モードでの auto-compact が誤解を招くトークン数の代わりに `auto`（小文字、トークン数なし）と表示するようになりました
- stdio MCP ツール呼び出し中に Esc を押すとサーバー接続全体が切断される問題を修正（2.1.105 からのリグレッション）
- `claude --resume` で起動後に `/rewind` などのインタラクティブオーバーレイがキーボード入力に反応しない問題を修正
- 非フルスクリーンモードでターミナルのスクロールバックが重複する問題を修正（リサイズ、ダイアログ閉じ、長いセッション）
- `DISABLE_TELEMETRY` / `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` が API および enterprise ユーザーの使用状況メトリクステレメトリを抑制しない問題を修正
- auto モードで pipe とリダイレクトの両方を含む複数行 bash コマンドで偽陽性の「Dangerous rm operation」許可プロンプトが表示される問題を修正
- フルスクリーンモードで長い選択メニューがターミナル下部でクリップされる問題を修正 — スクロール中もフォーカスされたオプションが画面内に留まるようになりました
- フルスクリーンモードで Write ツールの出力の「+N lines」をクリックしても展開されずに折りたたまれる問題を修正
- スラッシュコマンドピッカーが入力中にジャンプする問題を修正、またハイライトが青色で連続した部分文字列のみにマッチするよう改善
- 認識されないソース形式を使用するエントリが 1 つある場合に `/plugin` マーケットプレイスの読み込みに失敗する問題を修正 — そのエントリは表示されますがインストール時に更新を促すプロンプトが表示されます
- [VSCode] `/usage` がプレーンテキストのセッションコストを返す代わりにネイティブの Account & Usage ダイアログを開くようになりました
- [VSCode] 音声ディクテーションが `~/.claude/settings.json` の `language` 設定を尊重するようになりました
- `find` コマンドを含む Bash ツールが大きなディレクトリツリーでファイルディスクリプタを使い果たし、ホスト全体のクラッシュを引き起こす問題を修正（macOS/Linux ネイティブビルド）
