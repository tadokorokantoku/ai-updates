- `OTEL_RESOURCE_ATTRIBUTES` の値がメトリクスデータポイントのラベルとして含まれるようになり、チームやリポジトリなどカスタムディメンションで使用量メトリクスをスライスできるようになった
- `claude agents` の行で、作業がファンアウトしている場合に詳細の前に `done/total` を表示するようになった。ピーク表示では最も長時間実行中のアイテムを表示する
- `/mcp` で、一度もサインインしたことのない claude.ai コネクタが「Show unused connectors」行の後ろに折りたたまれるようになった
- 並列ツール呼び出し: Bash コマンドが失敗しても、同じバッチ内の他の呼び出しがキャンセルされなくなった。各ツールが独立して結果を返す
- フルスクリーンモード: Linux でクリップボードが利用可能な場合に `wl-copy`/`xclip`/`xsel` を使用するようになり、クリップボードと PRIMARY セレクションの両方にコピーしてミドルクリックでのペーストに対応。「{key} を押したままネイティブ選択」のヒントが端末ごとに正しいキーを表示するようになった
- `/effort` ダイアログ、ワークフローアニメーション、プロンプトキーワードシマーが「Reduce motion」設定を尊重しない問題を修正
- `forceLoginOrgUUID`/`forceLoginMethod` の managed-settings ポリシーが org ピンと併用した際にサードパーティプロバイダーのセッション（Bedrock、Vertex、Foundry、Mantle）をブロックしていた問題を修正（2.1.146 からのリグレッション）
- `--output-format text` または `json` 使用時にバックグラウンドサブエージェントの出力が `claude -p` の stdout を破壊していた問題を修正
- `/usage-credits` が Team および Enterprise の管理者に対して組織の使用量設定ページへ誘導せずに再ログインを開始していた問題を修正
- `/autofix-pr` がセッションが git worktree や別のリポジトリ内にある場合に「cannot run on the default branch」と報告していた問題を修正
- `--resume` ピッカーが git worktree でないディレクトリ（例: jj workspaces）の場合に現在のディレクトリのセッションを表示しない問題を修正
- Windows で bash を明示的に起動するフック（例: `/usr/bin/bash script.sh`）が「command not found」や「cannot execute binary file」で失敗していた問題を修正
- テレメトリの初期化完了前に発行された OpenTelemetry ログイベント（`user_prompt`、`api_request`、`tool_result`、`tool_decision`）がサイレントにドロップされていた問題を修正
- `claude mcp` の list/get/add がシークレットをターミナルに出力していた問題を修正: `${VAR}` 参照が展開されなくなり、クレデンシャルヘッダーと URL シークレットが編集されるようになった
- バックグラウンドセッションで `isolation: "worktree"` を使って生成された Workflow エージェントが自身の worktree 内のファイルを編集できなかった問題を修正
- `claude agents` から起動されたバックグラウンドセッションが `settings.json` のモデルではなくデーモンの環境のスタレなモデルで起動していた問題を修正
- セッション再開後に Write ツールの結果をレンダリングする際にクラッシュする可能性があった問題を修正
- 完了したサブエージェントが結果の最終処理中にエラーが発生した際に実行中として表示されたままになっていた問題を修正
- `CLAUDE_CODE_TMPDIR` が深いパスに設定されている場合に `$TMPDIR` 配下の Unix ソケットをバインドするツールで `EADDRINUSE` エラーが発生していた問題を修正
- レイアウトエンジンの JIT コンパイルプロファイルを安定させることでターミナルレンダリングのパフォーマンスを向上
- 大きなファイル書き込みのレンダリングパフォーマンスを向上
- [VSCode] 文字化けグリフの修正として、ターミナル GPU アクセラレーションの無効化（または `/terminal-setup` の実行）を提案するヒントを追加
