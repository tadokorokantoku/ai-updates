- プロキシがボディを解析せずにセッションごとにリクエストを集約できるよう、API リクエストに `X-Claude-Code-Session-Id` ヘッダーを追加
- `.jj` と `.sl` を VCS ディレクトリ除外リストに追加。Grep とファイル autocomplete が Jujutsu または Sapling のメタデータに降下しないように
- v2.1.85 より前に作成されたセッションで `--resume` が "tool_use ids were found without tool_result blocks" で失敗する問題を修正
- 条件付きスキルまたはルールが設定されているとき、プロジェクトルート外のファイル（例: `~/.claude/CLAUDE.md`）で Write/Edit/Read が失敗する問題を修正
- スキル呼び出しのたびに不要な設定ディスク書き込みが発生し、Windows でパフォーマンス問題と設定破損を引き起こす可能性がある問題を修正
- 大きなトランスクリプトファイルを持つ非常に長いセッションで `/feedback` を使用する際の out-of-memory クラッシュの可能性を修正
- `--bare` モードがインタラクティブセッションで MCP ツールをドロップし、ターンの途中でキューされたメッセージを無言で破棄する問題を修正
- `c` ショートカットが OAuth ログイン URL の最初の約 20 文字のみコピーする問題を修正
- 狭いターミナルでマスクされた入力（例: OAuth コードの貼り付け）が複数行にラップされたときにトークンの先頭がリークする問題を修正
- 公式マーケットプレイスプラグインスクリプトが v2.1.83 以降 macOS/Linux で "Permission denied" で失敗する問題を修正
- 複数の Claude Code インスタンスを実行していて、その 1 つで `/model` を使用しているときに、ステータスラインが別のセッションのモデルを表示する問題を修正
- 長い会話の最下部でホイールスクロールまたはクリック選択した後、スクロールが新しいメッセージに従わない問題を修正
- `/plugin` アンインストールダイアログで `n` を押すと、プラグインのデータディレクトリを保持しながら正しくアンインストールされるように修正
- クリック後に Enter を押すと、応答が到着するまでトランスクリプトが空白のまま残るリグレッションを修正
- キーワードを削除した後も `ultrathink` ヒントが残る問題を修正
- 長いセッションで markdown/highlight レンダーキャッシュが完全なコンテンツ文字列を保持することによるメモリ増加を修正
- 多くの claude.ai MCP コネクタが設定されているときの起動時イベントループストールを削減（macOS keychain キャッシュを 5s から 30s に延長）
- `@` でファイルをメンションする際のトークンオーバーヘッドを削減 — 生の文字列コンテンツが JSON エスケープされなくなった
- ツール記述から動的コンテンツを削除することで Bedrock、Vertex、Foundry ユーザーのプロンプトキャッシュヒット率を改善
- "Saved N memories" 通知のメモリファイル名がホバーでハイライトされクリックで開くように
- コンテキスト使用量を削減するため、`/skills` リストのスキル記述が 250 文字で cap されるように
- 閲覧しやすくするため `/skills` メニューがアルファベット順にソートされるように変更
- プラン制限によって無効化された場合、auto モードが "temporarily unavailable" ではなく "unavailable for your plan" を表示するように
- [VSCode] 長時間実行操作中に拡張機能が誤って "Not responding" を表示する問題を修正
- [VSCode] OAuth トークンが更新されたとき（ログイン後 8 時間）に Max プランユーザーがデフォルトで Sonnet になる問題を修正
- Read ツールがコンパクトな行番号フォーマットを使用し、変更のない再読み込みを重複排除。トークン使用量を削減
