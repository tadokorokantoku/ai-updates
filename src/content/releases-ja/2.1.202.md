- `/config` に「Dynamic workflow size」設定を追加。Claude が動的な workflow をどの程度の規模で作るか（small/medium/large のエージェント数）を制御できる — これは強制的な上限ではなく、目安のガイドライン
- workflow が生成したエージェントが送出するテレメトリに `workflow.run_id` と `workflow.name` の OpenTelemetry 属性を追加し、OTel データから workflow 実行のアクティビティを再構築できるようにした
- インラインの Ctrl+R 履歴検索で、履歴ファイルのスキャン中に確定またはキャンセルするとクラッシュする問題を修正
- バックグラウンドセッションでの `/rename` がジョブの再起動時に元に戻ってしまい、新しい名前でセッションを指定できなくなる問題を修正
- クライアント証明書のインプレースローテーション中に設定が再適用された際に、一時的な mTLS ハンドシェイクの失敗が発生する問題を修正
- Remote Control（モバイル/web）からインタラクティブセッションに送信したコマンドが「Unknown command」で失敗する問題を修正
- Remote Control のモバイルまたは web アプリからキャプションなしで送信した画像やファイルが黙って破棄される問題を修正
- `claude auth login` および `claude mcp login --no-browser` が出力するサインイン URL が、SSH 経由で折り返された際に確実にクリック可能にならない問題を修正 — 単一のハイパーリンクとして出力されるようになった
- `claude agents` からチャットを開く際に「currently running as a background agent」で失敗し、その後ワーカーのクラッシュ/再起動ループに陥ることがある問題を修正
- 文字列中に unicode のクォートエスケープを含む workflow スクリプトが、パース前に壊れてしまう問題を修正。workflow のパースエラーは、常に TypeScript が原因であるかのように表示するのではなく、問題の行を表示するようになった
- マイクまたはオーディオレコーダーが失敗した際に、音声入力の再試行が無制限に繰り返される問題を修正 — キャプチャの失敗が連続すると音声入力を一時停止するようになった
- `/remote-control` セッションがモバイルおよび web アプリで誤った permission mode を表示する問題を修正
- git worktree が多数あるリポジトリで、セッションを名前で resume する、または resume ピッカーを開く際に数分かかり大量のメモリを使用する問題を修正
- プロキシやネットワークが原因でダウンロード中に接続が切れた際に、インストーラーおよびアップデーターのダウンロードが即座に「aborted」で失敗する問題を修正 — 一時的な接続切断時にはリトライするようになった
- 既に読み込まれている skill を再度呼び出した際に、その指示内容が重複してコンテキストに追加される問題を修正
- `/workflows` のエージェント一覧レイアウトを改善：タイトル欄を広く、時間専用の列を追加、モデル名を短縮、行ごとのツール呼び出し数を非表示に
- MCP のエラーメッセージを改善：サーバー設定に `url` があり `type` がない場合、わかりにくい "command: expected string" ではなく `"type": "http"` を提案する明確なエラーを表示
- `/review <pr>` を高速なシングルパスレビューに戻した。効果レベルを選べるマルチエージェントレビューには `/code-review <level> <pr#>` を使用する
