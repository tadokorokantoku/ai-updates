- `/simplify` と `/batch` バンドルスラッシュコマンドを追加
- /cost のようなローカルスラッシュコマンドの出力が UI でシステムメッセージではなくユーザー送信メッセージとして表示される問題を修正
- プロジェクト設定と自動メモリが同じリポジトリの git worktree 間で共有されるように
- claude.ai MCP サーバーの利用をオプトアウトする `ENABLE_CLAUDEAI_MCP_SERVERS=false` 環境変数を追加
- スラッシュコマンドメニューで現在アクティブなモデルを表示するよう `/model` コマンドを改善
- シェルコマンド実行ではなく URL に JSON を POST して JSON を受信できる HTTP フックを追加
- ブリッジポーリングループのリスナーリークを修正
- MCP OAuth フローのクリーンアップでのリスナーリークを修正
- MCP OAuth 認証中の手動 URL 貼り付けフォールバックを追加。自動 localhost リダイレクトが動作しない場合、コールバック URL を貼り付けて認証を完了可能
- フック設定メニューをナビゲートするときのメモリリークを修正
- 自動承認中のインタラクティブ権限ハンドラのリスナーリークを修正
- ファイル数キャッシュが glob ignore パターンを無視する問題を修正
- bash コマンドプレフィックスキャッシュのメモリリークを修正
- サーバー再接続時の MCP ツール/リソースキャッシュリークを修正
- IDE ホスト IP 検出キャッシュがポート間で誤って結果を共有する問題を修正
- transport 再接続時の WebSocket リスナーリークを修正
- 長時間実行セッションで無制限に増加する可能性があった git root 検出キャッシュのメモリリークを修正
- 長いセッションで無制限に増加する JSON 解析キャッシュのメモリリークを修正
- VSCode: リモートセッションが会話履歴に表示されない問題を修正
- REPL ブリッジのレース条件を修正。初期接続フラッシュ中に新しいメッセージが履歴メッセージと交互にサーバーに到着し、メッセージの順序問題を引き起こす可能性があった
- 長時間実行のチームメイトが会話のコンパクション後も AppState にすべてのメッセージを保持するメモリリークを修正
- MCP サーバーの fetch キャッシュが切断時にクリアされず、頻繁に再接続するサーバーでメモリ使用量が増加するメモリリークを修正
- コンテキストコンパクション中に重い進捗メッセージペイロードを削除することで、サブエージェントを持つ長いセッションのメモリ使用量を改善
- `/copy` ピッカーに "Always copy full response" オプションを追加。選択すると、今後の `/copy` コマンドはコードブロックピッカーをスキップして完全な応答を直接コピー
- VSCode: セッションリストにセッションのリネームと削除アクションを追加
- `/clear` がキャッシュされたスキルをリセットしないことで、新しい会話に古いスキルコンテンツが残る可能性があった問題を修正
