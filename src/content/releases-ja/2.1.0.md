- スキルの自動ホットリロードを追加。`~/.claude/skills` または `.claude/skills` で作成または変更されたスキルが、セッションを再起動せずに即座に利用可能に
- スキルの frontmatter で `context: fork` を使って、スキルやスラッシュコマンドをフォークしたサブエージェントコンテキストで実行するサポートを追加
- 実行するエージェントタイプを指定する、スキルの `agent` フィールドのサポートを追加
- Claude の応答言語を設定する `language` 設定を追加（例: language: "japanese"）
- Shift+Enter を iTerm2、WezTerm、Ghostty、Kitty でターミナル設定を変更せずにそのまま動作するように変更
- @ メンションのファイルピッカー動作をプロジェクトごとに制御する `respectGitignore` を `settings.json` でサポート
- UI からメールアドレスと組織を隠す `IS_DEMO` 環境変数を追加。配信や録画セッションに便利
- 機密データ（OAuth トークン、API キー、パスワード）がデバッグログに露出する可能性があるセキュリティ問題を修正
- `-c` または `--resume` でセッションを再開するときにファイルとスキルが適切に検出されない問題を修正
- 上矢印や Ctrl+R 検索から履歴のプロンプトを再生したときに貼り付けたコンテンツが失われる問題を修正
- キューのプロンプトがあるときに Esc キーで実行中のタスクをキャンセルせず入力欄に戻すように修正
- 複雑な bash コマンドの権限プロンプトを減少
- コマンド検索を改善し、コマンド名の完全一致と前方一致を、説明文のあいまい一致より優先
- `PreToolUse` フックが `ask` 権限判定を返すときに `updatedInput` を許可するように修正。フックをミドルウェアとして機能させつつユーザー同意を求めることが可能に
- ファイルベースのマーケットプレイスソースのプラグインパス解決を修正
- LSP サーバーが設定されていないときに LSP ツールが誤って有効化される問題を修正
- 名前にドットを含むリポジトリで、バックグラウンドタスクが「git repository not found」エラーで失敗する問題を修正
- WSL 環境での Claude in Chrome サポートを修正
- 実行ファイル作成失敗時に Windows ネイティブインストーラーが静かに失敗する問題を修正
- CLI ヘルプ出力でオプションとサブコマンドをアルファベット順に表示してナビゲーションを容易に
- Bash ツール権限のワイルドカードパターンマッチングを追加。ルール内の任意の位置で `*` を使用可能（例: `Bash(npm *)`、`Bash(* install)`、`Bash(git * main)`）
- bash コマンドとエージェント両方の統一された Ctrl+B バックグラウンド化を追加。Ctrl+B を押すとすべての実行中フォアグラウンドタスクが同時にバックグラウンド化
- MCP `list_changed` 通知のサポートを追加。MCP サーバーが再接続なしに利用可能なツール、プロンプト、リソースを動的に更新可能に
- claude.ai のサブスクライバー向けに `/teleport` と `/remote-env` スラッシュコマンドを追加。リモートセッションの再開と設定が可能に
- settings.json の permissions または `--disallowedTools` CLI フラグで `Task(AgentName)` 構文を使って特定のエージェントを無効化するサポートを追加
- エージェントの frontmatter でフックサポートを追加。エージェントのライフサイクルにスコープを持つ PreToolUse、PostToolUse、Stop フックを定義可能に
- スキルおよびスラッシュコマンドの frontmatter でフックをサポート
- 新しい Vim モーションを追加: `;` と `,` で f/F/t/T モーションを繰り返し、`y` 演算子で yank と `yy`/`Y`、貼り付けの `p`/`P`、テキストオブジェクト（`iw`、`aw`、`iW`、`aW`、`i"`、`a"`、`i'`、`a'`、`i(`、`a(`、`i[`、`a[`、`i{`、`a{`）、インデント/デデントの `>>` と `<<`、行連結の `J`
- プロンプトから直接 plan モードを有効にする `/plan` コマンドショートカットを追加
- スラッシュコマンドの自動補完を、先頭だけでなく入力のどこに `/` が現れても動作するように追加
- 対話セッション中に Claude が使える組み込みツールを制限する、対話モードでの `--tools` フラグサポートを追加
- デフォルトのファイル読み込みトークン上限を上書きする `CLAUDE_CODE_FILE_READ_MAX_OUTPUT_TOKENS` 環境変数を追加
- フック用の `once: true` 設定サポートを追加
- クリーンなスキル宣言のために、frontmatter の `allowed-tools` フィールドで YAML スタイルのリストをサポート
- プラグインからのプロンプトおよびエージェントのフック種別のサポートを追加（以前はコマンドフックのみサポート）
- iTerm2 で画像ペースト用の Cmd+V サポートを追加（Ctrl+V にマップ）
- ダイアログでタブを巡回する左右矢印キーのナビゲーションを追加
- Ctrl+O トランスクリプトモードでリアルタイムの thinking ブロック表示を追加
- バックグラウンド bash タスク詳細ダイアログの全出力にファイルパスを追加
- コンテキスト可視化で Skills を独立したカテゴリとして追加
- サーバーがトークンの有効期限切れを報告しつつローカルの有効期限チェックと一致しないときに OAuth トークンのリフレッシュがトリガーされない問題を修正
- 一時的なサーバーエラー後にセッション永続化が停滞する問題を、エントリが実際に保存された場合の 409 コンフリクトから回復することで修正
- 同時ツール実行中の孤立したツール結果が原因でセッション再開が失敗する問題を修正
- 同時トークンリフレッシュ中にキーチェーンキャッシュから古い OAuth トークンが読み取られる競合状態を修正
- AWS Bedrock のサブエージェントが EU/APAC クロスリージョン推論モデル設定を継承せず、IAM 権限が特定のリージョンにスコープされているときに 403 エラーが発生する問題を修正
- バックグラウンドタスクが大きな出力を生成するときの API コンテキストオーバーフローを、ファイルパス参照と共に 30K 文字に切り詰めることで修正
- 特殊ファイル種別のシンボリックリンク解決をスキップすることで、FIFO ファイル読み込み時のハングを修正
- Ghostty、iTerm2、Kitty、WezTerm で終了時にターミナルのキーボードモードがリセットされない問題を修正
- iTerm2、Ghostty、Kitty、WezTerm で Alt+B と Alt+F（単語移動）が動作しない問題を修正
- プラグイン `allowed-tools` frontmatter で `${CLAUDE_PLUGIN_ROOT}` が置換されず、ツールが誤って承認を要求していた問題を修正
- Write ツールで作成されるファイルがシステムの umask を尊重せずハードコードされた 0o600 権限を使用する問題を修正
- `$()` コマンド置換を含むコマンドがパースエラーで失敗する問題を修正
- バックスラッシュ継続を含む複数行 bash コマンドが誤って分割され権限でフラグされる問題を修正
- グローバルオプション後のサブコマンドを bash コマンドプレフィックス抽出が正しく識別するように修正（例: `git -C /path log` が `Bash(git log:*)` ルールに正しくマッチ）
- CLI 引数として渡されたスラッシュコマンド（例: `claude /context`）が正しく実行されない問題を修正
- Tab でスラッシュコマンドを補完した後に Enter を押すと、補完したコマンドを送信するのではなく別のコマンドが選択される問題を修正
- 引数付きのコマンド入力時にスラッシュコマンドの引数ヒントがちらつき、表示が一貫しない問題を修正
- スラッシュコマンドを直接実行するときに Claude が冗長に Skill ツールを呼び出すことがある問題を修正

- Esc でキャンセルしたときにキューのプロンプト内の画像が「[object Object]」として表示される問題を修正

- CJK 文字（日本語、中国語、韓国語）を含む複数行プロンプトの余分な空行を修正

- 折り畳まれた「Reading X files…」インジケーターが途中で thinking ブロックが現れると過去形に誤って切り替わる問題を修正

- 多数のセルを持つ Jupyter notebook 読み込み時のパフォーマンスを改善
