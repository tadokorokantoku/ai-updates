- ログイン画面で「3rd-party platform」を選択したときにアクセスできるインタラクティブな Google Vertex AI セットアップウィザードを追加。GCP 認証、プロジェクトとリージョン設定、認証情報検証、モデル pinning をガイド
- `CLAUDE_CODE_PERFORCE_MODE` 環境変数を追加: 設定されると、Edit/Write/NotebookEdit が読み取り専用ファイルに対して無言で上書きする代わりに `p4 edit` ヒント付きで失敗
- バックグラウンドスクリプトからのイベントをストリーミングする Monitor ツールを追加
- `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB` が設定されたときの Linux での PID namespace 隔離付きのサブプロセスサンドボックス、および `CLAUDE_CODE_SCRIPT_CAPS` 環境変数でセッションごとのスクリプト呼び出しを制限
- クロスユーザープロンプトキャッシングを改善するため、print モードに `--exclude-dynamic-system-prompt-sections` フラグを追加
- 現在のディレクトリがリンクされた git worktree 内にあるときはいつでも設定される `workspace.git_worktree` をステータスライン JSON 入力に追加
- OTEL トレースが有効な場合に Bash ツールのサブプロセスに W3C `TRACEPARENT` 環境変数を追加。子プロセススパンが Claude Code のトレースツリーに正しく親子関係を持つように
- LSP: Claude Code が initialize リクエストの `clientInfo` 経由で言語サーバーに自己識別するように
- バックスラッシュエスケープされたフラグが読み取り専用として自動許可され、任意コード実行につながる Bash ツールの権限バイパスを修正
- auto モードと bypass-permissions モードで複合 Bash コマンドが安全性チェックと明示的な ask ルールの強制権限プロンプトをバイパスする問題を修正
- env-var プレフィックス付きの読み取り専用コマンドが、既知の安全な変数（`LANG`、`TZ`、`NO_COLOR` など）以外の場合にプロンプトを表示しない問題を修正
- `/dev/tcp/...` または `/dev/udp/...` へのリダイレクトが自動許可される代わりにプロンプトを表示するよう修正
- 停滞したストリーミング応答が非ストリーミングモードにフォールバックする代わりにタイムアウトする問題を修正
- サーバーが小さな `Retry-After` を返すときに 429 リトライが約 13 秒ですべての試行を燃やす問題を修正 — 指数バックオフが最小値として適用されるように
- 再起動後のトークン更新時に MCP OAuth の `oauth.authServerMetadataUrl` 設定オーバーライドが尊重されない問題を修正。ADFS と類似の IdP に影響
- kitty キーボードプロトコルがアクティブのとき、xterm と VS Code 統合ターミナルで大文字が小文字にドロップされる問題を修正
- macOS のテキスト置換が置換を挿入する代わりにトリガーワードを削除する問題を修正
- Bash 経由で保護されたパスへの書き込みを承認した後、`--dangerously-skip-permissions` が無言で accept-edits モードにダウングレードされる問題を修正
- 管理対象設定の allow ルールが管理者によって削除された後もプロセス再起動までアクティブに保たれる問題を修正
- `permissions.additionalDirectories` 変更がセッション中に適用されない問題を修正 — 削除されたディレクトリは即座にアクセスを失い、追加されたものは再起動なしで動作
- `additionalDirectories` からディレクトリを削除すると `--add-dir` 経由で渡された同じディレクトリへのアクセスが失効する問題を修正
- `Bash(cmd:*)` と `Bash(git commit *)` のワイルドカード権限ルールが余分なスペースやタブを含むコマンドにマッチしない問題を修正
- `Bash(...)` deny ルールが `cd` を他のセグメントと混合したパイプされたコマンドでプロンプトにダウングレードされる問題を修正
- `cut -d /`、`paste -d /`、`column -s /`、`awk '{print $1}' file`、および `%` を含むファイル名に対する誤った Bash 権限プロンプトを修正
- JavaScript プロトタイププロパティ（例: `toString`）と一致する名前の権限ルールが `settings.json` を無言で無視する問題を修正
- `--dangerously-skip-permissions` 使用時にエージェントチームメンバーがリーダーの権限モードを継承しない問題を修正
- フルスクリーンモードで MCP ツール結果上にホバーするときのクラッシュを修正
- フルスクリーンモードでラップされた URL をコピーすると行区切りにスペースが挿入される問題を修正
- 編集されたファイルが 10KB を超える場合、`--resume` で UI からファイル編集 diff が消える問題を修正
- 複数の `/resume` ピッカー問題を修正: `--resume <name>` が編集不可で開く、フィルターリロードが検索状態を消す、空のリストが矢印キーを飲み込む、クロスプロジェクト staleness、一時的なタスクステータステキストが会話要約を置き換える
- `/export` が絶対パスと `~` を尊重せず、ユーザー指定の拡張子を無言で `.txt` に書き換える問題を修正
- 未知または将来のモデル ID に対して `/effort max` が拒否される問題を修正
- プラグインの frontmatter `name` が YAML boolean キーワードのときにスラッシュコマンドピッカーが壊れる問題を修正
- メッセージ remount 後にレート制限アップセルテキストが非表示になる問題を修正
- `_meta["anthropic/maxResultSizeChars"]` を持つ MCP ツールがトークンベースの persist レイヤーをバイパスしない問題を修正
- 前のトランスクリプトがまだ処理中に push-to-talk キーを再度押し続けると、voice モードが入力に数十のスペース文字をリークする問題を修正
- `DISABLE_AUTOUPDATER` が npm ベースのインストールで npm レジストリバージョンチェックとシンボリックリンク変更を完全に抑制しない問題を修正
- セッションのライフタイムの間、Remote Control permission handler エントリが保持されるメモリリークを修正
- エラーで失敗するバックグラウンドサブエージェントが部分的な進捗を親エージェントに報告しない問題を修正
- 長いセッションでプロンプトタイプの Stop/SubagentStop フックが失敗する問題、およびフック評価 API エラーが実際のメッセージではなく "JSON validation failed" を表示する問題を修正
- 却下されたときのフィードバック調査レンダリングを修正
- Bash `grep -f FILE` / `rg -f FILE` が作業ディレクトリ外のパターンファイルを読むときにプロンプトを表示しない問題を修正
- 古いサブエージェント worktree クリーンアップが未追跡ファイルを含む worktree を削除する問題を修正
- `sandbox.network.allowMachLookup` が macOS で有効にならない問題を修正
- `/resume` フィルターヒントラベルを改善し、フィルターインジケータにプロジェクト/worktree/ブランチ名を追加
- 狭いターミナル幅でラップする代わりにモード指示行に残るようフッターインジケータ（Focus、notifications）を改善
- タブ付きレイアウトで `/agents` を改善: Running タブは live サブエージェントを表示し、Library タブは Run agent と View running instance アクションを追加
- 再起動なしでプラグイン提供のスキルを取得するよう `/reload-plugins` を改善
- 安全な env 変数またはプロセスラッパーでプレフィックスされたファイルシステムコマンドを自動承認するよう Accept Edits モードを改善
- Vim モードを改善: NORMAL モードの `j`/`k` が履歴をナビゲートし、入力境界でフッター pill を選択
- `--debug` なしで自己診断するために、トランスクリプトのフックエラーが stderr の最初の行を含むよう改善
- OTEL トレースを改善: インタラクションスパンが同時 SDK 呼び出しで完全なターンを正しくラップし、ヘッドレスターンがターンごとにスパンを終了
- ストリーミングプレースホルダーではなく最終的なトークン使用量を持つようトランスクリプトエントリを改善
- Claude API と並行して Managed Agents をカバーするよう `/claude-api` スキルを更新
- [VSCode] `CLAUDE_CODE_GIT_BASH_PATH` が設定されているか、Git がデフォルトの場所にインストールされている場合の Windows での誤判定 "requires git-bash" エラーを修正
- `CLAUDE_CODE_MAX_CONTEXT_TOKENS` が設定されたときに `DISABLE_COMPACT` を尊重するよう修正
- `DISABLE_COMPACT` が設定されたときに `/compact` ヒントをドロップ
