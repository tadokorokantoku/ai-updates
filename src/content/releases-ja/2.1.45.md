- Claude Sonnet 4.6 のサポートを追加
- `--add-dir` ディレクトリから `enabledPlugins` と `extraKnownMarketplaces` を読み込むサポートを追加
- `spinnerTipsOverride` 設定を追加。スピナーのヒントをカスタマイズ可能。`tips` にカスタムヒント文字列の配列を設定し、任意で `excludeDefault: true` を設定すると、組み込みのヒントの代わりにカスタムヒントのみを表示
- SDK に `SDKRateLimitInfo` と `SDKRateLimitEvent` 型を追加。コンシューマーが使用率、リセット時刻、超過情報などのレート制限ステータス更新を受け取れるように
- Agent Teams のチームメイトが Bedrock、Vertex、Foundry で失敗する問題を修正。API プロバイダーの環境変数を tmux で spawn したプロセスに伝播 (anthropics/claude-code#23561)
- macOS で一時ファイルを書き込む際のサンドボックス「operation not permitted」エラーを修正。正しいユーザー別 temp ディレクトリを使用 (anthropics/claude-code#21654)
- Task ツール（バックグラウンド化されたエージェント）が完了時に `ReferenceError` でクラッシュする問題を修正 (anthropics/claude-code#22087)
- 入力に画像が貼り付けられた際、Enter でオートコンプリートの候補が確定されない問題を修正
- サブエージェントが起動したスキルがコンパクション後にメインセッションコンテキストに誤って表示される問題を修正
- 起動のたびに `.claude.json.backup` ファイルが過剰に蓄積される問題を修正
- プラグインが提供するコマンド、エージェント、フックがインストール後に再起動しないと利用できない問題を修正
- 統計キャッシュ用のセッション履歴の早期ロードを削除して起動パフォーマンスを改善
- 大きな出力を生成するシェルコマンドのメモリ使用量を改善。RSS がコマンド出力サイズに応じて際限なく増えなくなった
- 折りたたまれた read/search グループを改善。実行中に要約行の下に処理中のファイルや検索パターンを表示
- [VSCode] 権限の保存先選択（project/user/session）をセッションをまたいで永続化するように改善
