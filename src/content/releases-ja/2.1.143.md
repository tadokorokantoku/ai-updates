## 変更内容

- プラグインの依存関係強制を追加：`claude plugin disable` は別の有効なプラグインが対象に依存している場合に拒否（コピー可能な無効化チェーンのヒントを表示）し、`claude plugin enable` は推移的依存関係を強制的に有効化するようになった
- `/plugin` マーケットプレイスの閲覧ペインに予測コンテキストコスト（ターンごと・起動ごとのトークン推定）を追加
- `worktree.bgIsolation: "none"` 設定を追加：worktree が実用的でないリポジトリで、バックグラウンドセッションが `EnterWorktree` なしにワーキングコピーを直接編集できるようにする
- PowerShell ツールが `-ExecutionPolicy Bypass` を渡すようになった。`CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1` でオプトアウト可能
- バックグラウンドセッションがアイドルから復帰した後に設定したモデルとエフォートレベルを保持するようになった
- アタッチされたエージェントセッションで Shift+Tab がオートモードをサイクルに含めるようになった
- `.credentials.json` の `scopes` 値が配列でない場合に CLI の起動時にハングしたり OAuth トークンリフレッシュが中断されたりする問題を修正
- Windows Terminal および WSL での `claude agents` の右クリックペーストを修正
- ストップフックが繰り返しブロックし続けて無限ループする問題を修正 — 8回連続ブロック後に警告を出してターンを終了（`CLAUDE_CODE_STOP_HOOK_BLOCK_CAP` で上書き可能）
- Claude がイテレーション間でアイドル中に Esc/Ctrl+C で保留中の `/loop` のウェイクアップをキャンセルできない問題を修正
- バックグラウンドシェルや委譲されたサブエージェントがまだ実行中に `/goal` エバリュエーターが起動する問題を修正
- settings.json の `env` に設定した `NO_COLOR`/`FORCE_COLOR` が Claude Code 自身の UI カラーを削除する問題を修正 — サブプロセスにのみ適用されるようになった
- セッション一覧表示時に Windows でエージェントビューが PowerShell プロセスを繰り返し生成する問題を修正
- プロンプトなしの `/bg` がフォークされたセッションに "continue" を送信する問題を修正 — フォークは入力を待つようになった
- `--agent <name>` で `plugin:` プレフィックスなしにプラグイン提供のエージェントが見つからない問題を修正
- エージェントビューからセッションを削除してもトランスクリプトファイルが削除されない問題を修正
- Windows Terminal でアタッチされたバックグラウンドセッションのスクロール時に古いフラグメントがレンダリングされる問題を修正
- ホストのスリープや macOS App Nap 後にバックグラウンドエージェントのワーカー停止検出が誤検知のストームを起こす問題を修正
- 5xx エラーメッセージが設定済みのゲートウェイやクラウドプロバイダーの名前を表示せず status.claude.com を指している問題を修正
- PowerShell ツールが Windows の Bedrock、Vertex、Foundry ユーザーでデフォルトで有効になった。`CLAUDE_CODE_USE_POWERSHELL_TOOL=0` でオプトアウト可能
- `claude agents` が `--add-dir`、`--settings`、`--mcp-config`、`--plugin-dir` を受け付け、ダッシュボードとそこから起動されるバックグラウンドセッションに適用されるようになった
- `claude agents` が `--permission-mode`、`--model`、`--effort`、`--dangerously-skip-permissions` を受け付け、そこから起動されるセッションのデフォルトを設定できるようになった
- `claude --bg --dangerously-skip-permissions` がリタイア→復帰後も永続するようになった
- バックグラウンドセッションがウォームスペアの入力に IDE ファイル参照を取り込む問題を修正 — `claude agents` から次のプロンプトの先頭に参照が付加されていた
- `git worktree remove` が失敗した場合に worktree クリーンアップが `rm -rf` にフォールバックしなくなり、gitignore されたファイルや作業中ファイルの損失を防止
- Full Disk Access が付与されていても macOS のバックグラウンドジョブセッションが `~/Documents`、`~/Desktop`、`~/Downloads` 配下のファイルを読み取る際に "Operation not permitted" エラーが発生する問題を修正
- `/bg` が `--mcp-config`、`--settings`、`--add-dir`、`--plugin-dir`、`--strict-mcp-config` を保持するようになり、バックグラウンドセッションが再起動後も MCP サーバーと設定を維持するようになった
- `claude agents` から起動されたバックグラウンドセッションが settings.json の `permissions.defaultMode` を尊重するようになった（以前はオートモードに上書きされていた）
- 修正：Windows で `claude agents` でレスポンスがストリーミング中に ← を押すとエージェントリストがすべての入力に反応しなくなる問題
- `/bg` と `←` でのデタッチが `--fallback-model` を保持するようになり、過負荷時にバックグラウンドワーカーがハードフェイルする代わりにフォールバックモデルに降格するようになった
- `/bg` と `←` でのデタッチが `--allow-dangerously-skip-permissions` を保持するようになり、フォークされたワーカーの Shift+Tab サイクルでバイパスパーミッションが利用可能になった
- 修正：`~/.local/bin/claude` ランチャーが存在しないか実行不可の場合、バックグラウンドデーモンの起動が実行中のバイナリにフォールバックするようになった
- `claude agents --allow-dangerously-skip-permissions` が起動されたセッションをデフォルトでバイパスモードにする問題を修正 — パーミッションサイクルで利用可能にするようになった
