- `ENABLE_TOOL_SEARCH` が設定されていれば `ANTHROPIC_BASE_URL` でもツール検索をアクティブにするよう修正
- フォーカスされた選択をクリップボードをバイパスして直接ファイルに書き出す `w` キーを `/copy` に追加（SSH 経由で便利）
- プランモードに入って即座に開始するオプションの description 引数を `/plan` に追加（例: `/plan fix the auth bug`）
- `EnterWorktree` セッションを離脱する `ExitWorktree` ツールを追加
- スケジュールされた cron ジョブをセッション中に即時停止する `CLAUDE_CODE_DISABLE_CRON` 環境変数を追加
- bash 自動承認 allowlist に `lsof`、`pgrep`、`tput`、`ss`、`fd`、`fdfind` を追加。一般的な読み取り専用操作の権限プロンプトを削減
- Agent ツールの `model` パラメータを復元。呼び出しごとのモデル上書きに
- effort レベルを low/medium/high にシンプル化（max を削除）、新しいシンボル（○ ◐ ●）と永続アイコンの代わりに簡潔な通知。`/effort auto` でデフォルトにリセット
- `/config` を改善。Escape で変更をキャンセル、Enter で保存して閉じる、Space で設定を切り替え
- 複数の同時セッションを実行しているときに up-arrow 履歴が現在のセッションのメッセージを最初に表示するよう改善
- リポジトリ名や一般的な開発用語（regex、OAuth、JSON）のための voice 入力転写精度を改善
- ネイティブモジュールに切り替えて bash コマンド解析を改善。初期化が速くメモリリークなし
- バンドルサイズを約 510 KB 削減
- 自動注入時に Claude から CLAUDE.md の HTML コメント（`<!-- ... -->`）を非表示に変更。Read ツールで読むとコメントは可視のまま
- バックグラウンドタスクやフックの応答が遅いときの終了の遅さを修正
- エージェントタスクの進捗が "Initializing…" でスタックする問題を修正
- フック有効スキルがモデルによって起動されると、イベントごとにスキルフックが 2 回発火する問題を修正
- 複数の voice モード問題を修正: 時折の入力ラグ、push-to-talk 解放後の誤った「No speech detected」エラー、送信後の古いトランスクリプトがプロンプトを再充填する問題
- `--compact` 後に `--continue` が最新ポイントから再開しない問題を修正
- bash セキュリティ解析のエッジケースを修正
- `.git` サフィックスなしのマーケットプレイス git URL のサポートを追加（Azure DevOps、AWS CodeCommit）
- git が stderr を生成しない場合でも診断情報を表示するようマーケットプレイスクローン失敗メッセージを改善
- 複数のプラグイン問題を修正: OneDrive フォルダでの `EEXIST` エラーによる Windows でのインストール失敗、プロジェクトスコープインストールが存在するときのマーケットプレイスのユーザースコープインストールブロック、`CLAUDE_CODE_PLUGIN_CACHE_DIR` がリテラル `~` ディレクトリを作成する問題、マーケットプレイス専用フィールドを持つ `plugin.json` のロード失敗
- 長いセッションで頻繁に表示されるフィードバック調査を修正
- 起動時の無関係な設定書き込みによって `--effort` CLI フラグがリセットされる問題を修正
- `/clear` 後にバックグラウンド化された Ctrl+B クエリがトランスクリプトを失うか新しい会話を破損する問題を修正
- `/clear` がバックグラウンドのエージェント/bash タスクを終了する問題を修正。フォアグラウンドタスクのみがクリアされるように
- worktree 隔離の問題を修正: Task ツールの resume が cwd を復元しない、バックグラウンドタスク通知で `worktreePath` と `worktreeBranch` が欠落する
- Claude が作業中に `/model` が結果を表示しない問題を修正
- プランモードの権限プロンプトのテキスト入力で digit キーがメニューオプションを選択する問題を修正
- サンドボックス権限問題を修正: 特定のファイル書き込み操作がプロンプトなしで誤って許可される、allowlisted ディレクトリ（`/tmp/claude/` など）への出力リダイレクトが不必要にプロンプトを表示する
- 長いセッションでの CPU 使用率を改善
- SDK `query()` 呼び出しでのプロンプトキャッシュ無効化を修正。入力トークンコストを最大 12 倍削減
- クエリをキャンセル後に Escape キーが無反応になる問題を修正
- バックグラウンドエージェントまたはタスクが実行中のときにダブル Ctrl+C で終了しない問題を修正
- チームエージェントがリーダーのモデルを継承するよう修正
- 二度とマッチしない権限ルールを保存する「Always Allow」を修正
- 複数のフック問題を修正: 再開/フォークセッションで `transcript_path` が誤ったディレクトリを指す、エージェントの `prompt` が設定書き込みのたびに settings.json から無言で削除される、PostToolUse ブロック理由が 2 回表示される、async フックが bash `read -r` で stdin を受け取らない、検証エラーメッセージが検証に失敗する例を表示する
- U+2028/U+2029 文字を含むファイルが Read されたときの Desktop/SDK のセッションクラッシュを修正
- `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` が設定されているときにも終了時にターミナルタイトルがクリアされる問題を修正
- 複数の権限ルールマッチング問題を修正: heredoc、埋め込み改行、引数のないコマンドにワイルドカードルールがマッチしない、env 変数プレフィックスで `sandbox.excludedCommands` が失敗する、「always allow」がネストされた CLI ツールに対して過度に広いプレフィックスを提案する、deny ルールがすべてのコマンド形式に適用されない
- Bash data-URL 出力からの過大で切り詰められた画像を修正
- Bedrock API エラーを含むセッションを再開するときのクラッシュを修正
- Edit、Bash、Grep ツール入力での断続的な "expected boolean, received string" 検証エラーを修正
- 最初のメッセージに改行が含まれる会話からフォークする際の複数行セッションタイトルを修正
- キューされたメッセージが添付画像を表示しない問題を修正。↑ を押してキューされたメッセージを編集する際に画像が失われる問題
- Read/WebFetch/Glob のいずれかが失敗すると兄弟がキャンセルされる並列ツール呼び出しの問題を修正。Bash エラーのみカスケード
- VSCode: 統合ターミナルのスクロール速度がネイティブターミナルと一致しない問題を修正
- VSCode: 古いキーバインドのユーザーで Shift+Enter が改行を挿入する代わりに入力を送信する問題を修正
- VSCode: 入力境界に effort レベルインジケータを追加
- VSCode: 新しい Claude Code タブをプログラム的に開く `vscode://anthropic.claude-code/open` URI ハンドラを追加。オプションの `prompt` と `session` クエリパラメータ付き
