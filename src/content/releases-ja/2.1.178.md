- パーミッションルールにツールの入力パラメータをマッチさせる `Tool(param:value)` 構文を追加（`*` ワイルドカード対応）。例：`Agent(model:opus)` で Opus サブエージェントをブロック
- ネストした `.claude/skills` ディレクトリ内のスキルが、そのディレクトリ内のファイルを編集する際に読み込まれるようになりました。名前が衝突する場合、ネストしたスキルは `<dir>:<name>` として表示され、両方を利用できます
- ネストした `.claude/` ディレクトリ：名前が衝突する場合、作業ディレクトリに最も近いエージェント・ワークフロー・出力スタイルが優先されるようになりました。プロジェクトスコープのワークフロー保存は最も近い既存の `.claude/workflows/` を対象にします
- auto モードを改善：サブエージェントの起動前に classifier が評価するようになり、サブエージェントがレビューなしにブロック対象のアクションをリクエストできた問題を解消
- `/doctor` を改善：全セクションで一貫したフラットツリーレイアウト、より明確なセクションステータスアイコン、ハイライトされたコマンド名を提供
- スキルリストの切り捨て警告に影響を受けるスキルの説明数を表示するよう改善
- ワークフロープロンプトのキーワードを紫のシマーハイライトに変更し、"run a workflow" や "workflow:" などの明示的なフレーズのみでトリガーするよう変更（単語への言及では発動しない）
- Remote Control エラーメッセージを改善：接続失敗時にフッターに永続的な赤い "/rc failed" インジケーターを表示し、"not yet enabled" エラーがゲート・チェック失敗・期限切れエンタイトルメント・組織ポリシーのどれに該当するか説明するようになりました
- `/bug` が送信前に説明の入力を必須とするようになり、モデルの拒否テキストを GitHub issue タイトルとして使用しなくなりました
- 親プロセスから古い websocket/OAuth ファイルデスクリプタ環境変数を継承した際に CLI がクラッシュ（メモリ不足）する問題を修正
- OAuth トークンが Claude Code のログインとは別のアカウントに属している場合に Chrome の Claude が無言で接続に失敗する問題を修正
- 非インタラクティブな実行でディレクトリ修飾名を持つネストした `.claude/skills` スキルがパーミッションプロンプトにブロックされる問題を修正
- サブエージェントの複数の問題を修正：サブエージェントのトランスクリプト表示でツール結果とライブ進捗が表示されるようになり、サブエージェントがターンを終了する際に送信されたメッセージが失われなくなり、実行中のサブエージェントをバックグラウンド化（`ctrl+b`）しても最初から再起動されなくなりました
- カスタム API ゲートウェイ（`ANTHROPIC_BASE_URL` および `ANTHROPIC_AUTH_TOKEN`）を持つシェルからデーモンが起動された際に `claude agents` のワーカーが `401 Invalid bearer token` で失敗する問題を修正
- `--fallback-model` が compaction で適用されない問題を修正：compaction がオーバーロードまたはモデル利用不可エラー時に設定されたフォールバックモデルチェーンにフォールバックするようになりました
- セッション外で認証情報が更新された後も古いキャッシュされたリクエスト設定により認証エラーでモデルリクエストが失敗し続ける問題を修正
- ターン終了後に `/bg` または `←←` で作成されたバックグラウンドセッションがエージェントリストで "Working" と表示し続ける問題を修正
- `CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1` が新規マーケットプレイスインストールのクローンを妨げる問題を修正
- サブエージェントの `disallowedTools` で MCP サーバーレベルのスペック（`mcp__server`、`mcp__server__*`、`mcp__*`）が無言で無視される問題を修正
- vim モードの undo を修正：`u` が素早い連続コマンドを単一の undo ステップにまとめる代わりに、NORMAL/VISUAL モードのコマンドを1つずつ元に戻すようになりました
- `claude agents` でカスタム URI スキーム（例：`vscode://`）を持つ statusline リンクをクリックしても開かない問題を修正
- [VSCode] CJK IME 候補ウィンドウを Esc で閉じると実行中の Claude タスクがキャンセルされる問題を修正
