- `Edit(src/**)` のような単一セグメントの `dir/**` allow ルールが、`<cwd>/dir` だけでなくツリー内のどこにあるネストした `dir/` ディレクトリへの書き込みも自動承認してしまう問題を修正
- Windows PowerShell 5.1 セッションで実行されたコマンドに影響する権限チェックのバイパスを修正
- Bash 権限チェックが、bash と権限アナライザーで解釈が異なるファイルディスクリプタのリダイレクト形式に対してフェイルクローズ（安全側に倒す）ように修正
- Bash 権限チェックが非常に長いコマンドを誤判定する問題を修正 — 10,000 文字を超えるコマンドは自動実行されず、常にプロンプトが表示されるように
- Bash 権限チェックが `[[ ]]` 比較内の zsh 変数の添字や修飾子を無害なテキストとして扱っていた問題を修正 — これらのコマンドは承認プロンプトが表示されるように
- Bash 権限チェックが、安全でないオプション、コマンド置換、バックスラッシュパスを実行しうる一部の `help` および `man` コマンドを自動承認してしまう問題を修正
- リモートセッションでの権限プロンプトが、ローカルの確認ダイアログより先に進んでしまう問題を修正
- EndConversation ツールを追加: 2025年以降 claude.ai で行われているように、Claude が極めて悪質なユーザーやジェイルブレイクの試みに対してセッションを終了できるように — 詳細は https://www.anthropic.com/research/end-subset-conversations
- 長時間実行されるツール呼び出しで、これまで無音状態になっていた箇所に定期的な進捗ハートビートを追加
- メモリファイルの frontmatter に ISO 形式の `modified` タイムスタンプを追加
- メッセージ単位の相関とツールの出所追跡のため、OpenTelemetry のログイベントに `message.uuid`、`client_request_id`、`tool_source` 属性を追加
- OpenTelemetry のコンテンツ属性の 60 KB 切り詰め上限を設定できる `CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH` を追加
- `subagentStatusLine` のペイロードに reasoning effort を追加し、カスタムエージェント行でモデルと effort を表示できるように
- デーモンリダイレクトフラグ（`--url`、`--connection`、`--identity`、および Podman のリモートモード）を伴う `docker` コマンド（Podman の `docker` シムを含む）に対する権限プロンプトを追加 — これまではプロンプトなしで実行されていた
- GrowthBook のフィーチャーが null と評価された際のクラッシュと、不正なフラグペイロードによってキャッシュされたフィーチャーフラグが消去されてしまう問題を修正
- `pkill -f` パターンが誤って CLI 自身のプロセスにマッチした際、Bash ツールが Claude セッションを終了させてしまう問題を修正（Linux）
- `--settings` がデバイスファイルや数GB規模の巨大ファイルを指している場合の無制限なメモリ増加を修正 — 2 MiB を超える設定ファイルは起動時に明確なエラーで失敗するように
- Windows のコーポレートプロキシ経由でストリーミングターンが "Socket is closed" で失敗する問題を修正
- 読み込みの遅い SDK/パイプラインのコンシューマーに対する stream-json 出力が終了時に切り詰められる問題を修正 — 終了時のドレイン処理が、固定 2 秒の上限ではなくキューされたバイト量に応じてスケールするように
- スケジュールされたタスクが、自身の設定済みプロンプトを信頼できない入力として拒否してしまう問題を修正 — 発火したプロンプトはセッションに割り当てられたタスクとして配信されるように
- 子プロセスが標準入力を待機している場合に、PowerShell ツールのコマンドがタイムアウトまでハングする問題を修正（Windows）
- PowerShell ツール経由の Python スクリプトが、標準入力から非 UTF-8 データを読み込む際に UnicodeDecodeError でクラッシュする問題を修正（Windows）
- PowerShell ツール経由で実行される Python スクリプトが非 ASCII 出力で UnicodeEncodeError によりクラッシュする問題と、PowerShell 7 のエラーメッセージに生の ANSI エスケープシーケンスが含まれる問題を修正（Windows）
- PowerShell ツールが、`where.exe`、`fc.exe`、`diff.exe` が有効な否定的な回答を返した際にエラーとして報告してしまう問題を修正（Windows）
- Windows PowerShell 5.1 の PowerShell ツール上で `>` および `>>` が UTF-16LE 形式でファイルを書き込み、他のツールが UTF-8 として読み込めなくなる問題を修正
- 取って代わられたバックグラウンドデーモンがシャットダウン時に後継のコントロールソケットを削除してしまい、次のクライアントが正常な後継デーモンを kill してしまう問題を修正
- `←` または `/background` でパークされ、アイドル状態のまま放置されたバックグラウンドセッションが、バックグラウンドデーモンとワーカープロセスを無期限に生かし続けてしまう問題を修正
- バックグラウンドサービスがアイドル状態になった後、完了したバックグラウンドセッションを `claude rm` またはエージェントビューから削除できなくなる問題を修正
- git 管理外のフォルダから起動されたバックグラウンドセッションを、エージェントビューから削除できない問題を修正
- セッションストア内に読み込み不可なフォルダが存在する場合、停止済みのバックグラウンドセッションを再度開いても保存済みの会話が復元されない問題を修正
- Remote Control が明示的に有効化されていないセッションに対しても、Remote Control の "session ready" プッシュ通知が発火してしまう問題を修正
- エージェントビューのセッションで `/install-github-app` と `/mcp` の設定メニューがブロックされてしまう問題を修正 — ターミナルが接続されていないバックグラウンドセッションでのみ拒否されるように
- `--settings` CLI フラグ経由で有効化されたプラグインが読み込まれない問題を修正（v2.1.181 からのリグレッション）
- OAuth トークンがローテーションした後、長時間実行セッションでフィーチャーフラグが古いままになる問題を修正
- マージベースが存在しないリポジトリで `/ultrareview` が実行を拒否する問題を修正 — 代わりに追跡対象の全ファイルをレビューする選択肢を提示するように
- シェル設定のパスがディレクトリである場合に、`claude update` と `claude doctor` が無言でハングし、`/status` のシステム診断セクションが空白になる問題を修正
- メモリファイル保存時に、frontmatter の値がインラインの `#` で暗黙的に切り詰められてしまう問題を修正
- 複数の累積 `message_delta` フレームを送出するストリームで、セッションのコストとトークンのテレメトリが二重カウントされる問題を修正
- advisor が思考中に表示される、誤った "check your network" 警告を修正
- フックの stdout JSON がスキーマ検証に失敗した際、終了コード 2 のフックがドキュメント通りにブロックしない問題を修正
- ターンの非同期コンテキスト外で発行された OTel ログイベントに、インタラクションスパンのトレースコンテキストが欠落する問題を修正
- prompts/resources のリフレッシュ中に発生する MCP の一時的なエラーが、サーバーのスラッシュコマンドとリソースをクリアしてしまう問題を修正
- ホームディレクトリでの `claude rc` のワークスペース信頼エラーを改善 — ホームディレクトリでは信頼設定が保存されないことを明示し、プロジェクトディレクトリから実行するよう提案するように
- 単一セグメントの `dir/**` フックの `if:` 条件が `<cwd>/dir` のみにマッチするように変更 — 任意の深さでマッチさせたい場合は `**/dir/**` を使用。`deny`/`ask` の権限ルールは引き続き任意の深さでマッチする
- `-m`/`--magic-file` または `-f`/`--files-from` を使用する `file` コマンドが、読み取り専用として自動許可されるのではなく権限を要求するように変更
- keep-alive のコネクションプーリングを、stale-connection エラー発生後は無効化するように変更 — リトライ時に新しいソケットを開くように
- SessionStart フックが、セッションがフォークとして開始された場合に source として `"resume"` ではなく `"fork"` を報告するように変更
