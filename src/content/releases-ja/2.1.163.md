- `requiredMinimumVersion` と `requiredMaximumVersion` のマネージド設定を追加 — バージョンが許可範囲外の場合、Claude Code の起動を拒否して承認済みバージョンへ誘導する
- インストール済みプラグインを一覧表示する `/plugin list` コマンドを追加（`--enabled`/`--disabled` フィルター付き）
- `/btw` に「c でコピー」ショートカットを追加 — raw Markdown の回答をクリップボードにコピーし、他の場所に貼り付けても書式を保持する
- Hooks: Stop および SubagentStop フックが `hookSpecificOutput.additionalContext` を返せるようになり、フックエラーとしてラベル付けされることなく Claude にフィードバックを渡してターンを継続できる
- Skills: コマンドボディで数字の前にリテラルの `$` を含めるための `\$` エスケープ構文を追加
- stdio MCP サーバーが `--resume` 時にフック/Bash と同じ `CLAUDE_CODE_SESSION_ID` を受け取るようになった
- バックグラウンドコマンドが終了しない場合に `claude -p` が最終結果後に永久にハングしていた問題を修正 — stdin が閉じた後、結果から約 5 秒でバックグラウンドシェルが停止されるようになった
- `CI=true` かつ Anthropic API キーが未設定の場合に Bedrock/Vertex/Foundry で `claude -p` が「ANTHROPIC_API_KEY required」で失敗していた問題を修正
- bazel および EDR 保護された Go ワークフローで Bash コマンドが失敗していた問題を修正: `$TMPDIR` がサンドボックス化されたコマンドのみではなく全コマンドで `/tmp/claude-{uid}` に上書きされていた（2.1.154 のリグレッション）
- セッション環境ディレクトリに読み取り専用属性がある場合や OneDrive 内にある場合に、Windows で Bash コマンドが「EEXIST: file already exists」で失敗していた問題を修正
- 新規設定ディレクトリでのスタートアップ中にマネージド設定の取得が完了した際に組織管理のパーミッションルールがセッション全体に適用されなかった問題を修正
- Claude Code のアップデート後に再アタッチすると `claude agents` のバックグラウンドセッションが実行中のバックグラウンドタスクを失っていた問題を修正
- Esc を押してエージェントビューを終了する際のターミナルのズレと数秒間のハングを修正
- デスクトップアプリでバックグラウンドタスクチップの Stop をクリックしても、基礎となるプロセスがすでに終了している場合にチップがクリアされなかった問題を修正
- ターミナルによって終了マーカーがドロップされるペースト操作後にキーボード入力が永久に無反応になっていた問題を修正
- `$()` や `$VAR` を含む全 Bash コマンドでフックの `if: "Bash(...)"` 条件が発火していた問題を修正。パターンがサブシェルやバッククォート内のコマンドにも正しく照合されるようになった
- ホームディレクトリパス（例: `Read(~/Desktop/**)`)の deny ルールが `$HOME` 経由でパスを参照する Bash コマンドをブロックしなかった問題を修正
- `/mcp` や `/plugins` などのパネルダイアログを閉じた後にトランスクリプトに「(no content)」の行が残っていた問題を修正
- バックグラウンドエージェントセッションがバックグラウンドで新しい Claude Code バージョンに更新されるようになり、アップデート後にセッションを開いてもコールドリスタートを待たなくなった
- `/` メニューの組み込みコマンドとスキルの説明を明確化
- サブスクリプション切り替えの提案がトーストではなくスタートアップアナウンスメントスロットに表示されるようになった
- `claude agents` の状態グループ表示からのディスパッチで、エージェントビューを開いたディレクトリからセッションが開始されるようになった
