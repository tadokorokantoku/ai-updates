- 大きなセッションでの `/resume` を大幅に高速化 (40MB 超のセッションで最大 67%)、多数の dead-fork エントリを持つセッションをより効率的に処理
- 複数の stdio サーバーが設定されている場合の MCP 起動を高速化; `resources/templates/list` が最初の `@` メンションまで遅延するように
- VS Code、Cursor、Windsurf ターミナルでのフルスクリーンスクロールをよりスムーズに — `/terminal-setup` がエディタのスクロール感度を設定するように
- Thinking スピナーが進捗をインライン表示するように ("still thinking"、"thinking more"、"almost done thinking")、別のヒント行を置き換え
- `/config` の検索がオプション値にもマッチするように (例: "vim" で検索すると Editor mode 設定がヒット)
- Claude の応答中でも現在のターンの完了を待たずに `/doctor` を開けるように
- `/reload-plugins` とバックグラウンドのプラグイン自動更新が、既に追加したマーケットプレイスから欠落しているプラグイン依存を自動インストールするように
- Bash tool が `gh` コマンドで GitHub API のレート制限に達した際にヒントを表示するように、エージェントがリトライではなくバックオフできるように
- Settings の Usage タブが 5 時間/週の使用量を即座に表示するように、usage エンドポイントがレート制限されても失敗しないように
- エージェント frontmatter の `hooks:` が `--agent` 経由でメインスレッドエージェントとして実行される際にも発火するように
- スラッシュコマンドメニューがフィルタ結果 0 件の場合、消えずに "No commands match" を表示するように
- Security: サンドボックスの auto-allow が `/`、`$HOME`、その他の重要なシステムディレクトリを対象とする `rm`/`rmdir` の危険パスチェックをバイパスしないように
- ターミナル UI でデーヴァナーガリー文字やその他のインド系文字の列揃えが崩れる問題を修正
- Kitty キーボードプロトコルを使用するターミナル (iTerm2、Ghostty、kitty、WezTerm、Windows Terminal) で Ctrl+- が undo をトリガーしない問題を修正
- Kitty キーボードプロトコルを使用するターミナル (Warp フルスクリーン、kitty、Ghostty、WezTerm) で Cmd+Left/Right が行頭/行末にジャンプしない問題を修正
- Claude Code がラッパープロセス (例: `npx`、`bun run`) 経由で起動された際に Ctrl+Z がターミナルをハングさせる問題を修正
- インラインモードでターミナルのリサイズや大量出力バーストにより過去の会話履歴が繰り返されるスクロールバック重複の問題を修正
- ターミナル高が低い際にモーダル検索ダイアログが画面をオーバーフローし、検索ボックスとキーボードヒントが隠れる問題を修正
- VS Code 統合ターミナルでスクロール中に空セルが散在し composer クロームが消える問題を修正
- リクエストセットアップ中に並列リクエストが完了した際に発生しうる cache control TTL 順序に関連する断続的な API 400 エラーを修正
- `/branch` が 50MB 超のトランスクリプトを持つ会話を拒否する問題を修正
- `/resume` が大きなセッションファイルでロードエラーを報告せず無音で空の会話を表示する問題を修正
- `/plugin` の Installed タブで、Needs attention または Favorites に現れるアイテムが二重表示される問題を修正
- セッション中に worktree に入った後に `/update` と `/tui` が動作しなくなる問題を修正
