- auto モードの安全性を強化：破壊的な git コマンド（`git reset --hard`、`git checkout -- .`、`git clean -fd`、`git stash drop`）はローカルの変更を破棄するよう明示的に依頼していない場合はブロック、`git commit --amend` は今セッションでエージェントが作成したコミット以外はブロック、`terraform destroy`/`pulumi destroy`/`cdk destroy` は特定のスタックを明示的に指定しない限りブロック
- リクエストされたモデルが非推奨または新しいモデルへ自動的に更新される場合に警告を追加。print モード（`-p`）では stderr に表示され、agent frontmatter に設定されたモデルにも対応
- `attribution.sessionUrl` 設定を追加。Web および Remote Control セッションのコミットや PR から claude.ai セッションリンクを省略できる
- `/config --help` を追加し、`/config key=value` で使用できるすべての短縮キーを一覧表示
- `/config` のトグル動作を変更：Enter と Space の両方で選択した設定を変更できるようになり、Esc が変更を取り消す代わりに保存して閉じるように
- ロゴ下の起動時「setup issues」行を削除 — 設定の問題を確認するには `/doctor` を実行するか `--debug` を使用
- 影響のある設定でサブエージェントのスポーンやセッションタイトル生成時に発生する `thinking.disabled.display: Extra inputs are not permitted` 400 エラーを修正
- サブエージェントで WebSearch が空の結果を返す問題を修正
- ネイティブカーソルが有効な vim モードで履歴を移動した後、ターミナルカーソルがプロンプトの上に残る問題を修正
- 重いネストされたサブエージェント負荷下で Windows Terminal でフルスクリーン TUI が破損する問題を修正（ステータスラインが画面中央に表示、スピナー行が重複、テキストが結合）
- モデルが thinking ブロックのみを返した場合にターンが視覚的な出力なしで静かに完了する問題を修正；Claude は一度再プロンプトするように
- 複数のプラグインが有効な場合に、ユーザーレベルのスキルがスラッシュコマンドのオートコンプリートに複数回表示される問題を修正
- ヘッドレス/SDK モードで認証が必要な MCP サーバーが auth-stub ツールをモデルに公開する問題を修正
- シェルの rc ファイル初期化が遅い場合に tmux テイムメイトペインの起動に失敗する問題と、エージェントスポーン中に入力されたキーストロークが新しい tmux ペインではなくリーダープロンプトに漏れる問題を修正
- テイムメイトがターンを終了したときに、テイムメイトが開始したバックグラウンドタスクが強制終了される問題を修正
- スケジュールされたタスクと webhook トリガーの配信がキーボード入力として扱われる問題を修正；タスク通知として分類されるようになり、auto モードで保留中のアクションを承認したりセッションタイトルを設定したりできなくなった
- フォーカスモードで各レスポンスの下に「Ran N PostToolUse hooks」タイミング行が表示される問題を修正
