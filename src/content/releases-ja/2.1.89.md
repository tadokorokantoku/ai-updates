- `PreToolUse` フックに `"defer"` 権限判定を追加 — ヘッドレスセッションがツール呼び出しで一時停止し、`-p --resume` で再開してフックを再評価可能
- 仮想化 scrollback 付きのちらつきのない alt-screen レンダリングにオプトインする `CLAUDE_CODE_NO_FLICKER=1` 環境変数を追加
- auto モードの分類器による拒否の後に発火する `PermissionDenied` フックを追加 — `{retry: true}` を返してモデルがリトライ可能と伝える
- 名前付きサブエージェントを `@` メンション typeahead 候補に追加
- `-p` モードで MCP 接続待機を完全にスキップする `MCP_CONNECTION_NONBLOCKING=true`、および最も遅いサーバーでブロックする代わりに `--mcp-config` サーバー接続を 5 秒で境界設定
- auto モード: 拒否されたコマンドが通知を表示し、`/permissions` → Recent タブに表示され、`r` でリトライ可能
- `Edit(//path/**)` と `Read(//path/**)` の allow ルールが、リクエストされたパスだけでなく解決されたシンボリックリンクターゲットをチェックするよう修正
- 一部の modifier 組み合わせバインディングで voice push-to-talk がアクティベートされない問題、および Windows での voice モードが "WebSocket upgrade rejected with HTTP 101" で失敗する問題を修正
- Edit/Write ツールが Windows で CRLF を重複させ、Markdown のハード改行（末尾の 2 スペース）を削除する問題を修正
- 複数のスキーマ使用時に約 50% の失敗率を引き起こす `StructuredOutput` スキーマキャッシュバグを修正
- 長時間実行セッションで大きな JSON 入力が LRU キャッシュキーとして保持されるメモリリークを修正
- 非常に大きなセッションファイル（50MB 以上）からメッセージを削除するときのクラッシュを修正
- クラッシュ後の LSP サーバーのゾンビ状態を修正 — セッション再起動まで失敗し続ける代わりに、次のリクエストでサーバーが再起動
- CJK または絵文字を含むプロンプト履歴エントリが `~/.claude/history.jsonl` の 4KB 境界に落ちると無言でドロップされる問題を修正
- サブエージェント使用量を除外してトークンを過小カウントする `/stats` と、stats キャッシュフォーマットが変更されたときに 30 日を超える履歴データが失われる問題を修正
- 遅延されたツール入力が 64KB を超えるか deferred マーカーが存在しない場合に `-p --resume` がハングする問題、および `-p --continue` が deferred ツールを再開しない問題を修正
- macOS で `claude-cli://` ディープリンクが開かない問題を修正
- サーバーが multi-element エラーコンテンツを返したときに MCP ツールエラーが最初のコンテンツブロックのみに切り詰められる問題を修正
- SDK 経由で画像付きメッセージを送信するときに、スキルリマインダーとその他のシステムコンテキストがドロップされる問題を修正
- Write/Edit/Read ツールの PreToolUse/PostToolUse フックが `file_path` を絶対パスとして受信するよう修正。文書化された動作と一致
- autocompact スラッシュループを修正 — 3 回連続でコンパクション直後にコンテキストが限界まで再充填されることを検出し、API 呼び出しを浪費するのではなく実行可能なエラーで停止
- セッション中にツールスキーマバイトが変更されることによる長いセッションでのプロンプトキャッシュミスを修正
- 多くのファイルを読む長いセッションでネストされた CLAUDE.md ファイルが何十回も再注入される問題を修正
- トランスクリプトに古い CLI バージョンからのツール結果または中断された書き込みが含まれるときの `--resume` クラッシュを修正
- API が entitlement エラーを返したときの誤解を招く "Rate limit reached" メッセージを修正 — 実行可能なヒント付きで実際のエラーを表示
- フックの `if` 条件フィルタリングが複合コマンド（`ls && git push`）または env-var プレフィックス付きコマンド（`FOO=bar git push`）にマッチしない問題を修正
- 重い並列ツール使用中にターミナル scrollback で折りたたまれた search/read グループバッジが重複する問題を修正
- 通知 `invalidates` が現在表示されている通知を即座にクリアしない問題を修正
- 処理中にバックグラウンドメッセージが到着すると送信後にプロンプトが一瞬消える問題を修正
- デバナガリなど結合マーク付きのテキストがアシスタント出力で切り詰められる問題を修正
- レイアウトシフト後のメインスクリーンターミナルでのレンダリングアーティファクトを修正
- macOS Apple Silicon で voice モードがマイク権限の要求に失敗する問題を修正
- Windows Terminal Preview 1.25 で Shift+Enter が改行挿入ではなく送信する問題を修正
- tmux 内で iTerm2 を実行中のストリーミング時の周期的な UI ジッターを修正
- Windows PowerShell 5.1 で `git push` のようなコマンドが stderr に進捗を書き込むときに PowerShell ツールが誤って失敗を報告する問題を修正
- 非常に大きなファイル（>1 GiB）に Edit ツールを使用したときの out-of-memory クラッシュの可能性を修正
- `ls`/`tree`/`du` の折りたたまれたツール要約を "Read N files" ではなく "Listed N directories" を表示するよう改善
- フォーマッタ/リンタコマンドが以前に読んだファイルを変更したときに Bash ツールが警告するよう改善。stale-edit エラーを防止
- 類似名の MCP リソースよりソースファイルを上位にランク付けするよう `@` メンション typeahead を改善
- バージョン適切な構文ガイダンス（5.1 vs 7+）を提供するよう PowerShell ツールプロンプトを改善
- 最初に別の `Read` 呼び出しを要求せず、`Bash` で `sed -n` または `cat` を介して表示されたファイルに対して `Edit` が動作するよう変更
- コンテキストに直接注入するのではなく、50K 文字を超えるフック出力をファイルパスとプレビュー付きでディスクに保存するよう変更
- settings.json の `cleanupPeriodDays: 0` が検証エラーで拒否されるよう変更 — 以前はトランスクリプト永続化を無言で無効化していた
- インタラクティブセッションで thinking 要約がデフォルトで生成されないよう変更 — 復元するには settings.json に `showThinkingSummaries: true` を設定
- `TaskCreated` フックイベントとそのブロック動作を文書化
- Ctrl+B で実行中のコマンドをバックグラウンド化するときのタスク通知を保持
- Windows 上の PowerShell ツール: 二重引用符と空白の両方を含む外部コマンド引数が自動許可ではなくプロンプトを表示（PS 5.1 引数分割の強化）
- `/env` が PowerShell ツールコマンドにも適用されるように（以前は Bash にのみ影響）
- `/usage` が Pro と Enterprise プランで冗長な "Current week (Sonnet only)" バーを非表示に
- 画像ペーストが末尾のスペースを挿入しなくなった
- 空のプロンプトに `!command` を貼り付けると bash モードに入るように（typed `!` の動作と一致）
- 4 月 1 日向けに `/buddy` が登場 — あなたのコーディングを見守る小さな生き物を孵化させよう
