  - "`/simplify` を `/code-review` に名称変更（エフォートレベル指定に対応）"
---

- `/simplify` を `/code-review` に名称変更し、オプションでエフォートレベルを指定できるようになった（例：`/code-review high`）
- Auto モードで、ユーザーまたはスキルが明示的に依存している場合に `AskUserQuestion` が抑制されなくなった
- winget または Microsoft Store 経由で `pwsh` がインストールされている場合に Windows PowerShell ツールが「command line is invalid」エラーで失敗する問題を修正した（v2.1.124 のリグレッション）
- MCP の `resources/list`、`resources/templates/list`、`prompts/list` でページネーションするサーバーの 2 ページ目以降のアイテムが消える問題を修正した
- Claude がストリーミング中に Windows Terminal でアタッチされたバックグラウンドセッションのフルスクリーンがちらつく問題を修正した
- 更新に失敗したときに自動アップデーターのステータスラインに現在のバージョンが表示されない問題を修正した
- Windows でバックグラウンドジョブの worktree を削除した際に NTFS ジャンクションを経由してメインリポジトリ内を辿ってしまう問題を修正した
- スキルまたはカスタムスラッシュコマンドのみが入力されたセッションを `/background` が拒否していた問題を修正した
- バックグラウンド化されたセッションで「今後聞かない」で承認済みのツール権限を再度確認してしまう問題を修正した
- `/theme` のカラーエディターおよび「New custom theme」ダイアログで Esc キーが反応しない問題を修正した
- Agent SDK 経由で実行した場合にストリーミングセッション終了時に未捕捉の例外が発生する問題を修正した
- `forceLoginOrgUUID` および `forceLoginMethod` のマネージド設定ポリシーがサードパーティプロバイダーおよび API キーのセッションに対して適用されない問題を修正した
- GNOME Terminal で右クリックおよびミドルクリックによるペーストがテキストを挿入しない問題を修正した
- マルチエージェントセッションで `CLAUDE_CODE_SUBAGENT_MODEL` が子プロセスに転送されない問題を修正した
- 自動アップデーターの信頼性を向上させた。ネイティブのバージョン確認とダウンロードで、一時的なネットワーク障害が発生した場合に即座に失敗するのではなくリトライするようになった
- 大きなファイルの編集における差分レンダリングのパフォーマンスを改善した
