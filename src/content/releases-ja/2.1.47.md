- FileWriteTool の行カウントを修正。`trimEnd()` で削除せず、意図的な末尾空行を保持
- 表示コード内の `os.EOL` (`\r\n`) により Windows ターミナルのレンダリングバグが発生する問題を修正。行数が Windows で常に 1 と表示される代わりに正しい値を表示するように
- VS Code プランプレビューを改善。Claude がイテレーションするたびに自動更新、プランがレビュー可能になったときのみコメント有効化、却下時もプレビューを開いたままにして Claude が修正できるように
- `\r\n` 改行により Windows で markdown 出力の太字や色付きテキストが誤った文字にシフトする問題を修正
- 会話に多数の PDF ドキュメントが含まれる際にコンパクションが失敗する問題を修正。コンパクション API に送信する前に画像と共にドキュメントブロックも削除 (anthropics/claude-code#26188)
- 長時間実行セッションで API ストリームバッファ、エージェントコンテキスト、スキル状態を使用後に解放してメモリ使用量を改善
- SessionStart フックの実行を遅延させて起動パフォーマンスを改善。interactive までの時間を約 500ms 短縮
- Windows の MSYS2 や Cygwin シェルで bash tool の出力が無言で破棄される問題を修正
- `@` ファイルメンションのパフォーマンスを改善。起動時にインデックスを事前ウォームし、バックグラウンド更新付きのセッションベースキャッシュを使用することで、ファイル候補がより速く表示されるように
- タスク完了後にエージェントタスクメッセージ履歴をトリムしてメモリ使用量を改善
- 進捗更新における O(n²) のメッセージ蓄積を解消し、長時間のエージェントセッション中のメモリ使用量を改善
- bash 権限分類器を修正。返されたマッチ記述が実際の入力ルールに対応するかを検証し、幻覚された記述が誤って権限を付与するのを防止
- ゼロ inode を報告する NFS/FUSE ファイルシステムでユーザー定義エージェントが 1 ファイルしか読み込まれない問題を修正 (anthropics/claude-code#26044)

- 折りたたまれたツール結果の検索パターンを明確さのために引用符で表示するように
- Windows: CWD 追跡の一時ファイルがクリーンアップされず無限に蓄積する問題を修正 (anthropics/claude-code#17600)
- ESC の二連打ではなく `ctrl+f` ですべてのバックグラウンドエージェントを停止。バックグラウンドエージェントは ESC でメインスレッドをキャンセルしても実行を継続し、エージェントのライフサイクルをより細かく制御可能に
