  - "インセッションのサブエージェントを起動する `/subtask` コマンドの追加"
---

- `/fork` が会話を新しいバックグラウンドセッション（`claude agents` の独立した行）にコピーするようになり、作業を続けながら分岐できるように — これまでインセッションのサブエージェントを起動していた役割は `/subtask` に移行
- デフォルトの auto-mode 設定に戻す `claude auto-mode reset` を追加（確認プロンプトあり、`--yes` でスキップ可能）
- WebSearch ツール呼び出しのセッション全体での上限を追加（デフォルト 200、`CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION` で調整可能）— 暴走する検索ループを防止
- サブエージェント生成のセッションごとの上限を追加（デフォルト 200、`CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION` で上書き可能）— 暴走する委譲ループを防止。`/clear` で予算がリセットされる
- 2分以上かかる MCP ツール呼び出しは自動的にバックグラウンドに移行し、セッションが使用可能な状態を保つように — しきい値の設定や無効化は `CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS` で
- エージェントビューで `/resume` と入力すると、過去のセッション（一覧から削除されたセッションを含む）のピッカーが開き、選択したセッションをバックグラウンドセッションとして再開するように
- プランモードがファイルを変更する Bash コマンド（`touch`、`rm` など）を、権限プロンプトや SDK の `canUseTool` コールバックなしに自動実行してしまう問題を修正
- リポジトリにコミットされた `.claude/worktrees` のシンボリックリンクを worktree 作成が辿ってしまい、リポジトリ外にファイルを作成してしまう可能性があった問題を修正
- ツールが失敗するかストリーミング途中で完了した際に `continue:false` フックの停止が無視されてしまう問題と、フック基盤のエラーがユーザーによる拒否として誤って報告される問題を修正
- print/SDK モードで実行中の Bash ツールへの SIGTERM がコマンドのプロセスツリーを孤立させてしまう問題を修正 — CLI はターンを中断し、プロセスツリーを kill してから終了コード 143 で終了するように
- Group Policy が PowerShell 5.1 をブロックしている Windows 環境で `/background` および `claude --bg` が "EUNKNOWN: unknown error, uv_spawn" で失敗する問題を修正 — デーモンは PowerShell 7 を優先的に使用するように
- パスの自動補完ポップアップが開いている間、ファイルパスを含むコマンドがシェルモード（`!`）で実行されない問題を修正
- 長い拒否理由が絵文字の途中で切り詰められた際、auto-mode の拒否通知が文字化けして表示される問題を修正
- 拡張キー報告に対応したターミナルのエージェントビュー ディスパッチ入力で Ctrl+J が改行を挿入しない問題を修正し、`?` ヘルプオーバーレイに改行ショートカットを表示するように
- `/ultrareview` が `#123`、`PR 123`、貼り付けられた PR URL などの PR 参照を拒否してしまう問題を修正 — エラーヒントには実際に入力したコマンド名が表示されるように
- `/ultrareview <branch>` が、ブランチがリモートに存在する場合に origin から fetch しない問題を修正 — タイプミス時には最も近いブランチ名を提案するように
- `/clear` 後の新しい会話で `/ultrareview` が課金確認をスキップしてしまう問題を修正
- Claude Desktop での `/ultrareview` の "not a git repository" エラーが、ターミナルコマンドの代わりにプロジェクトのリポジトリフォルダを提案するように修正
- リポジトリ設定で mTLS 証明書、追加の CA バンドル、OAuth スコープが構成されている場合に、ホスト管理セッションが起動時に失敗する問題を修正 — これらのトランスポート設定は警告付きで無視されるように
- セッション再開前に offset/limit 付きで読み込まれていたファイルを編集する際に、誤って "File has not been read yet" エラーが表示される問題を修正
- print/SDK モードで `--continue`/`--resume` を使ってセッションを再開した後、`ExitWorktree` が "no active EnterWorktree session" で失敗する問題を修正
- セッション途中から参加する Remote Control クライアントに対して、ワークフローエージェントグリッドが空のままになる問題を修正
- ストリーミングモードの制御リクエストが、ハンドラーの処理完了前に完了扱いとなり、セッション再起動時にリクエストが失われる可能性があった問題を修正
- `/fork` で作成されたバックグラウンドセッションが、状態書き込みの失敗後にライブ親の保護を失ってしまう問題を修正
- エージェントビューから停止済みのバックグラウンドセッションを再度開く操作がサイレントに失敗する問題を修正 — 現在はセッションを再開するか、再開できない理由を表示して強制再起動を可能にするように
- エージェントチーム: チーム初期化がセッション内で再実行された際、停止中のチームメイトがリーダーに重複したアイドル通知を送信してしまう問題を修正
- ファイルパスが長い場合に、プラン承認ダイアログのフッターの "ctrl+g to edit in <editor>" 表示が分断される問題を修正
- フルスクリーンモードで幅と高さの両方を変更するターミナルリサイズを行った後、ウェルカムバナーが古いパネル幅を保持し続ける問題を修正
- 狭いレイアウトで diff プレビューの行番号と +/- マーカーが失われる問題を修正
- 部分的なファイル読み込み後に @ メンションが何も添付しない問題、プラグインのアンインストールが誤ったマーケットプレイスを対象にしてしまう問題、終了コード 143 で誤って "Command timed out" と表示される問題を修正
- チャンク転送エンコーディングを受け付けない Azure Monitor などのエンドポイントで、OpenTelemetry の HTTP エクスポートが 411/400 で拒否される問題を修正
- SDK/ヘッドレスモードで `TRACEPARENT` が設定されている場合に、OTLP イベントログレコードから `trace_id`/`span_id` が欠落する問題を修正
- 画像を多く含む会話が誤って "Request too large" エラーで失敗する問題を修正し、実際の原因を説明するようエラーメッセージを改善
- API が過負荷状態のとき、web search と web fetch が "API Error" というテキストを検索結果やページ内容として返してしまう問題を修正
- 529 エラーやレート制限されたリクエストを上限付きバックオフでリトライすることで、web search と web fetch の信頼性を改善
- プロンプトキャッシュを改善: 会話途中の system ブロックが、LLM ゲートウェイやカスタム base URL（Bedrock, Vertex, 1P）の背後でも機能するように
- バックグラウンドエージェントのアタッチを改善: コールドアタッチ時、セッションの起動を待つ空白時間なしに、フォーマット済みトランスクリプトが即座に表示されるように
- エージェント間メッセージングのトークン使用量を削減: `SendMessage` の本文が、リプレイされる履歴やツール結果に重複して含まれないように
- セッションにタイトルがない場合、`/fork` がコピーをプロンプトにちなんだ名前にするように変更 — エージェントビューでその行を識別しやすく
- 単独の `/btw` が、直近のやり取りに対するサイドクエスチョンパネルを再度開くように変更 — 過去の回答を閲覧できるように
- 何も入力を必要とせずバックグラウンドエージェントが完了した際、フッターの `←` ヒントが一瞬 `N done` とパルス表示するように変更
- Task ツールの `mode` パラメータを非推奨化（現在は無視される）— サブエージェントはデフォルトで親セッションの権限モードを継承するように
- Enterprise の `forceLoginMethod` が、ターミナルだけでなく VS Code 拡張機能、SDK、`setup-token`、`install-github-app` のログインにも適用されるように変更
- セッションのトランスクリプトが、各アシスタントメッセージに reasoning effort レベルを記録するように変更
- ヘッドレス/SDK セッションが、ターン途中で `set_model` 制御リクエストを適用するように変更 — 次のモデルとのやり取りは、次のターンを待たずに新しいモデルを使用するように
- エージェントビュー / `claude agents --json`: サンドボックス、MCP 入力、managed-settings プロンプトを待っているセッションが "Working" ではなく "Needs input" と表示されるように変更
- 認証ステータスパネルのタイトルを "Cloud authentication" から "Authentication" に更新
- 以前のリリースノート（2.1.200）を訂正: 3.6 系までの tmux は同期出力（synchronized output）をサポートしておらず、対応している新しい tmux は自動的に検出される
