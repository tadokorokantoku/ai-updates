- `/powerup` を追加 — アニメーションデモ付きで Claude Code の機能を教えるインタラクティブレッスン
- `git pull` が失敗したときに既存のマーケットプレイスキャッシュを保持する `CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE` 環境変数を追加。オフライン環境で便利
- 保護ディレクトリ（acceptEdits モード）に `.husky` を追加
- 使用制限到達後にレート制限オプションダイアログが繰り返し自動オープンし、最終的にセッションがクラッシュする無限ループを修正
- 遅延ロードされたツール、MCP サーバー、カスタムエージェントを持つユーザーで `--resume` が最初のリクエストで完全なプロンプトキャッシュミスを引き起こす問題を修正（v2.1.69 からのリグレッション）
- PostToolUse の format-on-save フックが連続した編集間でファイルを書き換えるときに `Edit`/`Write` が "File content has changed" で失敗する問題を修正
- stdout に JSON を出力して exit code 2 で終了する `PreToolUse` フックがツール呼び出しを正しくブロックしない問題を修正
- ツール呼び出し中に CLAUDE.md ファイルが自動ロードされると、フルスクリーン scrollback で折りたたまれた search/read 要約バッジが複数回表示される問題を修正
- auto モードが明示的なユーザー境界（「push しない」、「X を X までするな」）を尊重しない問題を修正。アクションがそれ以外の場合許可されるときでも
- 不正なツール入力が権限ダイアログに到達したときの UI クラッシュを修正
- `/model`、`/config`、その他の選択画面でスクロール時にヘッダーが消える問題を修正
- PowerShell ツールの権限チェックを強化: 末尾 `&` バックグラウンドジョブバイパス、`-ErrorAction Break` デバッガハング、アーカイブ抽出 TOCTOU、parse-fail フォールバック deny ルール劣化を修正
- ターンごとの MCP ツールスキーマのキャッシュキールックアップで JSON.stringify を排除してパフォーマンスを改善
- SSE transport が大きなストリーミングフレームを線形時間で処理するように改善（以前は二次式）
- 長い会話を持つ SDK セッションがトランスクリプト書き込みで二次的に遅くならないように改善
- 多くのプロジェクトを持つユーザーのロード時間を改善するため、プロジェクトセッションを並列でロードするよう `/resume` の all-projects ビューを改善
- `claude -p` または SDK invocation で作成されたセッションを表示しないよう `--resume` ピッカーを変更
- 自動許可から `Get-DnsClientCache` と `ipconfig /displaydns` を削除（DNS キャッシュのプライバシー）
