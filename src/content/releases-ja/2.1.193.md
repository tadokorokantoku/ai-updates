- すべての Bash/PowerShell コマンドを任意コード実行パターンのみでなく auto-mode クラシファイアにルーティングする `autoMode.classifyAllShell` 設定を追加
- auto-mode 拒否理由をトランスクリプト、拒否トースト、`/permissions` の最近の拒否に追加
- モデルのレスポンステキストを含む `claude_code.assistant_response` OpenTelemetry ログイベントを追加。`OTEL_LOG_ASSISTANT_RESPONSES=1` を設定しない限りリダクション済み。未設定の場合は `OTEL_LOG_USER_PROMPTS` に従うため、プロンプト内容をログに記録しているデプロイメントはアップグレード時にレスポンス内容も受信するようになる — プロンプトのみに留める場合は `OTEL_LOG_ASSISTANT_RESPONSES=0` を設定する。
- bash モード（`!`）にライブファイルパス補完を追加
- MCP サーバーが認証を必要とする場合に `/mcp` を示すスタートアップ通知を追加
- アイドル状態のバックグラウンドシェルコマンドに対するメモリ圧迫時の自動リープを追加（`CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1` で無効化可能）
- `/login` 直後に `/model` などのクライアントデータゲートの UI が古い/空の状態を表示する問題を修正
- バックグラウンド化（←←）で実行中のタスクがすべて新しいセッションに引き継がれるにもかかわらず「N個のバックグラウンドタスクが中断される」と誤キャンセルされる問題を修正
- 固定されたバックグラウンドエージェントが自動更新のたびに「前回の続きから再開しますか」と再プロンプトされる問題を修正
- メインターンのバックグラウンド化でメイン会話を再実行するゴースト「general-purpose (resumed)」サブエージェントが生成される問題を修正
- サブエージェント表示中にエージェントパネルが兄弟エージェントを非表示にする問題を修正
- バックグラウンドエージェントの改善: 起動結果が Claude に「レスポンスを終了する」よう指示しなくなり、エージェント実行中も他のタスクに取り組み続けられるようになった
- MCP `headersHelper` 認証の改善: ツール呼び出しが 401/403 を返したときにヘルパーが自動的に再実行して再接続するようになった
- プラグイン自動リネームの改善: マーケットプレイスの `renames` マップが自動的に追従され、設定が新しい名前に更新されるようになった
- ディレクトリがすでに作業ディレクトリである場合の `/add-dir` メッセージを改善
