- 外部ビルド用の `claude remote-control` サブコマンドを追加。すべてのユーザー向けにローカル環境の提供を可能に
- プラグインマーケットプレイスのデフォルト git タイムアウトを 30s から 120s に更新。`CLAUDE_CODE_PLUGIN_GIT_TIMEOUT_MS` で設定可能に
- npm ソースからプラグインをインストールする際のカスタム npm レジストリと特定バージョン固定のサポートを追加
- シェルスナップショットが利用可能な場合、BashTool はデフォルトでログインシェル（`-l` フラグ）をスキップし、コマンド実行パフォーマンスを改善。以前は `CLAUDE_BASH_NO_LOGIN=true` の設定が必要だった
- インタラクティブモードでワークスペースの信頼受諾なしに `statusLine` と `fileSuggestion` フックコマンドが実行可能だったセキュリティ問題を修正
- 50K 文字より大きいツール結果がディスクに永続化されるように（以前は 100K）。これによりコンテキストウィンドウ使用量が減少し、会話の寿命が改善
- 重複した `control_response` メッセージ（例: WebSocket 再接続から）が重複したアシスタントメッセージを会話に push することで API 400 エラーを引き起こすバグを修正
- SDK 呼び出し元がアカウント情報を同期的に提供可能な `CLAUDE_CODE_ACCOUNT_UUID`、`CLAUDE_CODE_USER_EMAIL`、`CLAUDE_CODE_ORGANIZATION_UUID` 環境変数を追加。早期のテレメトリイベントがアカウントメタデータを欠くレース条件を解消
- プラグインの SKILL.md description が YAML 配列または他の非文字列型の場合、スラッシュコマンドのオートコンプリートがクラッシュする問題を修正
- `/model` ピッカーが pin されたモデルバージョンに対して生のモデル ID ではなく人間可読なラベル（例: "Sonnet 4.5"）を表示するように。新しいバージョンが利用可能な場合はアップグレードヒント付き
- macOS plist または Windows レジストリ経由で管理対象設定を設定可能に。詳細は https://code.claude.com/docs/en/settings#settings-files
