- 折りたたまれたツールサマリー行にライブの経過時間カウンターを追加し、長時間実行中のツール呼び出しがフリーズしているように見えず、時間が進んでいることが視覚的にわかるように
- `Write(path)`、`NotebookEdit(path)`、`Glob(path)` の permission ルールに対する起動時の警告を追加 — 代わりに `Edit(path)` または `Read(path)` を使用してください
- `isolation: 'worktree'` のサブエージェントが、自身の独立した worktree ではなくメインのリポジトリチェックアウトに対して git を変更するコマンドを実行できてしまう問題を修正
- webhook のペイロードや転送された PR コメントなど、人間由来ではない入力に対して `ultracode` キーワードのオプトインが発火してしまう問題を修正
- UI コンポーネントがスタイル付きテキスト要素以外のコンテンツを返した際に、レンダリングされたテキストの断片がクラッシュのテレメトリに漏れる問題を修正
- Claude Code から開いた外部エディタにペーストマーカーが漏れ、ペーストしたテキストの周りに È/É のような不要な文字が現れる問題を修正
- セッション遷移中に `claude attach` が "job not found" や "agent is still starting" エラーで失敗することがある問題を修正 — attach はデーモンが安定するのを待つようになり、attach 中の遅いターミナルリサイズは完了後に一度だけ適用されるように
- ツールの結果レンダラーが UI 要素ではなく数値の bigint やプレーンテキストを返した際にセッションがクラッシュする問題を修正
- フックコールバックのタイムアウトがモデルに対してユーザーによる拒否として誤って報告され、無人セッションが停止して待機してしまう問題を修正
- `cd` がバックグラウンドに移動された後もそのコマンドが反映されたと Claude が想定してしまう問題を修正 — ツールの結果には作業ディレクトリが変更されていないことが明示されるように
- MCP サーバーがセッション中に再同期される際に、プラグイン提供の MCP サーバーが停止されてしまう問題を修正
- 編集を伴わないプラン承認が「(edited by user)」とラベル付けされ、プランファイルが古いスナップショットで上書きされる問題を修正
- Bedrock、Vertex、Foundry ではオプトインが不要になった auto mode について、`/doctor` がその提案をスキップしてしまう問題を修正
- 結果の末尾を超えてページングした際に Grep のコンテンツモードが「No matches found」と誤って表示する問題を修正
- スキルやコマンド内で未対応の `$1`/`$2` の位置引数プレースホルダーが黙って取り除かれてしまう問題を修正 — そのまま保持されるように
- プラグインキャッシュへの書き込みが失敗時に一時ファイルを残し、Windows やネットワークファイルシステムでロックされたファイルのリネームに失敗する問題を修正
- クライアントがバックグラウンドサービスへの接続をリセットした際にバックグラウンドワーカーがクラッシュループする問題を修正
- `claude agents --effort ultracode` が送出先のセッションに反映されず、値が黙って破棄されてしまう問題を修正
- ← を押してエージェントビューを開くと、セッションに戻った際にタスクトラッカーが失われる問題を修正
- エージェントダッシュボードが、セッション削除後も破棄された返信ドラフトからペーストされた画像を保持し続ける問題を修正
- kill されたバックグラウンドセッションが `git worktree lock` を永続的に残してしまう問題を修正 — 定期的なスイープが、所有プロセスが存在しないロックを解放するように
- `initialize` コントロールリクエスト経由で登録された SDK の MCP サーバーが、次のターンまで接続を開始しない問題を修正
- `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1` の状態でセッションからエージェントビューに戻ると、ゴーストフレームが重なって残る問題を修正
- 遅れて現れる `.claude/*` のシンボリックリンクが、サンドボックスの書き込み拒否リストに反映されない問題を修正
- サブエージェントが読み込んだコンテンツ経由の間接的なプロンプトインジェクションに対する Agent tool の耐性を強化
- Bash/PowerShell ツールがタイムアウトしてバックグラウンドに自動移行した際のメッセージを改善し、モデルがハングと明示的なバックグラウンド要求を区別できるように
- auto mode を改善: 権限分類器は外部セッションではデフォルトで Sonnet 5 を使用し、セッションの最初のリクエストで検証されてそのセッション中は固定されるように
- バンドルされている dataviz スキルのチャート配色検証を、知覚的な OKLab 色差と再調整された色覚多様性のしきい値により改善
- MEMORY.md のインデックスが読み取り上限を超える書き込みが行われた場合、サイレントな切り捨てではなく明示的なエラーを出すように
- スクリーンリーダーモードで、Shift+Tab によるモード切り替え時に permission モードの変更を音声で読み上げるように
- エージェントのフッターのヒントに、入力待ちのバックグラウンドエージェントの数を表示するようになり、件数が変化した際は一瞬色が強調されるように
- エージェントビュー: ← を押した際の選択元セッションは、マウスホバーや矢印キーで選択が移動した後も視覚的にマークされたままに
- Fable アドバイザーの失敗を引き起こしているサーバー側の問題が修正されるまでの間、アドバイザー選択画面で Fable が一時的に利用不可と表示されるように
