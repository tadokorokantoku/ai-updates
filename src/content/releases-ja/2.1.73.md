- モデルピッカーエントリをカスタムプロバイダーモデル ID にマップする `modelOverrides` 設定を追加（例: Bedrock 推論プロファイル ARN）
- SSL 証明書エラーで OAuth ログインまたは接続チェックが失敗したときの実行可能なガイダンスを追加（企業プロキシ、`NODE_EXTRA_CA_CERTS`）
- 複雑な bash コマンドの権限プロンプトでトリガーされるフリーズと 100% CPU ループを修正
- 多くのスキルファイルが一度に変更される（例: 大きな `.claude/skills/` ディレクトリを持つリポジトリの `git pull`）と Claude Code がフリーズするデッドロックを修正
- 同じプロジェクトディレクトリで複数の Claude Code セッションを実行する際に Bash ツールの出力が失われる問題を修正
- `model: opus`/`sonnet`/`haiku` のサブエージェントが Bedrock、Vertex、Microsoft Foundry で古いモデルバージョンに無言でダウングレードされる問題を修正
- サブエージェントによって spawn されたバックグラウンド bash プロセスが、エージェント終了時にクリーンアップされない問題を修正
- `/resume` がピッカーに現在のセッションを表示する問題を修正
- 拡張機能を自動インストールする際に `/ide` が `onInstall is not defined` でクラッシュする問題を修正
- `/loop` が Bedrock/Vertex/Foundry で利用できず、テレメトリが無効になったときに利用できない問題を修正
- `--resume` または `--continue` でセッションを再開するときに SessionStart フックが 2 回発火する問題を修正
- JSON 出力フックがターンごとにモデルのコンテキストに no-op の system-reminder メッセージを注入する問題を修正
- 遅い接続が新しい録音と重なる voice モードのセッション破損を修正
- Linux サンドボックスがネイティブビルドで「ripgrep (rg) not found」で起動に失敗する問題を修正
- Linux ネイティブモジュールが Amazon Linux 2 やその他の glibc 2.26 システムでロードされない問題を修正
- Remote Control 経由で画像を受信するときの「media_type: Field required」API エラーを修正
- Desktop フォルダがすでに存在するときの `EEXIST` エラーで Windows で `/heapdump` が失敗する問題を修正
- Claude を中断した後の Up arrow を改善 — 中断されたプロンプトを復元し、会話を 1 ステップで巻き戻し
- 起動時の IDE 検出速度を改善
- macOS でのクリップボード画像貼り付けパフォーマンスを改善
- Claude が応答中にも `/effort` が動作するよう改善。`/model` の動作と一致
- 素早い push-to-talk 再押下中の一時的な接続失敗を自動リトライするよう voice モードを改善
- Remote Control spawn モード選択プロンプトをより良いコンテキストで改善
- Bedrock、Vertex、Microsoft Foundry のデフォルト Opus モデルを Opus 4.6 に変更（以前は Opus 4.1）
- `/output-style` コマンドを非推奨化 — 代わりに `/config` を使用。プロンプトキャッシュを改善するため、出力スタイルはセッション開始時に固定
- VSCode: Claude 4.5 モデルで Bedrock/Vertex やプロキシ経由のユーザーの HTTP 400 エラーを修正
