- セッションのトランスクリプトファイルの改ざんをブロックするオートモードのルールを追加
- `--json-schema` が不正なスキーマを指定した際に非構造化出力を黙って生成してしまう問題と、`format` キーワードを使うスキーマが拒否される問題を修正
- Claude が作業中に送られたメッセージが、ターンが `--max-turns` の上限で終了した際に黙って失われる問題を修正
- Windows でワークツリー内に NTFS ジャンクションやディレクトリシンボリックリンクが存在する場合、ワークツリー削除時にワークツリー外のファイルまで削除されてしまう問題を修正
- `SendMessage` で再開したバックグラウンドエージェントが、エージェント一覧で「failed」または「completed」のまま表示され続ける問題を修正
- エージェントのターンに読み取り可能なテキストが含まれない場合に、バックグラウンドジョブがエージェント一覧で「needs input」から「working」に戻ってしまう問題を修正
- バックグラウンドエージェントがアップグレードによる再起動の途中である場合に、復帰を待たずに `claude attach` がエラーになる問題を修正
- 30K のインライン上限を超える出力を持つ Bash 呼び出しで作成された PR が、セッションと PR の紐付けから漏れる問題を修正
- サーバー名にサポートされていない文字が含まれる場合に `claude mcp add-from-claude-desktop` が停止してしまう問題を修正。不正な名前は報告され、残りのサーバーはインポートされるようになった
- 初期化に失敗するプラグインの LSP サーバーが、同じファイル拡張子を扱う別プラグインの正常な LSP サーバーの動作を妨げる問題を修正
- コマンド実行中に Claude の起動元ディレクトリが削除・ロック・アンマウントされた場合の Windows でのクラッシュを修正
- ディレクトリスキャンの実行中にファイルウォッチャーが閉じられた際のクラッシュを修正
- プロジェクトの verify スキルが、ドキュメント化されたコマンドが変更された場合のみでなく毎セッション書き換えられてしまう問題を修正
- ジョブ一覧がわずかに画面をあふれる際に、エージェントビューが1行分高くレンダリングされヘッダーが欠けてしまう問題を修正
- Web・モバイルの Remote Control パネルでバックグラウンドタスクが古い「Running」ステータスのまま表示される問題を修正。メンバーシップが変化するたびにタスクの全状態を送るように変更
- オートモードが、コンテキストから解決できない変数に対して `rm -rf` を実行する前に確認を求めるよう改善
- 自動更新のバイナリダウンロードがメモリにバッファせずディスクへストリーミングされるようになり、アップデーターのピークメモリ使用量を約400MB削減
- バックグラウンドタスクの通知が、人間による入力が発生していないことを明示するようになり、トランスクリプト内に捏造された承認が実行されてしまうことを防止
- エージェントビューの改善: 既存の PR を編集・マージ・コメント・push したセッションが `claude agents` にその PR へのリンクを表示するようになった
- エージェントビューの改善: 各行が生のツール呼び出しテキストの代わりに色付きの状態語と分類器が生成した見出しを表示するようになり、ピーク表示ではブロックされたセッションの正確な依頼内容を含む完全なステータスが開くようになった
- `/doctor` がセットアップの問題を診断・修正できる本格的なチェックアップ機能になった。`/checkup` はそのエイリアス
- Claude Desktop のペイン名変更に先立ち、「Claude Browser」MCP サーバー名を（「Claude Preview」とともに）予約。ユーザー設定の MCP サーバーはこれらの名前で登録できなくなった
- CLI 2.1.203 以降で Cowork の VM モードのローカルエージェントセッションが「Not logged in · Please run /login」で起動に失敗する問題を修正
