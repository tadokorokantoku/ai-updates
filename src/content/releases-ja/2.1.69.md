---

- Claude API と Anthropic SDK を使ったアプリケーション構築のための `/claude-api` スキルを追加
- 空の bash プロンプト（`!`）で bash モードを終了する Ctrl+U を追加。`escape` と `backspace` と同じ動作
- Claude のインタビュー質問でオプションを選択するためのテンキーのサポートを追加（以前は QWERTY の上の数字行のみ機能）
- `/remote-control` と `claude remote-control` にオプションの name 引数を追加（`/remote-control My Project` または `--name "My Project"`）。claude.ai/code で可視のカスタムセッションタイトルを設定
- 10 の新しい言語（合計 20）で Voice STT サポートを追加 — ロシア語、ポーランド語、トルコ語、オランダ語、ウクライナ語、ギリシャ語、チェコ語、デンマーク語、スウェーデン語、ノルウェー語
- ロゴとスピナーに effort レベル表示（例: "with low effort"）を追加。どの effort 設定がアクティブか見やすく
- `claude --agent` 使用時にターミナルタイトルにエージェント名表示を追加
- `sandbox.enableWeakerNetworkIsolation` 設定（macOS のみ）を追加。`gh`、`gcloud`、`terraform` などの Go プログラムが `httpProxyPort` でカスタム MITM プロキシを使う際に TLS 証明書を検証できるように
- `includeGitInstructions` 設定（および `CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS` 環境変数）を追加。Claude のシステムプロンプトから組み込みのコミットおよび PR ワークフロー指示を削除
- 再起動せずに保留中のプラグイン変更をアクティベートする `/reload-plugins` コマンドを追加
- macOS と Windows で Claude Code Desktop を提案する一度限りの起動プロンプトを追加（最大 3 回表示、却下可能）
- スキルが SKILL.md コンテンツ内で自身のディレクトリを参照するための `${CLAUDE_SKILL_DIR}` 変数を追加
- CLAUDE.md または `.claude/rules/*.md` ファイルがコンテキストにロードされたときに発火する `InstructionsLoaded` フックイベントを追加
- フックイベントに `agent_id`（サブエージェント用）と `agent_type`（サブエージェントと `--agent` 用）を追加
- `--worktree` セッションで実行中にステータスラインフックコマンドに name、path、branch、元のリポジトリディレクトリを持つ `worktree` フィールドを追加
- インストール前に表示されるプラグイン信頼警告に組織固有のコンテキストを追加するための `pluginTrustMessage` を管理対象設定に追加
- エンタープライズだけでなく Team プランの OAuth ユーザー向けにポリシー制限の取得（例: リモートコントロール制限）を追加
- `hostPattern` 制限と並行してファイル/ディレクトリマーケットプレイスソースを regex マッチするための `pathPattern` を `strictKnownMarketplaces` に追加
- git リポジトリ内のサブディレクトリを指す `git-subdir` プラグインソースタイプを追加
- 標準の discovery が失敗した場合に MCP サーバーがカスタム OAuth メタデータ discovery URL を指定できる `oauth.authServerMetadataUrl` 設定オプションを追加
- ネストされたスキル discovery が `node_modules` のような gitignore されたディレクトリからスキルをロードできるセキュリティ問題を修正
- 初回実行時に信頼ダイアログがすべての `.mcp.json` サーバーを無言で有効化する問題を修正。期待通りサーバーごとの承認ダイアログが表示されるように
- npm インストール時に `claude remote-control` が "bad option: --sdk-url" で即座にクラッシュする問題を修正 (anthropics/claude-code#28334)
- `--model claude-opus-4-0` と `--model claude-opus-4-1` が最新ではなく非推奨の Opus バージョンに解決される問題を修正
- 複数の OAuth MCP サーバーを使用する際の macOS keychain 破損を修正。大きな OAuth メタデータ blob が `security -i` stdin バッファをオーバーフローし、古い認証情報を無言で残して繰り返し `/login` プロンプトを引き起こす可能性があった
- トークン更新中にプロファイルエンドポイントが一時的に失敗すると `.credentials.json` が `subscriptionType` を失う（"Claude Pro"/"Claude Max" の代わりに "Claude API" を表示）問題を修正 (anthropics/claude-code#30185)
- Linux 上でサンドボックス化された Bash コマンドの後、作業ディレクトリに未追跡ファイルとしてゴースト dotfile（`.bashrc`、`HEAD` など）が表示される問題を修正
- Ghostty over SSH で Shift+Enter が改行を挿入する代わりに `[27;2;13~` を印字する問題を修正
- Claude が作業中にメッセージを送信する際、スタッシュ（Ctrl+S）がクリアされる問題を修正
- 多くのファイル編集を持つ長いセッションで ctrl+o（トランスクリプトトグル）が数秒フリーズする問題を修正
- プランモードのフィードバック入力が複数行テキスト入力をサポートしない問題を修正（backslash+Enter と Shift+Enter で改行を挿入可能に）
- 入力ボックス上部の空行にカーソルが下に移動しない問題を修正
- トランスクリプトファイルにタイムスタンプが欠落または不正な形式のエントリがある場合の `/stats` クラッシュを修正
- 長いセッションでのストリーミングエラー後の短いハングを修正（1 行をドロップするためにトランスクリプト全体が書き直されていたのを、インプレースで切り詰めるように）
- `--setting-sources user` が動的に発見されたプロジェクトスキルをブロックしない問題を修正
- メインリポジトリ内にネストされた worktree から実行した際の CLAUDE.md、スラッシュコマンド、エージェント、ルールの重複を修正（例: `claude -w`）
- プラグイン Stop/SessionEnd などのフックが任意の `/plugin` 操作後に発火しない問題を修正
- 2 つのプラグインが同じ `${CLAUDE_PLUGIN_ROOT}/...` コマンドテンプレートを使用すると、プラグインフックが無言で落ちる問題を修正
- 長時間実行 SDK/CCR セッションで会話メッセージが不要に保持されるメモリリークを修正
- ツールバッチ中に中断されたセッションを再開する際、フォークされたエージェント（autocompact、summarization）での API 400 エラーを修正
- 孤立したツール結果で始まる会話を再開する際の "unexpected tool_use_id found in tool_result blocks" エラーを修正
- チームメイトが Agent ツールの `name` パラメータ経由で誤ってネストされたチームメイトを spawn する問題を修正
- 会話のコンパクション中に `CLAUDE_CODE_MAX_OUTPUT_TOKENS` が無視される問題を修正
- SDK コンシューマー（Claude Code Remote web UI、VSCode 拡張機能）で `/compact` サマリーがユーザーバブルとしてレンダリングされる問題を修正
- 失敗した voice activation 後に voice space bar がスタックする問題を修正（モジュールロードレース、cold GrowthBook）
- Windows での worktree ファイルコピーを修正
- Windows でのグローバル `.claude` フォルダ検出を修正
- シンボリックリンクされた親ディレクトリ経由で新しいファイルを書き込むと、`acceptEdits` モードで作業ディレクトリをエスケープできるシンボリックリンクバイパスを修正
- 管理対象設定で `allowManagedDomainsOnly` が有効な場合、サンドボックスが許可されていないドメインの承認をユーザーに求める問題を修正。バイパスなしで自動的にブロック
- インタラクティブツール（例: `AskUserQuestion`）がスキルの allowed-tools にリストされている場合、権限プロンプトをバイパスして空の回答で実行される問題を修正
- 作業ツリーに大きな未追跡バイナリファイルがある状態でのコミット時のマルチ GB メモリスパイクを修正
- Remote Control セッションでローカルスラッシュコマンド（`/voice`、`/cost`）を実行する際の Android アプリのクラッシュを修正
- 長いセッションで古いメッセージ配列バージョンが React Compiler の `memoCache` に蓄積されるメモリリークを修正
- 長いセッションで REPL render スコープが蓄積するメモリリーク（1000 ターンで約 35MB）を修正
- インプロセスチームメイトで親の完全な会話履歴がチームメイトのライフタイムの間 pin され、`/clear` や autocompact 後の GC を妨げるメモリ保持を修正
- 長いセッション中にインタラクティブモードでフックイベントが無制限に蓄積する可能性があるメモリリークを修正
- `--mcp-config` が破損したファイルを指す場合のハングを修正
- 多くのスキル/プラグインがインストールされている場合の起動の遅さを修正
- `cd <outside-dir> && <cmd>` の権限プロンプトを修正。"Yes, allow reading from <dir>/" のみを表示する代わりに chained コマンドを明示
- 条件付き `.claude/rules/*.md` ファイル（`paths:` frontmatter 付き）とネストされた CLAUDE.md ファイルが print モード（`claude -p`）でロードされない問題を修正
- `/clear` がすべてのセッションキャッシュを完全にクリアせず、長いセッションでメモリ保持を引き起こす問題を修正
- scrollback 境界でのアニメーション要素によるターミナルのちらつきを修正
- OAuth 付きの MCP サーバーを使用時の macOS UI フレームドロップを修正（2.1.x からのリグレッション）
- 同期デバッグログフラッシュによる入力中の時折のフレームストールを修正
- `TeammateIdle` と `TaskCompleted` フックを修正。`{"continue": false, "stopReason": "..."}` によるチームメイト停止をサポート。`Stop` フックの動作と一致
- `WorktreeCreate` と `WorktreeRemove` プラグインフックが無言で無視される問題を修正
- コロンを含むスキル記述（例: "Triggers include: X, Y, Z"）が SKILL.md frontmatter からロードされない問題を修正
- `description:` frontmatter フィールドのないプロジェクトスキルが Claude の利用可能なスキルリストに表示されない問題を修正
- `/context` がサーバーからのすべての MCP ツールに対して同じトークンカウントを表示する問題を修正
- モデルが Git Bash で CMD スタイルの `2>nul` リダイレクションを使うと Windows でリテラル `nul` ファイルが作成される問題を修正
- 展開されたサブエージェントトランスクリプトビュー（Ctrl+O）で各ツール呼び出しの下に余分な空行が表示される問題を修正
- `/config` 検索ボックスにフォーカスがあるが空のとき、Tab/矢印キーが Settings タブを循環しない問題を修正
- サービスキー OAuth セッション（CCR コンテナ）が profile スコープのエンドポイントからの 403 で `[ERROR]` ログをスパムする問題を修正
- "Remote Control active" ステータスインジケータの色が一貫しない問題を修正
- 入力中に dictating する際、Voice waveform カーソルが最初のサフィックス文字を覆う問題を修正
- Voice 入力がウォームアップ中に 5 スペースすべてを表示する問題を修正（「keep holding…」ヒントと整合するように ~2 スペースで cap）
- 50ms アニメーションループを周囲のシェルから分離してスピナーパフォーマンスを改善。ターン中のレンダリングと CPU オーバーヘッドを削減
- React Compiler を使用してネイティブバイナリの UI レンダリングパフォーマンスを改善
- 起動パスで git サブプロセスを削減して `--worktree` の起動を改善
- 管理対象設定が解決されたときの冗長な設定ファイル再読み込みを削減して macOS 起動を改善
- 不要な keychain ルックアップをスキップして Claude.ai エンタープライズ/チームユーザーの macOS 起動を改善
- ローカル接続と並行して claude.ai config fetch をパイプライン化し、順次バッチングの代わりに同時実行プールを使用して MCP `-p` 起動を改善
- 再レンダリングのスタッターを引き起こしていた知覚できないウォームアップパルスアニメーションを削除して voice 起動を改善
- MCP バイナリコンテンツ処理を改善。PDF、Office ドキュメント、オーディオを返すツールは、生の base64 を会話コンテキストにダンプする代わりに、正しいファイル拡張子でデコードされたバイトをディスクに保存。WebFetch もバイナリレスポンスを要約とともに保存
- メッセージ更新間で `onSubmit` を安定化させて長いセッションでのメモリ使用量を改善
- ファイル全体を読み込まないように LSP ツールレンダリングとメモリコンテキスト構築を改善
- サイズ/バイナリチェックの前に大きなファイルをメモリに読み込むのを避けるようにセッションアップロードとメモリ同期を改善
- 存在チェックのためにファイル内容を読まないようにしてファイル操作パフォーマンスを改善（6 か所）
- `--append-system-prompt-file` と `--system-prompt-file` がインタラクティブモードで動作することを明確にするドキュメントを改善（以前のドキュメントは print モードのみと記載）
- Yoga WASM の事前ロードを遅延させてベースラインメモリを約 16MB 削減
- stream-json 出力を使う SDK と CCR セッションのメモリフットプリントを削減
- 大きなセッション（コンパクション済み履歴を含む）を再開する際のメモリ使用量を削減
- より簡潔なサブエージェント最終レポートでマルチエージェントタスクのトークン使用量を削減
- Pro/Max/Team Premium の Sonnet 4.5 ユーザーを Sonnet 4.6 に自動移行するように変更
- `/resume` ピッカーを最初のプロンプトではなく最新のプロンプトを表示するように変更。一部のタイトルが `(session)` として表示される問題も解決
- claude.ai MCP コネクタの失敗を、ツールリストから無言で消える代わりに通知を表示するように変更
- Haiku を呼び出す代わりに決定論的に生成するようにコマンド例候補を変更
- コンパクション後の再開で continuing 前のプリアンブルまとめを生成しないように変更
- [SDK] タスク作成で `activeForm` フィールドが不要になるように変更。スピナーはタスクサブジェクトにフォールバック
- [VSCode] コンパクション表示を折りたたみ可能な「Compacted chat」カードとして追加し、その中に要約を表示
- [VSCode] 権限モードピッカーが `permissions.disableBypassPermissionsMode` を尊重（管理対象/ポリシー設定を含む有効な Claude Code 設定）。`disable` に設定すると、ピッカーからバイパス権限モードが非表示に
- [VSCode] チャットパネルで RTL テキスト（アラビア語、ヘブライ語、ペルシア語）が逆向きにレンダリングされる問題を修正（v2.1.63 のリグレッション）
