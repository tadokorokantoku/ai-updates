- Claude Opus 4.6 のデフォルト最大出力トークン制限を 64k トークンに引き上げ。Opus 4.6 と Sonnet 4.6 モデルの上限を 128k トークンに
- `denyRead` 領域内での読み取りアクセスを再度許可する `allowRead` サンドボックスファイルシステム設定を追加
- `/copy` がオプションのインデックスを受け入れるように: `/copy N` で N 番目に最新のアシスタント応答をコピー
- 複合 bash コマンド（例: `cd src && npm test`）に対する「Always Allow」がサブコマンドごとではなく文字列全体の単一ルールを保存し、dead ルールと繰り返される権限プロンプトにつながる問題を修正
- スラッシュコマンドオーバーレイが繰り返し開閉することで自動アップデーターが重複するバイナリダウンロードを開始し、数十 GB のメモリを蓄積する問題を修正
- メモリ抽出書き込みとメインのトランスクリプト間のレースによって `--resume` が最近の会話履歴を無言で切り詰める問題を修正
- `"allow"` を返す PreToolUse フックがエンタープライズ管理対象設定を含む `deny` 権限ルールをバイパスする問題を修正
- Write ツールが CRLF ファイルを上書きするか CRLF ディレクトリ内にファイルを作成する際に無言で行末を変換する問題を修正
- コンパクション後も進捗メッセージが生き残る長時間実行セッションのメモリ増加を修正
- API が非ストリーミングモードにフォールバックした際にコストとトークン使用量が追跡されない問題を修正
- `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS` が beta ツールスキーマフィールドを削除せず、プロキシゲートウェイがリクエストを拒否する問題を修正
- システム temp ディレクトリパスにスペースが含まれる場合、Bash ツールが成功したコマンドに対してエラーを報告する問題を修正
- 貼り付け直後にタイピングするとペーストが失われる問題を修正
- `/feedback` テキスト入力での Ctrl+D が前に削除するのではなく、二度目でセッションを終了するよう修正
- 0 バイトの画像ファイルをプロンプトにドラッグしたときの API エラーを修正
- Claude Desktop セッションがターミナル CLI の設定済み API キーを OAuth の代わりに誤って使用する問題を修正
- 同じモノレポコミットの異なるサブディレクトリにある `git-subdir` プラグインがプラグインキャッシュで衝突する問題を修正
- ターミナル UI で順序付きリストの番号がレンダリングされない問題を修正
- 古い worktree クリーンアップが前回のクラッシュから再開されたばかりのエージェント worktree を削除するレース条件を修正
- エージェントが実行中に `/mcp` や類似のダイアログを開いた際の入力デッドロックを修正
- vim NORMAL モードで Backspace と Delete キーが動作しない問題を修正
- vim モードがオン/オフに切り替わったときにステータスラインが更新されない問題を修正
- VS Code、Cursor、その他の xterm.js ベースのターミナルで Cmd+click 時にハイパーリンクが 2 回開く問題を修正
- デフォルト設定の tmux 内部で背景色がターミナルデフォルトとしてレンダリングされる問題を修正
- SSH 経由の tmux 内部でテキスト選択時の iTerm2 セッションクラッシュを修正
- tmux セッションでクリップボードコピーが無言で失敗する問題を修正。コピートーストは `⌘V` または tmux `prefix+]` で貼り付けるかを示すように
- リストをナビゲート中に設定、権限、サンドボックスダイアログで `←`/`→` が誤ってタブを切り替える問題を修正
- Claude Code が tmux または screen 内部で起動されたときに IDE 統合が自動接続しない問題を修正
- 右端でクリップされた際に CJK 文字が隣接する UI 要素に視覚的にあふれる問題を修正
- リーダー終了時にチームメイトペインが閉じない問題を修正
- iTerm2 auto モードがネイティブ分割ペインチームメイトのために iTerm2 を検出しない問題を修正
- macOS での起動を約 60ms 高速化 — モジュールロードと並列に keychain 認証情報を読み取り
- フォーク重視と非常に大きなセッションでの `--resume` を最大 45% 高速化、ピークメモリを約 100-150MB 削減
- 実行中の非ストリーミング API リクエストを中断する Esc を改善
- `claude plugin validate` を改善。スキル、エージェント、コマンドの frontmatter に加え `hooks/hooks.json` をチェックし、YAML 解析エラーとスキーマ違反をキャッチ
- バックグラウンド bash タスクの出力が 5GB を超えると強制終了されるように。暴走プロセスによるディスク溢れを防止
- プラン受諾時にプラン内容からセッション名が自動付与されるように
- `CLAUDE_CODE_PLUGIN_SEED_DIR` と正しく合成するようヘッドレスモードのプラグインインストールを改善
- `apiKeyHelper` が 10 秒以上かかったときに通知を表示。メインループのブロックを防止
- Agent ツールが `resume` パラメータを受け入れなくなった — 以前に spawn されたエージェントを続けるには `SendMessage({to: agentId})` を使用
- `SendMessage` が停止したエージェントをバックグラウンドで自動再開するように（エラーを返す代わりに）
- `/fork` を `/branch` にリネーム（`/fork` はエイリアスとして引き続き動作）
- [VSCode] プランプレビュータブのタイトルを "Claude's Plan" ではなくプランの見出しを使うよう改善
- [VSCode] macOS で option+click がネイティブ選択をトリガーしない場合、フッターが `macOptionClickForcesSelection` 設定を指し示すように
