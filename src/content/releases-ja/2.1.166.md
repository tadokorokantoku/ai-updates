- `fallbackModel` 設定を追加 — プライマリモデルが過負荷または利用不可の場合に順番に試すフォールバックモデルを最大3つ設定可能に；`--fallback-model` がインタラクティブセッションにも適用されるようになった
- deny ルールのツール名位置にグロブパターンのサポートを追加（`"*"` ですべてのツールを拒否）；allow ルールは非 MCP グロブを拒否し、deny ルール内の不明なツール名は起動時に警告を表示する
- クロスセッションメッセージングを強化：他の Claude セッションから `SendMessage` 経由でリレーされたメッセージはユーザー権限を持たなくなった — 受信側はリレーされた権限リクエストを拒否し、auto モードではブロックする
- `MAX_THINKING_TOKENS=0`、`--thinking disabled`、およびモデルごとの thinking トグルが、Claude API 経由でデフォルトで thinking するモデルの thinking を無効化できるようになった（サードパーティプロバイダーは変更なし）
- 予期しない非リトライ可能エラーが API から返された場合、フォールバックモデルで1回リトライするようになった；認証、レート制限、リクエストサイズ、トランスポートエラーはすぐに表示される
- `claude update` がダウンロード前にターゲットバージョンを通知するようになった（以前はサイレントだった）
- `claude agents`：リストへの URL 入力時、最初のプロンプトにその URL が含まれるセッションにフィルタリングするようになった
- セッションで処理できない画像を送信した際に繰り返し発生していた「image could not be processed」エラーと余分なトークン使用を修正
- 起動時のワーカー登録中に短いバックエンド障害が発生した場合にリモートセッションが永続的にスタックする問題を修正
- JetBrains IDE ターミナル（IntelliJ、PyCharm、WebStorm など）2026.1+ でシンクロナイズド出力を有効化することでちらつきを修正
- Kitty キーボードプロトコルを使用するターミナル（WezTerm、Ghostty、kitty）で Shift+非 ASCII 文字（例：Shift+ä → Ä）がドロップされる問題を修正
- Windows で killed プロセスの子プロセスが出力パイプを保持していた場合に PowerShell コマンド検証がタイムバジェットをはるかに超えてハングすることがある問題を修正
- macOS でデーモンが接続中に終了した後、孤立した `claude --bg-pty-host` プロセスが 100% CPU でスピンする問題を修正
- `/voice` をトグルした後に古い認証チェックをクリアするために voice モードが `/login` を必要とする問題を修正
- 無効なエントリを持つ managed 設定が残りの有効なポリシーの適用をサイレントに無効化する問題を修正
- managed-settings の `allowedMcpServers`/`deniedMcpServers` 述語が `${VAR}` 参照を使用している場合にマッチしない問題を修正
- git worktree に入った background agent セッションが `claude agents` から再度開いた際に「No conversation found」でクラッシュループする問題を修正
- ストリーミング中に Ctrl+O トランスクリプトビューで thinking テキストが重複する問題を修正
- `/doctor` をリモートセッション内で実行した際に矛盾した失敗「Not inside a remote session」チェックが表示される問題を修正
- `claude agents` のディスパッチおよび返信入力でマルチラインプロンプトを入力する際にカーソルが最初の行の末尾に固定される問題を修正
- Unicode サポートのないターミナルでタスクリストの background agent 行間に空白行が表示される問題を修正
