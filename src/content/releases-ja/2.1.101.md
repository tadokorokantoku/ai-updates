- ローカルの Claude Code 利用状況からチームメイト用の立ち上げガイドを生成する `/team-onboarding` コマンドを追加
- OS の CA 証明書ストアのトラストをデフォルトで有効化。追加設定なしでエンタープライズ TLS プロキシが動作（バンドルされた CA のみ使用する場合は `CLAUDE_CODE_CERT_STORE=bundled` を設定）
- `/ultraplan` などのリモートセッション機能が、Web セットアップを先に行わずにデフォルトのクラウド環境を自動作成するように
- brief モードで Claude がプレーンテキストで応答したときに構造化メッセージを 1 回リトライするように改善
- focus モードを改善: ユーザーが最終メッセージしか見ないことを Claude が把握しているため、より自己完結的な要約を書くように
- 現在のコンテキストで利用できないツールをモデルが呼び出した場合、その理由と進め方を説明するよう tool-not-available エラーを改善
- レート制限リトライメッセージを改善し、不透明な秒単位のカウントダウンではなく、どの制限に到達したかとリセット時刻を表示
- API が提供する説明が利用可能な場合、拒否エラーメッセージに含めるように改善
- `claude -p --resume <name>` を改善し、`/rename` や `--name` で設定されたセッションタイトルを受け付けるように
- 設定の耐障害性を改善。`settings.json` に認識できないフックイベント名があっても、ファイル全体が無視されないように
- 管理設定で強制有効化されたプラグインからのプラグインフックを、`allowManagedHooksOnly` 設定時に実行するように改善
- マーケットプレイスが更新できなかったときに古いバージョンを静かに報告するのではなく警告を表示するように `/plugin` と `claude plugin update` を改善
- ユーザーの組織または認証設定が Web 上の Claude Code に到達できない場合、plan モードで「Refine with Ultraplan」オプションを非表示にするように改善
- beta tracing が `OTEL_LOG_USER_PROMPTS`、`OTEL_LOG_TOOL_DETAILS`、`OTEL_LOG_TOOL_CONTENT` を尊重するように改善。機密な span 属性はオプトインしない限り出力されないように
- SDK の `query()` を改善し、consumer が `for await` から `break` したり `await using` を使う場合にサブプロセスと一時ファイルをクリーンアップ
- LSP バイナリ検出で使用される POSIX の `which` フォールバックにおけるコマンドインジェクションの脆弱性を修正
- 長時間のセッションで仮想スクロール内のメッセージリストの履歴コピーが数十個保持されるメモリリークを修正
- ローダーがライブの会話ではなく行き止まりのブランチにアンカーされたときに、大きなセッションで `--resume`/`--continue` が会話コンテキストを失う問題を修正
- サブエージェントメッセージがメインチェーンの書き込みギャップ付近に着地したときに、`--resume` チェーンの回復が無関係なサブエージェント会話につながる問題を修正
- 永続化された Edit/Write ツール結果に `file_path` が欠落している場合の `--resume` 時のクラッシュを修正
- `API_TIMEOUT_MS` に関係なく、低速のバックエンド（ローカル LLM、拡張 thinking、低速ゲートウェイ）を中断していたハードコードされた 5 分のリクエストタイムアウトを修正
- `permissions.deny` ルールが PreToolUse フックの `permissionDecision: "ask"` を上書きしない問題を修正。以前はフックが deny をプロンプトに降格できた
- `--setting-sources` に `user` がないと、バックグラウンドクリーンアップが `cleanupPeriodDays` を無視して 30 日以上前の会話履歴を削除する問題を修正
- `ANTHROPIC_AUTH_TOKEN`、`apiKeyHelper`、`ANTHROPIC_CUSTOM_HEADERS` が Authorization ヘッダを設定する場合に Bedrock SigV4 認証が 403 で失敗する問題を修正
- 前のセッションのワークツリークリーンアップで古いディレクトリが残ると `claude -w <name>` が「already exists」で失敗する問題を修正
- サブエージェントが動的に注入されたサーバーから MCP ツールを継承しない問題を修正
- 分離されたワークツリーで実行されているサブエージェントが、自身のワークツリー内のファイルに対する Read/Edit アクセスを拒否される問題を修正
- 新規ブート後に sandbox 内の Bash コマンドが `mktemp: No such file or directory` で失敗する問題を修正
- `outputSchema` を検証する MCP クライアントで `claude mcp serve` のツール呼び出しが「Tool execution failed」で失敗する問題を修正
- `RemoteTrigger` ツールの `run` アクションが空のボディを送信してサーバーに拒否される問題を修正
- いくつかの `/resume` ピッカーの問題を修正: デフォルトのビューが狭くて他のプロジェクトのセッションが隠れる、Windows Terminal でプレビューが到達不能、ワークツリーで cwd が誤る、session-not-found エラーが stderr に出ない、ターミナルタイトルが設定されない、resume ヒントがプロンプト入力に重なる
- 埋め込み ripgrep バイナリパスが古くなった場合（VS Code 拡張機能の自動更新、macOS App Translocation）の Grep ツール ENOENT を修正。システムの `rg` にフォールバックしてセッション中に自己修復
- `/btw` が使用のたびに会話全体のコピーをディスクに書き込む問題を修正
- `/context` の Free space と Messages 内訳がヘッダの割合と一致しない問題を修正
- いくつかのプラグインの問題を修正: 重複する `name:` frontmatter でスラッシュコマンドが間違ったプラグインに解決される、`/plugin update` が `ENAMETOOLONG` で失敗する、Discover にインストール済みプラグインが表示される、ディレクトリソースプラグインが古いバージョンキャッシュからロードされる、スキルが `context: fork` と `agent` frontmatter フィールドを尊重しない
- `headersHelper` で設定された MCP サーバーに対して `/mcp` メニューが OAuth 固有のアクションを提供する問題を修正。代わりにヘルパースクリプトを再呼び出しする Reconnect を提供
- 生の C0 制御バイトを送信するターミナル（Terminal.app、デフォルトの iTerm2、xterm）で `ctrl+]`、`ctrl+\`、`ctrl+^` のキーバインドが発火しない問題を修正
- `/login` OAuth URL がクリーンなマウス選択を妨げるパディング付きでレンダリングされる問題を修正
- レンダリングの問題を修正: 非全画面モードで表示領域外のコンテンツが変更されたときのちらつき、非全画面モードでの長時間セッション中にターミナルのスクロールバックがワイプされる、マウススクロールのエスケープシーケンスがテキストとしてプロンプトに漏れる
- `settings.json` の env 値が文字列ではなく数値の場合のクラッシュを修正
- アプリ内の設定書き込み（例: `/add-dir --remember`、`/config`）がメモリ内スナップショットを更新せず、セッション中に削除されたディレクトリが取り消されない問題を修正
- カスタムキーバインド（`~/.claude/keybindings.json`）が Bedrock、Vertex、その他のサードパーティプロバイダーでロードされない問題を修正
- `claude --continue -p` が `-p` または SDK で作成されたセッションを正しく継続しない問題を修正
- いくつかの Remote Control の問題を修正: セッションクラッシュ時にワークツリーが削除される、接続失敗がトランスクリプトに永続化されない、ローカルセッションの brief モードで不正確な「Disconnected」インジケーターが表示される、`CLAUDE_CODE_ORGANIZATION_UUID` のみが設定されているときに SSH 経由で `/remote-control` が失敗する
- `/insights` がレポートファイルリンクを応答から省略することがある問題を修正
- [VSCode] 最後のエディタタブを閉じてもチャット入力下のファイル添付がクリアされない問題を修正
