- `/clear` 実行前の状態から会話を再開できる `/rewind` サポートを追加
- ストリーミングレスポンス中に以前の出力を読んでいるときにスクロール位置が最下部へジャンプする問題を修正
- 停止後にバックグラウンドエージェントが復活する問題を修正 — タスクパネルからエージェントを停止すると恒久的に停止するようになった
- `/voice` が組織のポリシーで無効化されているときに汎用の「利用不可」メッセージを表示していた問題を修正 — 制限内容を説明するようになった
- Windows Terminal で `/login` の URL が折り返して切り詰められる問題を修正
- Ghostty over ssh/tmux でフルスクリーンモード時の Cmd+クリックリンク問題を修正
- `claude agents` がバックグラウンドセッションへ `/usage` などの組み込みスラッシュコマンドをプロンプトテキストとして送信していた問題を修正 — ヒントを表示するようになった
- `claude agents` のジョブ行で貼り付け画像のファイルシステムフルパスが `[Image #N]` プレースホルダーの代わりに表示されていた問題を修正
- コンマ区切りマッチャー（例: `"Bash,PowerShell"`）を持つフックがサイレントに発火しない問題を修正
- `/permissions` の「最近拒否した」タブ: 拒否の承認がクローズ時に保持されず破棄されていた問題を修正
- エージェントパネルのロスターをオーバーフローキャップを超えてスクロールしたときに1行ジャンプする問題を修正
- ウェルカムスプラッシュアートがデフォルトの 80×24 macOS Terminal ウィンドウからはみ出す問題を修正
- 管理設定の修正: `forceRemoteSettingsRefresh` が MDM またはファイルポリシー経由で設定されたときに反映されるようになり、フェッチが `Cache-Control: no-cache` を送信してプロキシが古いレスポンスを返すのを防ぐようになった
- サンドボックスネットワーク許可ダイアログの改善: 「Yes」で許可したホストはセッション終了まで記憶され、接続のたびに再プロンプトされなくなった
- MCP サーバーの信頼性向上: 機能検出（`tools/list`、`prompts/list`、`resources/list`）が短いバックオフで一時的なネットワークエラーをリトライするようになった
- MCP OAuth の改善: 検出とトークンリクエストが一時的なネットワークエラー後に1回リトライするようになり、ヘッドレス環境ではブラウザポップアップをスキップして URL を貼り付けるプロンプトに直接移行するようになった
- MCP エラーメッセージの改善: HTTP 404 エラーに URL と MCP 設定へのポインタが表示されるようになった
- vim モードのプロンプト履歴検索（NORMAL `/`）でスラッシュコマンドへのアクセス方法をヒント表示するよう改善
- テキスト更新を 100ms にまとめることでストリーミングレスポンス中の CPU 使用率を約37% 削減
- ターミナル出力キャッシュによる長時間セッションのメモリ増加を削減
