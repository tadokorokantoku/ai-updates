- ログイン画面で「3rd-party platform」を選択したときにアクセスできるインタラクティブな Bedrock セットアップウィザードを追加 — AWS 認証、リージョン設定、認証情報検証、モデル pinning をガイド
- サブスクリプションユーザー向けに `/cost` にモデルごととキャッシュヒットの内訳を追加
- `/release-notes` がインタラクティブなバージョンピッカーに
- Remote Control セッション名がデフォルトプレフィックスとしてホスト名を使用するように（例: `myhost-graceful-unicorn`）。`--remote-control-session-name-prefix` で上書き可能
- プロンプトキャッシュが期限切れになった後にセッションに戻るとき、次のターンでおよそ何トークンがキャッシュされずに送信されるかを表示するフッターヒントが Pro ユーザーに表示されるように
- 長時間実行セッション中に tmux ウィンドウが kill または renumber された後、サブエージェントの spawn が "Could not determine pane count" で永続的に失敗する問題を修正
- 小型の fast モデルが `ok:false` を返したときにプロンプトタイプの Stop フックが誤って失敗する問題を修正。非 Stop プロンプトタイプフックの `preventContinuation:true` セマンティクスを復元
- ストリーミングが配列/オブジェクトフィールドを JSON エンコードされた文字列として発行するときのツール入力検証失敗を修正
- 拡張 thinking が実際のコンテンツと並行して空白のみのテキストブロックを生成したときに発生する API 400 エラーを修正
- auto-pilot のキー入力と連続プロンプトの digit 衝突による意図しないフィードバック調査送信を修正
- フルスクリーンモードでテキスト選択が存在するときに、処理中に「esc to clear」と並んで誤解を招く「esc to interrupt」ヒントが表示される問題を修正
- cask のリリースチャンネル（`claude-code` → stable、`claude-code@latest` → latest）を使用するよう Homebrew インストール更新プロンプトを修正
- 既に行末にあるときの multiline プロンプトで `ctrl+e` が次の行の末尾にジャンプする問題を修正
- フルスクリーンモードで上にスクロールするときに同じメッセージが 2 つの位置に表示される可能性がある問題を修正（iTerm2、Ghostty、その他の DEC 2026 サポートターミナル）
- アイドル復帰の "/clear to save X tokens" ヒントが現在のコンテキストサイズではなく累積セッショントークンを表示する問題を修正
- 認証されていない claude.ai コネクタを複製するプラグイン MCP サーバーが、セッション開始時に "connecting" でスタックする問題を修正
- 大きなファイル（タブ/`&`/`$` を含むファイルで 60% 高速）の Write ツールの diff 計算速度を改善
- `/tag` コマンドを削除
- `/vim` コマンドを削除（`/config` → Editor mode 経由で vim モードを切り替え）
- Linux サンドボックスに npm とネイティブビルドの両方で `apply-seccomp` ヘルパーを同梱し、サンドボックス化されたコマンドの unix-socket ブロッキングを復元
