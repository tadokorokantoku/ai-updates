# /copy

`/copy` コマンドを全ユーザーに開放

## 関連リリース

### v2.1.113

- `/copy` の "Full response" が GitHub、Notion、Slack に貼り付ける際に markdown テーブル列を整列しない問題を修正
### v2.1.77

- `/copy` がオプションのインデックスを受け付けるように: `/copy N` で N 番目に新しいアシスタント応答をコピー
### v2.1.72

- `/copy` に `w` キーを追加。フォーカスされた選択をクリップボードを経由せずに直接ファイルに書き込み可能（SSH 経由で便利）
### v2.1.63

- `/copy` ピッカーに "Always copy full response" オプションを追加。選択すると、以降の `/copy` コマンドはコードブロックピッカーをスキップしてフルレスポンスを直接コピー
### v2.1.59

- コードブロックが存在する場合にインタラクティブピッカーを表示し、個別のコードブロックまたはフルレスポンスを選択可能な `/copy` コマンドを追加
### v2.1.20

- `/copy` コマンドを全ユーザーに開放
