# /clear

`/clear` コマンドをまたいで plan ファイルが残り続ける問題を修正。会話クリア後は新しい plan ファイルが使われるように

## 関連リリース

### v2.1.111

- `/clear` が `/rename` で設定したセッション名を破棄し、statusline 出力から `session_name` が消える問題を修正
### v2.1.84

- 75分以上経過してから戻ってきたユーザーに `/clear` を促す idle-return プロンプトを追加。古いセッションでの不要なトークン再キャッシュを削減
### v2.1.72

- バックグラウンド化した Ctrl+B クエリがトランスクリプトを失ったり、`/clear` 後に新しい会話を破損させる問題を修正
- `/clear` がバックグラウンドの agent/bash タスクを強制終了する問題を修正 — フォアグラウンドタスクのみクリアされるように
### v2.1.69

- 親の会話履歴全体が teammate の生存期間中ピン留めされ、`/clear` または auto-compact 後も GC されない in-process teammate のメモリ保持問題を修正
- `/clear` がすべてのセッションキャッシュを完全にクリアしない問題を修正。長時間セッションでのメモリ保持を削減
### v2.1.63

- `/clear` がキャッシュされた skill をリセットしない問題を修正。古い skill の内容が新しい会話に残る可能性があった
### v2.1.49

- `/clear` などのコマンドで始まるセッションで `--resume` セッションピッカーが生の XML タグを表示する問題を修正。セッション ID フォールバックに正しく遷移するように
### v2.1.47

- `/clear` コマンド実行後、ステータスバーにセッション名が残る問題を修正 (anthropics/claude-code#26082)
### v2.1.3

- `/clear` コマンドをまたいで plan ファイルが残り続ける問題を修正。会話クリア後は新しい plan ファイルが使われるように
