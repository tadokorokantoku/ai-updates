# /plugin

`/plugin` がネイティブビルドで動作しない問題を修正

## 関連リリース

### v2.1.110

- `/plugin` の Installed タブを改善。注意が必要な項目とお気に入りを上部に表示、無効化された項目は折りたたみ内に隠し、`f` キーで選択項目をお気に入り登録
- マーケットプレイス登録情報が省略していても、`plugin.json` に宣言された依存関係を尊重するようにプラグインインストールを修正。`/plugin` のインストールで自動インストールされる依存関係を一覧表示
### v2.1.101

- `/plugin` と `claude plugin update` を改善。マーケットプレイスを更新できなかった場合に警告を表示し、古いバージョンを黙って表示しないように変更
### v2.1.86

- `/plugin` のアンインストールダイアログを修正。`n` を押すとプラグインのデータディレクトリを保持したまま正しくアンインストール
### v2.1.75

- 管理者により無効化されたプラグインが `/plugin` の Installed タブに表示される問題を修正。組織により強制的に無効化されたプラグインは非表示に
### v2.1.71

- プラグイン提供の MCP サーバーの重複排除を改善。手動設定されたサーバーと重複するサーバー (同一コマンド/URL) をスキップし、接続やツールセットの重複を防止。抑制されたものは `/plugin` メニューに表示
### v2.1.70

- プラグインが `/plugin` で不正確にインストール済み表示される問題を修正
### v2.1.69

- `/plugin` 操作後にプラグインの Stop/SessionEnd などのフックが発火しない問題を修正
### v2.0.13

- `/plugin` がネイティブビルドで動作しない問題を修正
